# usagi — 画面設計書

`usagi hop` で起動する TUI の画面設計をまとめます。本書は実装状況を反映しており、
**実装済み** と **予定（usagi.ai から移植）** を明示して区別します。全体像は
[overview.md](./overview.md) を参照してください。

## 画面一覧

| # | 画面 | 役割 | 状態 |
|---|---|---|---|
| 1 | 起動画面（Launch） | アプリ起動時のトップメニュー。Open / New / Config / Quit を選択する | ✅ 実装済み |
| 2 | プロジェクト選択画面（Open） | 登録済みワークスペースの一覧から開く対象を選ぶ | ✅ 実装済み |
| 3 | 新規プロジェクト画面（New） | リポジトリ URL を入力して `init` を実行する | ✅ 実装済み |
| 4 | 設定画面（Config） | グローバル設定を編集する | 🔜 予定 |
| 5 | ホーム画面（Home） | 選択したワークスペースの worktree 一覧を表示するメイン操作画面 | ✅ 実装済み（worktree 一覧） |

> 現時点で実装されている画面は **起動画面（Launch）**・**プロジェクト選択画面（Open）**・
> **新規プロジェクト画面（New）**・**ホーム画面（Home）** の 4 つです。Open でワークスペースを
> 選択するとホーム画面へ遷移し、そのワークスペースの worktree 一覧を表示します。ホーム画面で
> worktree を選択する操作（セッション画面への遷移）は未実装で、選択すると「coming soon」通知を
> 表示します。起動画面で Config を選ぶと「coming soon」通知を表示するプレースホルダーになっています。

## 画面遷移図

```text
                 [起動画面 Launch]
                       │
      ┌────────┬───────┼────────┬────────┐
   Open(o)   New(e)  Config(c)  Quit(q) / Esc / Ctrl+C
      │        │        │           │
      ▼        ▼        ▼           ▼
 [プロジェクト [新規      [設定        アプリ終了
   選択画面]   プロジェクト 画面]      （"USAGI run away
      │        画面]      (予定)        ( ^-^)ノ" を表示）
      │（ワークスペース選択）
      ▼
 [ホーム画面 Home] ──(Esc)──▶ プロジェクト選択画面へ戻る
```

※ Open を選ぶとプロジェクト選択画面へ遷移し、ワークスペースを選択するとホーム画面へ進みます。
ホーム画面で `Esc` を押すとプロジェクト選択画面へ戻ります。New を選ぶと新規プロジェクト画面へ
遷移します。Config からの遷移先は予定で、現状は通知表示にとどまります。

## 1. 起動画面（Launch）✅ 実装済み

`usagi hop` 実行時に最初に表示される画面。代替スクリーン（alternate screen）上に描画され、
終了時に元の端末状態へ復帰します。

### レイアウト

端末サイズに応じて全要素を水平方向に中央寄せして描画します（幅・高さが取得できない環境では
80×24 をフォールスバック値として使用）。

```text
┌────────────────────────────────────────────────────┐
│                                                      │  ← 上部パディング
│                       (\(\                           │  ┐
│                      (='-')                          │  ├ うさぎ AA（マゼンタ・太字）
│                      o(_(")(")                        │  ┘
│                                                      │
│                       USAGI                          │  ← タイトル（緑・太字）
│                                                      │
│                 > Open           o                   │  ┐
│                                                      │  │
│                   New            e                   │  ├ メニュー（選択行は
│                                                      │  │   赤の ">" + シアン太字ラベル）
│                   Config         c                   │  │
│                                                      │  │
│                   Quit           q                   │  ┘
│                                                      │
│              New is coming soon 🐰                   │  ← 通知行（任意・黄色）
│                                                      │
│      v0.1.0 | ↑↓: move / Enter: select / q: quit     │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

### 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| うさぎ AA | 3 行のアスキーアート `(\(\` / `(='-')` / `o(_(")(")`| マゼンタ・太字 |
| タイトル | `USAGI` | 緑・太字 |
| メニュー | Open / New / Config / Quit の 4 項目。各行は「カーソル + ラベル(10桁左詰め) + キー(5桁右詰め)」 | 選択行：`>` 赤太字／ラベル シアン太字／キー 黄色。非選択行：素のテキスト |
| 通知行 | メニュー選択結果のメッセージ（例: `Open is coming soon 🐰`）。未表示時は何も描画しない | 黄色 |
| フッター | バージョンと操作ヘルプ `v<version> \| ↑↓: move / Enter: select / q: quit` | 淡色（dim） |

### メニュー項目

| ラベル | キー | 動作 | 状態 |
|---|---|---|---|
| Open | `o` | プロジェクト選択画面へ遷移 | ✅ 実装済み |
| New | `e` | 新規プロジェクト画面へ遷移 | ✅ 実装済み |
| Config | `c` | 設定画面へ（予定）。現状は通知表示 | プレースホルダー |
| Quit | `q` | アプリを終了する | ✅ 実装済み |

### キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `Enter` | 選択中の項目を実行。Quit なら終了、それ以外は通知を表示 |
| `o` / `e` / `c` | 対応するメニュー項目を直接実行（ショートカット） |
| `q` / `Esc` / `Ctrl+C` | アプリを終了 |

### 終了時の挙動

- 正常終了（Quit / `q` / `Esc` / `Ctrl+C`）時：代替スクリーンを解除しカーソルを復帰したうえで、
  別れのメッセージ `USAGI run away ( ^-^)ノ` を表示します。
- キー読み取りが中断された場合（シグナル等）：終了として扱います。
- キー読み取りでエラーが発生した場合：別れのメッセージは出さずに端末を復帰し、エラーを返します。

これらの端末復帰は `AlternateScreenGuard`（RAII ガード）により、早期 return や
パニック時でも保証されます。

## 3. 新規プロジェクト画面（New）✅ 実装済み

起動画面で `New`（`e` / Enter）を選ぶと遷移する、リポジトリ URL を入力するフォーム画面。
VS Code の「Clone Git Repository…」や IntelliJ の「Get from Version Control」といった
エディタのクローン画面を参考に、**URL を入力するとディレクトリ名が自動で導出される** UX を採用しています。
起動画面と同じ代替スクリーン上に描画されます。

### レイアウト

フォーム全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。

```text
┌────────────────────────────────────────────────────┐
│                       (\(\                           │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                          │  │
│                      o(_(")(")                        │  ┘
│                    New Project                       │  ← タイトル（緑・太字）
│        Clone a Git repository into a new workspace   │  ← サブタイトル（淡色）
│                                                      │
│        Repository URL                                │  ← ラベル（淡色）
│      > https://github.com/owner/repo.git▏            │  ← 入力欄（フォーカス: 赤">" + シアン太字 + キャレット）
│                                                      │
│        Directory                                     │
│        repo                                          │  ← URL から自動導出（シアン）
│                                                      │
│        Branch (optional)                             │
│        repository default                            │  ← 未入力時はプレースホルダー（淡色斜体）
│                                                      │
│        enter a repository URL                        │  ← 通知行（任意・赤太字／バリデーション失敗時）
│                                                      │
│      ↑↓/Tab: move field / Enter: create / Esc: back  │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

### 入力フィールド

| フィールド | 役割 | 備考 |
|---|---|---|
| Repository URL | クローン対象の Git URL | HTTPS / `git@host:owner/repo.git`（scp 形式）/ `ssh://` に対応 |
| Directory | クローン先ディレクトリ名 | URL の末尾セグメント（`.git`・末尾スラッシュ除去）から自動導出。手動編集も可能 |
| Branch (optional) | チェックアウトするブランチ | 未入力時はリポジトリのデフォルトブランチ |

### ディレクトリ名の自動導出（エディタ準拠の挙動）

- URL を入力するたびに Directory を `末尾セグメント` から再導出します
  （例: `https://github.com/owner/repo.git` → `repo`）。
- ユーザーが Directory を手動で編集すると自動導出を停止し、以降の URL 変更で上書きしません。
- Directory を空になるまで削除すると自動導出が再び有効化され、次の URL 変更で復活します
  （その場で即座に再充填はせず、ユーザーが独自名を入力できる余地を残します）。
- URL の末尾がスラッシュの場合（例: `owner/` 入力途中）はセグメントが未確定とみなし、何も提案しません。

### キー操作

| キー | 動作 |
|---|---|
| 文字キー | フォーカス中のフィールドに追記 |
| `Backspace` | フォーカス中のフィールドの末尾を 1 文字削除 |
| `Tab` / `↓` | 次のフィールドへフォーカス移動 |
| `Shift+Tab` / `↑` | 前のフィールドへフォーカス移動 |
| `Enter` | フォームを検証して確定。失敗時は赤い通知を表示 |
| `Esc` | 起動画面へ戻る |
| `Ctrl+C` | アプリを終了 |

### バリデーションと確定後の挙動

- URL が空、または URL らしき形（パス区切りと末尾セグメント）を持たない場合は確定せず、
  通知行にエラーメッセージを表示します。
- 検証に成功すると起動画面へ戻り、`Ready to init "<directory>" from <url> 🐰` の通知を表示します。
  実際のクローン + 初期化（`usagi init` 相当）の実行は今後の作業（画面 #5 と連携）で実装予定です。

> 描画ロジックは `src/presentation/tui/new/`、URL のパース・ディレクトリ名導出は
> `src/domain/repository.rs` に実装されています。

## 2. プロジェクト選択画面（Open）✅ 実装済み

起動画面で `Open`（`o` / Enter）を選ぶと遷移する、登録済みワークスペースの一覧から
開く対象を選ぶ画面。一覧は `workspaces.json` 由来で、最終更新の新しい順に並びます。
起動画面と同じ代替スクリーン上に描画されます。

> ワークスペースを選択するとホーム画面（画面 #5）へ遷移し、そのワークスペースの worktree 一覧を
> 表示します。ホーム画面で `Esc` を押すとこのプロジェクト選択画面へ戻ります。

### レイアウト

リスト全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。

```text
┌────────────────────────────────────────────────────┐
│                       (\(\                           │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                          │  │
│                      o(_(")(")                        │  ┘
│                    Open Project                      │  ← タイトル（緑・太字）
│              Open a registered workspace             │  ← サブタイトル（淡色）
│                                                      │
│      > my-project    /path/to/my-project             │  ┐ プロジェクト一覧
│        other-repo    /path/to/other-repo             │  ┘  選択行：赤 ">" + 名前シアン太字
│                                                      │     パスは淡色（長い場合は先頭を … で省略）
│                                                      │  ← 通知行（任意・黄色／読み込み失敗時など）
│                                                      │
│        ↑↓: move / Enter: open / Esc: back            │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

ワークスペースが 1 つも登録されていない場合は、一覧の代わりに
`No workspaces yet. Choose "New" to add one.`（淡色）を中央に表示します。

### 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| タイトル | `Open Project` | 緑・太字 |
| サブタイトル | `Open a registered workspace` | 淡色（dim） |
| プロジェクト一覧 | 登録済みワークスペースを「カーソル + 名前(幅揃え) + パス」で 1 行ずつ表示。パスはブロック幅に収まるよう先頭を省略 | 選択行：`>` 赤太字／名前 シアン太字。パスは常に淡色 |
| 空表示 | 登録が無い場合のメッセージ | 淡色（dim） |
| 通知行 | 読み込み失敗時などのメッセージ。未表示時も 1 行ぶんの枠を確保しレイアウトを安定させる | 黄色 |
| フッター | 操作ヘルプ `↑↓: move / Enter: open / Esc: back` | 淡色（dim） |

### キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `Enter` | 選択中のワークスペースを開き、ホーム画面（画面 #5）へ遷移する |
| `q` / `Esc` | 起動画面へ戻る |
| `Ctrl+C` | アプリを終了 |

### 読み込みと確定後の挙動

- ワークスペースの読み込みに失敗した場合は、空一覧の状態でエラー内容を通知行に表示します。
- `Enter` でワークスペースを選択するとホーム画面へ遷移します。ホーム画面から戻る（`Esc`）と
  この一覧へ復帰し、アプリ終了（`Ctrl+C`）はそのまま上位へ伝播します。

> 描画ロジックは `src/presentation/tui/open/`（一覧状態 `state.rs`・描画 `ui.rs`・
> イベントループ `event.rs` に分離）に実装されています。

## 5. ホーム画面（Home）✅ 実装済み（worktree 一覧）

プロジェクト選択画面でワークスペースを選ぶと遷移する、選択したワークスペースの
worktree 一覧を表示するメイン操作画面。worktree の情報はワークスペースの
`<workspace>/.usagi/state.json`（画面外で `usagi` が同期した内容）から読み込みます。
起動画面と同じ代替スクリーン上に描画されます。

### レイアウト

一覧全体を 1 つのブロック（幅 56）として端末中央に寄せ、ブロック内は左揃えで描画します。

```text
┌────────────────────────────────────────────────────┐
│                       (\(\                           │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                          │  │
│                      o(_(")(")                        │  ┘
│                       usagi                          │  ← タイトル＝ワークスペース名（緑・太字）
│                    3 worktrees                       │  ← サブタイトル＝worktree 数（淡色）
│                                                      │
│      > ● main       pushed  76e906f                  │  ┐ worktree 一覧
│        feature/x    local   12f0c5a                  │  │  選択行：赤 ">"、primary は ● マゼンタ
│        fix/y        merged  d7a1850                  │  ┘  ブランチ名 シアン、status 色分け、hash 淡色
│                                                      │
│        Opening "main" is coming soon 🐰              │  ← 通知行（任意・黄色）
│                                                      │
│        ↑↓: move / Enter: open / Esc: back            │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

worktree の記録が無い（`state.json` 未生成など）場合は、一覧の代わりに
`No worktrees recorded yet. Run usagi to sync.`（淡色）を中央に表示します。

### 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| タイトル | 選択したワークスペース名 | 緑・太字 |
| サブタイトル | worktree 数（`N worktree` / `N worktrees`） | 淡色（dim） |
| worktree 一覧 | 「カーソル + primary マーカー + ブランチ名(幅揃え) + status + 短縮ハッシュ」で 1 行ずつ表示。ブランチ名は 24 桁を超えると末尾を省略 | 選択行：`>` 赤太字／primary：`●` マゼンタ／ブランチ名 シアン（選択行は太字）／status は色分け／ハッシュ 淡色 |
| status | ブランチの状態 `local` / `pushed` / `merged` | `local`：黄色／`pushed`：緑／`merged`：淡色 |
| 空表示 | worktree の記録が無い場合のメッセージ | 淡色（dim） |
| 通知行 | 選択結果・読み込み失敗時のメッセージ（例: `Opening "main" is coming soon 🐰`）。未表示時も 1 行ぶんの枠を確保しレイアウトを安定させる | 黄色 |
| フッター | 操作ヘルプ `↑↓: move / Enter: open / Esc: back` | 淡色（dim） |

### キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `Enter` | 選択中の worktree を開く（現状は「coming soon」通知を表示） |
| `q` / `Esc` | プロジェクト選択画面へ戻る |
| `Ctrl+C` | アプリを終了 |

### 読み込みと今後の予定

- worktree 状態の読み込みに失敗した場合は、空一覧の状態でエラー内容を通知行に表示します。
- 最終形ではこの画面を、worktree 一覧（左ペイン）・コマンド履歴/出力（右ペイン）・
  コマンド入力欄（下部）の 3 ペイン構成へ拡張し、サイドメニューモードと
  コマンドモード（TUI 内コマンド `session` / `space` / `ai` / `terminal` / `history` /
  `doctor` / `man`、Tab 補完・履歴遡り付き）を切り替えて操作できるようにする予定です。
  worktree を選択したあとのセッション画面への遷移も今後の作業で実装します。

```text
┌──────────────┬───────────────────────────────┐
│ worktree 一覧 │ コマンド履歴 / 出力            │  ← 最終形（予定）の 3 ペイン構成
│ (左ペイン)    │ (右ペイン)                     │
│              │                                │
├──────────────┴───────────────────────────────┤
│ > コマンド入力欄                              │
└───────────────────────────────────────────────┘
```

> 描画ロジックは `src/presentation/tui/home/`（一覧状態 `state.rs`・描画 `ui.rs`・
> イベントループ `event.rs` に分離）に実装されています。

## 4. 設定画面（Config）🔜 予定

グローバル設定の閲覧・編集を行う画面。usagi.ai から移植予定で、詳細仕様は移植時に本書へ追記します。

## 設計メモ

- 配色には `console` クレートのスタイル（magenta / green / cyan / yellow / red / dim）を使用。
- 中央寄せは「端末幅 − コンテンツ幅」を 2 で割った左パディングで実現（`centered_padding`）。
  端末が狭い場合は `saturating_sub` によりパディング 0 にフォールバックします。
- 起動画面の描画ロジックは `src/presentation/tui/launch/`、プロジェクト選択画面は
  `src/presentation/tui/open/`、新規プロジェクト画面は `src/presentation/tui/new/`、
  ホーム画面は `src/presentation/tui/home/`（いずれも状態 `state.rs` と描画 `ui.rs`・
  イベントループ `event.rs` に分離）、端末制御は `src/presentation/tui/screen.rs` に
  実装されています。
