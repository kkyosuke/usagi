# usagi — 画面設計書

`usagi hop` で起動する TUI の画面設計をまとめます。本書は実装状況を反映しており、
**実装済み** と **予定（usagi.ai から移植）** を明示して区別します。全体像は
[overview.md](./overview.md) を参照してください。

## 画面一覧

| # | 画面 | 役割 | 状態 |
|---|---|---|---|
| 1 | 起動画面（Welcome） | アプリ起動時のトップメニュー。Open / New / Config / Quit を選択する | ✅ 実装済み |
| 2 | プロジェクト選択画面（Open） | 登録済みワークスペースの一覧から開く対象を選ぶ | ✅ 実装済み |
| 3 | 新規プロジェクト画面（New） | リポジトリのクローン、または既存ディレクトリの登録でワークスペースを追加する | ✅ 実装済み |
| 4 | 設定画面（Config） | グローバル設定を編集する | ✅ 実装済み |
| 5 | ホーム画面（Home） | 選択したワークスペースの worktree 一覧とコマンド入力を備えたメイン操作画面 | ✅ 実装済み（3 ペイン + コマンドモード） |

> 現時点で実装されている画面は **起動画面（Welcome）**・**プロジェクト選択画面（Open）**・
> **新規プロジェクト画面（New）**・**設定画面（Config）**・**ホーム画面（Home）** の 5 つです。
> Open でワークスペースを選択するとホーム画面へ遷移し、左ペインにそのワークスペースの worktree
> 一覧、右ペインにコマンド履歴・出力、下部にコマンド入力欄を表示します。`:`（または `i`）で
> コマンドモードに入り、TUI 内コマンドを実行できます。worktree を選択する操作（セッション画面への
> 遷移）と一部のコマンド（`session` / `space` / `ai` / `terminal` / `doctor`）はプレースホルダーで、
> 実行すると「coming soon」を表示します。起動画面で Config を選ぶと設定画面へ遷移し、テーマ・
> 既定ワークスペース・通知の ON/OFF・使用する Agent CLI（Claude / Gemini）を編集できます。

## 画面遷移図

```text
                 [起動画面 Welcome]
                       │
      ┌────────┬───────┼────────┬────────┐
   Open(o)   New(e)  Config(c)  Quit(q) / Esc / Ctrl+C
      │        │        │           │
      ▼        ▼        ▼           ▼
 [プロジェクト [新規      [設定        アプリ終了
   選択画面]   プロジェクト 画面] ──(Esc)──▶ 起動画面へ戻る
      │        画面]                （"USAGI run away
      │（ワークスペース選択）          ( ^-^)ノ" を表示）
      ▼
 [ホーム画面 Home] ──(Esc / q)──▶ プロジェクト選択画面へ戻る
   （3 ペイン + コマンドモード）
```

※ Open を選ぶとプロジェクト選択画面へ遷移し、ワークスペースを選択するとホーム画面へ進みます。
ホーム画面で `Esc` を押すとプロジェクト選択画面へ戻ります。New を選ぶと新規プロジェクト画面へ、
Config を選ぶと設定画面へ遷移します。設定画面・新規プロジェクト画面で `Esc` を押すと起動画面へ
戻ります。

## 1. 起動画面（Welcome）✅ 実装済み

`usagi hop` 実行時に最初に表示される画面。代替スクリーン（alternate screen）上に描画され、
終了時に元の端末状態へ復帰します。

### レイアウト

端末サイズに応じて全要素を水平方向に中央寄せして描画します（幅・高さが取得できない環境では
80×24 をフォールスバック値として使用）。

```text
┌────────────────────────────────────────────────────┐
│                                                      │  ← 上部パディング
│                       (\(\                           │  ┐
│                      (='-')                          │  ├ うさぎ AA（マゼンタ・太字）
│                      o(_(")(")                        │  ┘
│                                                      │
│                       USAGI                          │  ← タイトル（緑・太字）
│                                                      │
│                 > Open           o                   │  ┐
│                                                      │  │
│                   New            e                   │  ├ メニュー（選択行は
│                                                      │  │   赤の ">" + シアン太字ラベル）
│                   Config         c                   │  │
│                                                      │  │
│                   Quit           q                   │  ┘
│                                                      │
│                 Opened "repo" 🐰                     │  ← 通知行（任意・黄色）
│                                                      │
│      v0.1.0 | ↑↓: move / Enter: select / q: quit     │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

### 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| うさぎ AA | 3 行のアスキーアート `(\(\` / `(='-')` / `o(_(")(")`| マゼンタ・太字 |
| タイトル | `USAGI` | 緑・太字 |
| メニュー | Open / New / Config / Quit の 4 項目。各行は「カーソル + ラベル(10桁左詰め) + キー(5桁右詰め)」 | 選択行：`>` 赤太字／ラベル シアン太字／キー 黄色。非選択行：素のテキスト |
| 通知行 | メニュー操作の結果メッセージ（例: New 完了後の `Opened "repo" 🐰`）。未表示時は何も描画しない | 黄色 |
| フッター | バージョンと操作ヘルプ `v<version> \| ↑↓: move / Enter: select / q: quit` | 淡色（dim） |

### メニュー項目

| ラベル | キー | 動作 | 状態 |
|---|---|---|---|
| Open | `o` | プロジェクト選択画面へ遷移 | ✅ 実装済み |
| New | `e` | 新規プロジェクト画面へ遷移 | ✅ 実装済み |
| Config | `c` | 設定画面へ遷移 | ✅ 実装済み |
| Quit | `q` | アプリを終了する | ✅ 実装済み |

### キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `Enter` | 選択中の項目を実行。Open / New / Config は対応する画面へ遷移、Quit なら終了 |
| `o` / `e` / `c` | 対応するメニュー項目を直接実行（ショートカット） |
| `q` / `Esc` / `Ctrl+C` | アプリを終了 |

### 終了時の挙動

- 正常終了（Quit / `q` / `Esc` / `Ctrl+C`）時：代替スクリーンを解除しカーソルを復帰したうえで、
  別れのメッセージ `USAGI run away ( ^-^)ノ` を表示します。
- キー読み取りが中断された場合（シグナル等）：終了として扱います。
- キー読み取りでエラーが発生した場合：別れのメッセージは出さずに端末を復帰し、エラーを返します。

これらの端末復帰は `AlternateScreenGuard`（RAII ガード）により、早期 return や
パニック時でも保証されます。

## 3. 新規プロジェクト画面（New）✅ 実装済み

起動画面で `New`（`e` / Enter）を選ぶと遷移する、プロジェクトを追加するフォーム画面。
**作成方法（Type）を上部のタブで切り替えられます。**

- **Clone** — Git リポジトリをクローンして新しいディレクトリを作成する。VS Code の
  「Clone Git Repository…」や IntelliJ の「Get from Version Control」を参考に、
  **URL を入力するとディレクトリ名が自動で導出される** UX を採用しています。
- **Existing** — 既にローカルに存在するディレクトリをワークスペースとして登録する。
  クローンは行わず、ディレクトリの末尾セグメントからワークスペース名を自動導出します。

起動画面と同じ代替スクリーン上に描画されます。

### レイアウト

フォーム全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。
先頭にモード切り替えタブを置き、タブの状態に応じて以降の入力フィールドが切り替わります。

```text
┌────────────────────────────────────────────────────┐
│                       (\(\                           │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                          │  │
│                      o(_(")(")                        │  ┘
│                    New Project                       │  ← タイトル（緑・太字）
│   Clone a repository or register an existing directory │ ← サブタイトル（淡色）
│                                                      │
│        Type  (←→ to switch)                          │  ← モード選択ラベル（淡色）
│      > [Clone]   Existing                            │  ← タブ（選択中: 角括弧＋シアン太字／フォーカス時: 赤">"）
│                                                      │
│        Repository URL                                │  ← ラベル（淡色）
│      > https://github.com/owner/repo.git▏            │  ← 入力欄（フォーカス: 赤">" + シアン太字 + キャレット）
│                                                      │
│        Location                                      │
│        /Users/you/git                                │  ← クローン先のベース（既定値を事前入力）
│                                                      │
│        Directory                                     │
│        repo                                          │  ← URL から自動導出（シアン）
│                                                      │
│        Branch (optional)                             │
│        repository default                            │  ← 未入力時はプレースホルダー（淡色斜体）
│                                                      │
│        that does not look like a repository URL      │  ← 通知行（任意・赤太字／バリデーション失敗時）
│                                                      │
│  ←→: switch type / ↑↓/Tab: move field / Enter: create / Esc: back │ ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

`Existing` に切り替えると、URL / Location / Branch の代わりに次の 2 フィールドを表示します。

```text
│        Directory                                     │
│        /path/to/an/existing/project                  │  ← 既存ディレクトリのパス
│                                                      │
│        Name                                          │
│        project                                       │  ← パスの末尾セグメントから自動導出（シアン）
```

### 入力フィールド

モードタブはどちらのモードでも先頭に表示され、Tab 巡回の起点になります。

**Clone モード**

| フィールド | 役割 | 備考 |
|---|---|---|
| Repository URL | クローン対象の Git URL | HTTPS / `git@host:owner/repo.git`（scp 形式）/ `ssh://` に対応 |
| Location | クローン先のベースディレクトリ | 設定の `workspace_root`（未設定なら `~/git`）を既定値として事前入力。手動編集も可能。最終的なクローン先は `<Location>/<Directory>` |
| Directory | クローン先ディレクトリ名 | URL の末尾セグメント（`.git`・末尾スラッシュ除去）から自動導出。手動編集も可能 |
| Branch (optional) | チェックアウトするブランチ | 未入力時はリポジトリのデフォルトブランチ |

**Existing モード**

| フィールド | 役割 | 備考 |
|---|---|---|
| Directory | 登録する既存ディレクトリのパス | 既にローカルに存在するディレクトリを指定する |
| Name | ワークスペース名 | パスの末尾セグメントから自動導出。手動編集も可能 |

### 名前の自動導出（エディタ準拠の挙動）

Clone モードの **Directory**、Existing モードの **Name** はどちらも同じ挙動で自動導出されます。

- Clone: URL を入力するたびに Directory を `末尾セグメント` から再導出します
  （例: `https://github.com/owner/repo.git` → `repo`）。
- Existing: Directory（パス）を入力するたびに Name を末尾コンポーネントから再導出します
  （例: `/home/me/my-app` → `my-app`）。
- ユーザーが導出先フィールドを手動で編集すると自動導出を停止し、以降の元フィールド変更で上書きしません。
- 導出先フィールドを空になるまで削除すると自動導出が再び有効化され、次の元フィールド変更で復活します
  （その場で即座に再充填はせず、ユーザーが独自名を入力できる余地を残します）。
- Clone で URL の末尾がスラッシュの場合（例: `owner/` 入力途中）はセグメントが未確定とみなし、何も提案しません。

### キー操作

| キー | 動作 |
|---|---|
| `←` / `→` | フォーカス位置に関わらず作成方法（Clone / Existing）を切り替え |
| 文字キー | フォーカス中のフィールドに追記（モードタブにフォーカス中は無視） |
| `Backspace` | フォーカス中のフィールドの末尾を 1 文字削除 |
| `Tab` / `↓` | 次のフィールドへフォーカス移動（先頭のモードタブを含めて巡回） |
| `Shift+Tab` / `↑` | 前のフィールドへフォーカス移動 |
| `Enter` | フォームを検証して確定。失敗時は赤い通知を表示 |
| `Esc` | 起動画面へ戻る |
| `Ctrl+C` | アプリを終了 |

### バリデーションと確定後の挙動

**Clone モード**

- URL が空、または URL らしき形（パス区切りと末尾セグメント）を持たない場合は確定せず、
  通知行にエラーメッセージを表示します。
- Location が空の場合も確定せず、`choose where to create the project` を通知します。
- 検証に成功すると、その場でリポジトリのクローンとワークスペース登録を実行します（`usecase::project::create`）。
  具体的には次を行います。
  1. クローン先 `<Location>/<Directory>` が既に存在する場合は中断し、既存チェックアウトを壊さない
  2. `git clone`（Branch 指定時は当該ブランチをチェックアウト）でリポジトリを取得
  3. クローンしたディレクトリをワークスペースとして `workspaces.json` に登録（`usecase::workspace::add`）
  4. `<repo>/.usagi/state.json` に初期 worktree 状態を同期（`usecase::workspace_state::sync`）

**Existing モード**

- Directory（パス）が空の場合は確定せず、`choose an existing directory` を通知します。
- パスから名前が導出できず Name も空の場合は `enter a name for the workspace` を通知します。
- 検証に成功すると、その場で既存ディレクトリをワークスペースとして登録します（`usecase::project::register_existing`）。
  1. 指定パスがディレクトリでない（存在しない）場合は中断し、`<path> is not a directory` を通知
  2. ディレクトリを Name でワークスペースとして `workspaces.json` に登録（`usecase::workspace::add`）
  3. **Git リポジトリの場合のみ** `<repo>/.usagi/state.json` に worktree 状態を同期。
     Git 管理下にない素のディレクトリはクローン同様に登録しつつ、同期はスキップします

- どちらのモードも、成功すると起動画面へ戻り `Opened "<name>" 🐰` の通知を表示します。失敗した場合
  （不正な URL・ネットワーク・名前衝突・存在しないパスなど）は `Could not create project: <理由>` を通知し、
  メニューは維持されるためユーザーは修正して再試行できます。

> 描画ロジックは `src/presentation/tui/new/`、URL のパース・ディレクトリ名導出は
> `src/domain/repository.rs`、クローン / 既存登録 + 同期の一連の処理は `src/usecase/project.rs`
> （Git 操作・リポジトリ判定は `src/infrastructure/git.rs`）に実装されています。

## 2. プロジェクト選択画面（Open）✅ 実装済み

起動画面で `Open`（`o` / Enter）を選ぶと遷移する、登録済みワークスペースの一覧から
開く対象を選ぶ画面。一覧は `workspaces.json` 由来で、最終更新の新しい順に並びます。
起動画面と同じ代替スクリーン上に描画されます。

> ワークスペースを選択するとホーム画面（画面 #5）へ遷移し、そのワークスペースの worktree 一覧を
> 表示します。ホーム画面で `Esc` を押すとこのプロジェクト選択画面へ戻ります。

### レイアウト

リスト全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。

```text
┌────────────────────────────────────────────────────┐
│                       (\(\                           │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                          │  │
│                      o(_(")(")                        │  ┘
│                    Open Project                      │  ← タイトル（緑・太字）
│              Open a registered workspace             │  ← サブタイトル（淡色）
│                                                      │
│      > my-project    /path/to/my-project             │  ┐ プロジェクト一覧
│        other-repo    /path/to/other-repo             │  ┘  選択行：赤 ">" + 名前シアン太字
│                                                      │     パスは淡色（長い場合は先頭を … で省略）
│                                                      │  ← 通知行（任意・黄色／読み込み失敗時など）
│                                                      │
│        ↑↓: move / Enter: open / Esc: back            │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

ワークスペースが 1 つも登録されていない場合は、一覧の代わりに
`No workspaces yet. Choose "New" to add one.`（淡色）を中央に表示します。

### 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| タイトル | `Open Project` | 緑・太字 |
| サブタイトル | `Open a registered workspace` | 淡色（dim） |
| プロジェクト一覧 | 登録済みワークスペースを「カーソル + 名前(幅揃え) + パス」で 1 行ずつ表示。パスはブロック幅に収まるよう先頭を省略 | 選択行：`>` 赤太字／名前 シアン太字。パスは常に淡色 |
| 空表示 | 登録が無い場合のメッセージ | 淡色（dim） |
| 通知行 | 読み込み失敗時などのメッセージ。未表示時も 1 行ぶんの枠を確保しレイアウトを安定させる | 黄色 |
| フッター | 操作ヘルプ `↑↓: move / Enter: open / Esc: back` | 淡色（dim） |

### キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `Enter` | 選択中のワークスペースを開き、ホーム画面（画面 #5）へ遷移する |
| `q` / `Esc` | 起動画面へ戻る |
| `Ctrl+C` | アプリを終了 |

### 読み込みと確定後の挙動

- ワークスペースの読み込みに失敗した場合は、空一覧の状態でエラー内容を通知行に表示します。
- `Enter` でワークスペースを選択するとホーム画面へ遷移します。ホーム画面から戻る（`Esc`）と
  この一覧へ復帰し、アプリ終了（`Ctrl+C`）はそのまま上位へ伝播します。

> 描画ロジックは `src/presentation/tui/open/`（一覧状態 `state.rs`・描画 `ui.rs`・
> イベントループ `event.rs` に分離）に実装されています。

## 5. ホーム画面（Home）✅ 実装済み（3 ペイン + コマンドモード）

プロジェクト選択画面でワークスペースを選ぶと遷移する、ワークスペース操作のメイン画面。
worktree 一覧（左ペイン）・コマンド履歴/出力（右ペイン）・コマンド入力欄（下部）の 3 ペイン
構成で、**サイドバーモード**（worktree 一覧の操作）と**コマンドモード**（TUI 内コマンドの入力）を
切り替えて操作します。worktree の情報はワークスペースの `<workspace>/.usagi/state.json`
（画面外で `usagi` が同期した内容）から読み込みます。起動画面と同じ代替スクリーン上に描画されます。

### レイアウト

端末全体を使い、上部にタイトルバー、中央を縦の罫線 `│` で左右 2 ペインに分割、下部に
コマンド入力欄とフッターを配置します。左ペインの幅は端末幅の約 1/3（16〜40 桁にクランプ）です。

```text
┌──────────────────────────────────────────────────────────┐
│                  usagi · 3 worktrees                     │  ← タイトルバー（緑・太字、中央寄せ）
│                                                          │
│  > ● main       pushed  │ Type ":" to enter a command…   │  ┐ 左：worktree 一覧
│    feature/x    local   │ ❯ man                          │  │ 右：コマンド履歴/出力
│    fix/y        merged  │ Available commands:            │  │   （新しい行ほど下、溢れたら古い行から省略）
│                         │   session  Create or manage…   │  ┘
│                         │ Opening "main" is coming soon 🐰│
│ ❯ man▏                                                    │  ← コマンド入力欄（コマンドモード時）
│ Tab: complete / ↑↓: history / Enter: run / Esc: cancel    │  ← フッター（モード別・淡色）
└──────────────────────────────────────────────────────────┘
```

worktree の記録が無い（`state.json` 未生成など）場合は、左ペインに
`No worktrees recorded yet. Run usagi to sync.` を表示します。

### 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| タイトルバー | `<ワークスペース名> · N worktree(s)` | 緑・太字（中央寄せ） |
| worktree 一覧（左ペイン） | 「カーソル + primary マーカー + ブランチ名(幅揃え) + status」で 1 行ずつ表示。ブランチ名は左ペイン幅に収まるよう末尾を省略 | 選択行：`>` 赤太字／primary：`●` マゼンタ／ブランチ名 シアン（選択行は太字）／status は色分け |
| status | ブランチの状態 `local` / `pushed` / `merged` | `local`：黄色／`pushed`：緑／`merged`：淡色 |
| コマンド履歴/出力（右ペイン） | 入力したコマンドのエコー（`❯` 付き）・出力・エラー・通知を時系列で表示。ペインに収まらない古い行は上から省略 | コマンド：シアン太字／出力：素／エラー：赤／通知：黄色 |
| コマンド入力欄 | コマンドモードでは `❯ <入力>▏`（プロンプト赤太字・入力シアン・末尾にキャレット）。サイドバーモードでは入り方のヒント | コマンドモード：プロンプト赤太字・入力シアン／サイドバー：淡色 |
| フッター | モード別の操作ヘルプ | 淡色（dim） |

### モードとキー操作

#### サイドバーモード（既定）

| キー | 動作 |
|---|---|
| `↑` / `k` | worktree 選択を 1 つ上へ移動（先頭で押すと末尾へラップ） |
| `↓` / `j` | worktree 選択を 1 つ下へ移動（末尾で押すと先頭へラップ） |
| `Enter` | 選択中の worktree を開く（現状は右ペインに「coming soon」通知を表示） |
| `:` / `i` | コマンドモードへ切り替え |
| `q` / `Esc` | プロジェクト選択画面へ戻る |
| `Ctrl+C` | アプリを終了 |

#### コマンドモード

| キー | 動作 |
|---|---|
| 文字キー | 入力欄に追記 |
| `Backspace` | 入力欄の末尾を 1 文字削除 |
| `Tab` | コマンド名を補完（一意なら確定、曖昧なら共通接頭辞まで補完し候補を右ペインに列挙） |
| `↑` / `↓` | コマンド履歴を遡る／戻る（末尾を越えると空行に戻る） |
| `Enter` | 入力を実行（右ペインにエコー＋出力を追記、履歴に記録） |
| `Esc` | 入力を破棄してサイドバーモードへ戻る |
| `Ctrl+C` | アプリを終了 |

### TUI 内コマンド

| コマンド | 動作 | 状態 |
|---|---|---|
| `man` / `help` | コマンド一覧、または `man <command>` で個別の説明を表示 | ✅ 実装済み |
| `history` | これまでに入力したコマンドの履歴を番号付きで表示 | ✅ 実装済み |
| `clear` | 右ペインの出力ログを消去 | ✅ 実装済み |
| `quit` / `exit` | アプリを終了 | ✅ 実装済み |
| `session` / `space` / `ai` / `terminal` / `doctor` | 認識はするが本体は未実装。「coming soon」を表示 | 🚧 プレースホルダー |
| 上記以外 | `unknown command: "…" (try "man")` を赤エラーで表示 | ✅ 実装済み |

### 読み込みと今後の予定

- worktree 状態の読み込みに失敗した場合は、空一覧の状態でエラー内容を右ペインに表示します。
- `session` / `space` / `ai` / `terminal` / `doctor` の本体実装と、worktree を選択したあとの
  セッション画面への遷移は今後の作業で実装します（前者は Git 操作・疑似ターミナル・AI 連携などの
  インフラ層に依存するため別途）。

> 描画ロジックは `src/presentation/tui/home/`（画面状態 `state.rs`・描画 `ui.rs`・
> イベントループ `event.rs`・コマンド解析/補完 `command.rs` に分離）に実装されています。

## 4. 設定画面（Config）✅ 実装済み

起動画面で `Config`（`c` / Enter）を選ぶと遷移する、グローバル設定の閲覧・編集を行う画面。
各設定項目の値は常に `< 値 >` の形（左右セレクタ）で表示し、選択中の項目を `←→` で切り替えます。
編集はメモリ上に保持され、変更した項目はラベル左の黄色 `●`（および値の黄色）で「未保存の変更あり」と
明示されます。`settings.json` への書き込みは、フィールドの下にある **Save ボタン**（変更があるときだけ
有効化）を押したときに行われます。起動画面と同じ代替スクリーン上に描画されます。

### レイアウト

設定全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。

```text
┌────────────────────────────────────────────────────┐
│                       (\(\                           │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                          │  │
│                      o(_(")(")                        │  ┘
│                       Config                         │  ← タイトル（緑・太字）
│                Adjust your preferences               │  ← サブタイトル（淡色）
│                                                      │
│          Theme              < System >               │  ┐ 設定一覧（全項目 `< 値 >`）
│          Default Workspace  < (none) >               │  │  値は常に山かっこで囲み列を揃える
│      >   Notifications      < On >                    │  │  選択行：左端の赤 ">" + 値が明色
│        ● Agent CLI          < Gemini >               │  ┘  変更済み：ラベル左に黄色 ● + 値も黄色
│                                                      │
│                           [ Save ]                   │  ← Save ボタン（変更時のみ緑で有効）
│                                                      │
│        Saved 🐰                                      │  ← 通知行（任意・黄色）
│                                                      │
│    ↑↓: move · ←→: change · Enter: save · Esc: back   │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

### 設定項目

| 項目 | 値 | 循環する選択肢 |
|---|---|---|
| Theme | UI のカラーテーマ | `Light` → `Dark` → `System`（OS 追従）の順に循環 |
| Default Workspace | 既定で開くワークスペース | `(none)` ＋ 登録済みワークスペース名を順に循環。未登録時は変更不可（ヒントを表示） |
| Notifications | デスクトップ通知（`hop` 時など）の ON/OFF | `On` ⇄ `Off` をトグル |
| Agent CLI | usagi が起動する AI エージェント CLI | `Claude` → `Gemini` の順に循環 |

> 全項目を共通の chooser widget で描画し、現在値のみを常に `< 値 >` の形で表示します。

> `settings.json` には `workspace_root`（新規プロジェクトのクローン先ベースディレクトリ）も
> 保存されますが、本画面での編集には未対応です（今後追加予定）。

### widget による値の表示

設定値は共通の chooser widget（`src/presentation/tui/widgets.rs` の `chooser`）で統一して描画します。

- **chooser**: 現在値のみを表示する左右セレクタです。値を常に `< Dark >` のように山かっこで囲み、
  `←→` で前後の選択肢に切り替えられることを示します。山かっこは全行で同じ列に揃います。フォーカスの
  状態は色で表し、選択行は明色（シアン・太字）、それ以外は淡色（dim）で描画します。
- **変更の明示**: 保存済みの値と異なる項目は、ラベルの左に黄色の `●` を表示し、あわせて値も黄色・太字に
  して未保存の編集であることを示します（`●` はカーソルの隣の専用列に置くため、選択と変更を同時に表せます）。

### 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| タイトル | `Config` | 緑・太字 |
| サブタイトル | `Adjust your preferences` | 淡色（dim） |
| 設定一覧 | 各設定を「カーソル列 + 変更マーク列 + ラベル(幅揃え) + chooser」で 1 行ずつ表示 | 選択行：左端 `>` 赤太字／ラベル シアン太字。値は chooser（常に `< 値 >`）。変更済みはラベル左に `●`（黄色）＋値も黄色 |
| Save ボタン | 設定一覧の下に 1 行空けて配置する `[ Save ]`。値の列に揃える | 変更ありで緑・太字（有効）、変更なしで淡色（無効）。選択行は `>` 赤太字 |
| 通知行 | 保存結果などのメッセージ（例: `Saved 🐰`）。未表示時も 1 行ぶんの枠を確保しレイアウトを安定させる | 黄色 |
| フッター | 操作ヘルプ `↑↓: move · ←→: change · Enter: save · Esc: back` | 淡色（dim） |

### キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（設定一覧と Save ボタンの間をラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾の Save ボタンの次は先頭へラップ）。通知をクリア |
| `→` / `l` | 選択中の項目の値を次の選択肢へ切り替え（メモリ上のみ。保存はしない）。Save ボタン上では無効 |
| `←` / `h` | 選択中の項目の値を前の選択肢へ切り替え（メモリ上のみ。保存はしない）。Save ボタン上では無効 |
| `Enter` | Save ボタン上では編集内容を `settings.json` へ保存。設定項目上では `→` と同じく値を次へ切り替え |
| `q` / `Esc` | 起動画面へ戻る（未保存の編集は破棄） |
| `Ctrl+C` | アプリを終了 |

### 読み込みと保存の挙動

- 設定の読み込みに失敗した場合は、既定値の状態でエラー内容を通知行に表示します。
- 値の切り替え（`←→` / 設定項目上の `Enter`）はメモリ上の編集のみで、ディスクには書き込みません。
  保存済みの値から変わった項目は黄色＋ `●` で示し、変更がある間は Save ボタンが緑（有効）になります。
- Save ボタンで `Enter` を押すと `settings.json` へ保存し、`Saved 🐰` を通知して変更マークを消します。
  変更が無い状態で押した場合は `No changes to save 🐰` を表示し、書き込みは行いません。保存に失敗した
  場合は `Failed to save: <理由>` を表示し、画面はそのまま継続します（再操作・離脱が可能）。
- 既定ワークスペースの選択肢が無い（ワークスペース未登録）状態で値を切り替えようとすると、
  `No workspaces to choose from 🐰` を表示します。

> 描画ロジックは `src/presentation/tui/config/`（設定状態 `state.rs`・描画 `ui.rs`・
> イベントループ `event.rs` に分離）、設定の永続化は `src/usecase/settings.rs` と
> `src/infrastructure/storage.rs` に実装されています。

## 設計メモ

- 配色には `console` クレートのスタイル（magenta / green / cyan / yellow / red / dim）を使用。
- 中央寄せは「端末幅 − コンテンツ幅」を 2 で割った左パディングで実現（`centered_padding`）。
  端末が狭い場合は `saturating_sub` によりパディング 0 にフォールバックします。
- 起動画面の描画ロジックは `src/presentation/tui/welcome/`、プロジェクト選択画面は
  `src/presentation/tui/open/`、新規プロジェクト画面は `src/presentation/tui/new/`、
  設定画面は `src/presentation/tui/config/`、ホーム画面は `src/presentation/tui/home/`
  （状態 `state.rs`・描画 `ui.rs`・イベントループ `event.rs` に分離。ホーム画面はさらに
  コマンド解析/補完を `command.rs` に分離）、端末制御は
  `src/presentation/tui/screen.rs` に実装されています。
- 中央寄せ・うさぎ AA・タイトル/サブタイトル/フッターなどの共通描画は
  `src/presentation/tui/widgets.rs` に集約しています。
