# usagi — 画面設計書

`usagi hop` で起動する TUI の画面設計をまとめます。本書は実装状況を反映しており、
**実装済み** と **予定（usagi.ai から移植）** を明示して区別します。全体像は
[overview.md](./overview.md) を参照してください。

## 画面一覧

| # | 画面 | 役割 | 状態 |
|---|---|---|---|
| 1 | 起動画面（Home） | アプリ起動時のトップメニュー。Open / New / Config / Quit を選択する | ✅ 実装済み |
| 2 | プロジェクト選択画面（Open） | 初期化済みプロジェクトの一覧から開く対象を選ぶ | 🔜 予定 |
| 3 | 新規プロジェクト画面（New） | リポジトリ URL を入力して `init` を実行する | ✅ 実装済み |
| 4 | 設定画面（Config） | グローバル設定を編集する | 🔜 予定 |
| 5 | ワークスペース画面 | worktree 一覧・コマンド履歴・コマンド入力欄を持つメイン操作画面 | 🔜 予定 |

> 現時点で実装されている画面は **起動画面（Home）** と **新規プロジェクト画面（New）** の 2 つです。
> Open / Config を選ぶと「coming soon」通知を表示するプレースホルダーになっています。

## 1. 起動画面（Home）✅ 実装済み

`usagi hop` 実行時に最初に表示される画面。代替スクリーン（alternate screen）上に描画され、
終了時に元の端末状態へ復帰します。

### レイアウト

端末サイズに応じて全要素を水平方向に中央寄せして描画します（幅・高さが取得できない環境では
80×24 をフォールスバック値として使用）。

```text
┌────────────────────────────────────────────────────┐
│                                                      │  ← 上部パディング
│                       (\(\                           │  ┐
│                      (='-')                          │  ├ うさぎ AA（マゼンタ・太字）
│                      o(_(")(")                        │  ┘
│                                                      │
│                       USAGI                          │  ← タイトル（緑・太字）
│                                                      │
│                 > Open           o                   │  ┐
│                                                      │  │
│                   New            e                   │  ├ メニュー（選択行は
│                                                      │  │   赤の ">" + シアン太字ラベル）
│                   Config         c                   │  │
│                                                      │  │
│                   Quit           q                   │  ┘
│                                                      │
│              New is coming soon 🐰                   │  ← 通知行（任意・黄色）
│                                                      │
│      v0.1.0 | ↑↓: move / Enter: select / q: quit     │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

### 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| うさぎ AA | 3 行のアスキーアート `(\(\` / `(='-')` / `o(_(")(")`| マゼンタ・太字 |
| タイトル | `USAGI` | 緑・太字 |
| メニュー | Open / New / Config / Quit の 4 項目。各行は「カーソル + ラベル(10桁左詰め) + キー(5桁右詰め)」 | 選択行：`>` 赤太字／ラベル シアン太字／キー 黄色。非選択行：素のテキスト |
| 通知行 | メニュー選択結果のメッセージ（例: `Open is coming soon 🐰`）。未表示時は何も描画しない | 黄色 |
| フッター | バージョンと操作ヘルプ `v<version> \| ↑↓: move / Enter: select / q: quit` | 淡色（dim） |

### メニュー項目

| ラベル | キー | 動作 | 状態 |
|---|---|---|---|
| Open | `o` | プロジェクト選択画面へ（予定）。現状は通知表示 | プレースホルダー |
| New | `e` | 新規プロジェクト画面へ（予定）。現状は通知表示 | プレースホルダー |
| Config | `c` | 設定画面へ（予定）。現状は通知表示 | プレースホルダー |
| Quit | `q` | アプリを終了する | ✅ 実装済み |

### キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `Enter` | 選択中の項目を実行。Quit なら終了、それ以外は通知を表示 |
| `o` / `e` / `c` | 対応するメニュー項目を直接実行（ショートカット） |
| `q` / `Esc` / `Ctrl+C` | アプリを終了 |

### 終了時の挙動

- 正常終了（Quit / `q` / `Esc` / `Ctrl+C`）時：代替スクリーンを解除しカーソルを復帰したうえで、
  別れのメッセージ `USAGI run away ( ^-^)ノ` を表示します。
- キー読み取りが中断された場合（シグナル等）：終了として扱います。
- キー読み取りでエラーが発生した場合：別れのメッセージは出さずに端末を復帰し、エラーを返します。

これらの端末復帰は `AlternateScreenGuard`（RAII ガード）により、早期 return や
パニック時でも保証されます。

## 3. 新規プロジェクト画面（New）✅ 実装済み

起動画面で `New`（`e` / Enter）を選ぶと遷移する、リポジトリ URL を入力するフォーム画面。
VS Code の「Clone Git Repository…」や IntelliJ の「Get from Version Control」といった
エディタのクローン画面を参考に、**URL を入力するとディレクトリ名が自動で導出される** UX を採用しています。
起動画面と同じ代替スクリーン上に描画されます。

### レイアウト

フォーム全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。

```text
┌────────────────────────────────────────────────────┐
│                       (\(\                           │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                          │  │
│                      o(_(")(")                        │  ┘
│                    New Project                       │  ← タイトル（緑・太字）
│        Clone a Git repository into a new workspace   │  ← サブタイトル（淡色）
│                                                      │
│        Repository URL                                │  ← ラベル（淡色）
│      > https://github.com/owner/repo.git▏            │  ← 入力欄（フォーカス: 赤">" + シアン太字 + キャレット）
│                                                      │
│        Directory                                     │
│        repo                                          │  ← URL から自動導出（シアン）
│                                                      │
│        Branch (optional)                             │
│        repository default                            │  ← 未入力時はプレースホルダー（淡色斜体）
│                                                      │
│        enter a repository URL                        │  ← 通知行（任意・赤太字／バリデーション失敗時）
│                                                      │
│      ↑↓/Tab: move field / Enter: create / Esc: back  │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

### 入力フィールド

| フィールド | 役割 | 備考 |
|---|---|---|
| Repository URL | クローン対象の Git URL | HTTPS / `git@host:owner/repo.git`（scp 形式）/ `ssh://` に対応 |
| Directory | クローン先ディレクトリ名 | URL の末尾セグメント（`.git`・末尾スラッシュ除去）から自動導出。手動編集も可能 |
| Branch (optional) | チェックアウトするブランチ | 未入力時はリポジトリのデフォルトブランチ |

### ディレクトリ名の自動導出（エディタ準拠の挙動）

- URL を入力するたびに Directory を `末尾セグメント` から再導出します
  （例: `https://github.com/owner/repo.git` → `repo`）。
- ユーザーが Directory を手動で編集すると自動導出を停止し、以降の URL 変更で上書きしません。
- Directory を空になるまで削除すると自動導出が再び有効化され、次の URL 変更で復活します
  （その場で即座に再充填はせず、ユーザーが独自名を入力できる余地を残します）。
- URL の末尾がスラッシュの場合（例: `owner/` 入力途中）はセグメントが未確定とみなし、何も提案しません。

### キー操作

| キー | 動作 |
|---|---|
| 文字キー | フォーカス中のフィールドに追記 |
| `Backspace` | フォーカス中のフィールドの末尾を 1 文字削除 |
| `Tab` / `↓` | 次のフィールドへフォーカス移動 |
| `Shift+Tab` / `↑` | 前のフィールドへフォーカス移動 |
| `Enter` | フォームを検証して確定。失敗時は赤い通知を表示 |
| `Esc` | 起動画面へ戻る |
| `Ctrl+C` | アプリを終了 |

### バリデーションと確定後の挙動

- URL が空、または URL らしき形（パス区切りと末尾セグメント）を持たない場合は確定せず、
  通知行にエラーメッセージを表示します。
- 検証に成功すると起動画面へ戻り、`Ready to init "<directory>" from <url> 🐰` の通知を表示します。
  実際のクローン + 初期化（`usagi init` 相当）の実行は今後の作業（画面 #5 と連携）で実装予定です。

> 描画ロジックは `src/presentation/tui/new/`、URL のパース・ディレクトリ名導出は
> `src/domain/repository.rs` に実装されています。

## 2・4・5. その他の画面 🔜 予定

以下は usagi.ai から移植予定の画面です。詳細仕様は移植時に本書へ追記します。

### 2. プロジェクト選択画面（Open）
初期化済みプロジェクトの一覧（グローバルレジストリ `repositories.json` 由来）を表示し、
開く対象を選択する画面。選択後はワークスペース画面へ遷移します。

### 4. 設定画面（Config）
グローバル設定の閲覧・編集を行う画面。

### 5. ワークスペース画面
`usagi hop` のメイン操作画面。次の 3 ペインで構成される予定です。

```text
┌──────────────┬───────────────────────────────┐
│ worktree 一覧 │ コマンド履歴 / 出力            │
│ (左ペイン)    │ (右ペイン)                     │
│              │                                │
├──────────────┴───────────────────────────────┤
│ > コマンド入力欄                              │
└───────────────────────────────────────────────┘
```

- **サイドメニューモード**：左ペインの worktree を選択・操作するモード
- **コマンドモード**：下部の入力欄から TUI 内コマンド（`session` / `space` / `ai` /
  `terminal` / `history` / `doctor` / `man`）を実行するモード。Tab 補完・履歴遡りに対応

## 設計メモ

- 配色には `console` クレートのスタイル（magenta / green / cyan / yellow / red / dim）を使用。
- 中央寄せは「端末幅 − コンテンツ幅」を 2 で割った左パディングで実現（`centered_padding`）。
  端末が狭い場合は `saturating_sub` によりパディング 0 にフォールバックします。
- 起動画面の描画ロジックは `src/presentation/tui/home/`、新規プロジェクト画面は
  `src/presentation/tui/new/`（フォーム状態 `state.rs` と描画 `ui.rs` に分離）、端末制御は
  `src/presentation/tui/screen.rs` に実装されています。
