# 3.2 TUI 内コマンド

> [コマンドリファレンス目次](README.md) ｜ ← 前へ [CLI コマンド](01-cli.md)

`usagi hop` のホーム画面で実行する TUI 内コマンドの一覧です。コマンドの入力面は**物理的に 2 つ**あり、
**統括（Overview）の下部コマンドライン**でワークスペース全体を、**在席（Focus）の右ペイン**で選択中セッションを
操作します。入力欄の直上には入力中の内容に応じた候補・ヒント（コマンド一覧の絞り込み、または引数入力中の `usage` / `examples`）が
表示されます。`Tab` で補完、`↑↓` で履歴を遡れます。状態記号の凡例は [README.md](README.md#凡例) を、
画面側の挙動・モード遷移・入力候補の詳細は [design/05-home.md](../design/05-home.md#入力候補ヒントコマンドライン) を参照してください。

コマンドは入力面に応じた**スコープ**で絞られます。以前の「入れ子」モデルは廃止され、2 つの入力面は**物理的に分かれて**います。
統括の下部コマンドライン（Workspace スコープ）には**ワークスペース全体**のコマンド（`session` / `config` / `doctor`）が、
在席の右ペイン（Session スコープ）には**個別セッション**のコマンド（`terminal` / `agent` / `ai`）が出ます。各スコープに
他方のコマンドは出ません。`man` / `history` / `clear` / `quit` は両方で共通です。フッターに `[workspace]` / `[session: <名前>]` が
表示されます。`session switch`（名前なし）は**切替（Switch）**モードへ入り、`terminal` / `agent` は**在席の右ペイン**から
実行します。詳細は [design/05-home.md](../design/05-home.md#コマンドスコープ) を参照。

| コマンド | 説明 | issue | 状態 |
|---|---|---|---|

| `history` | 入力したコマンドの履歴を番号付きで表示 | [007](../../issues/007-history.md) | ✅ |
| `clear` | 右ペインの出力ログを消去 | — | ✅ |
| `quit` / `exit` | アプリを終了 | — | ✅ |
| `session` | **統括の下部ライン**で実行。`session create <name>` でセッション（`.usagi/sessions/<name>/` 配下に再帰的に worktree を構築）を作成（名前省略時は**切替（Switch）**の左ペイン内インライン入力で作成）。`session list` で一覧、`session switch <name>` でアクティブセッション切り替え＆**在席**へ（引数なしで**切替**を開く、`session switch root` でルート行へ）、`session remove <name> [--force]` で削除（未コミット変更があれば警告し `--force` で破棄。名前省略時はセッション一覧モーダルを開き、Space で選択/解除して Enter で一括削除）。サブコマンドは短縮形を受け付ける（`create`=`c`/`new`、`list`=`ls`、`remove`=`rm`） | [003](../../issues/003-session.md) / [004](../../issues/004-space.md) | ✅ 実装済み |

| `config` | Config（設定）画面を**ワークスペーススコープ**で開き、現在のワークスペースのローカル設定（`<workspace>/.usagi/settings.json`）のみを編集する。グローバル設定は CLI（`usagi config`）／起動画面の Config で編集する。`Esc` / `q` でワークスペース画面へ復帰、`Ctrl+C` でアプリ終了 | [029](../../issues/029-home-config.md) | ✅ 実装済み |
| `doctor` | 依存関係チェック（TUI 版） | [019](../../issues/019-doctor-fix.md) | 🚧 |

> 🚧 のうち `ai` / `doctor` はホーム画面で名前としては認識され、本体が未実装のため「coming soon」を
> 表示します（プレースホルダーとして登録済み）。`diff` はまだコマンドとして登録されておらず、入力すると
> `unknown command` になります。`session` / `ai` などが司る worktree オーケストレーションの全体像は
> [4. オーケストレーション](../04-orchestration.md) を参照してください。
>
> `terminal` は左ペインの worktree 一覧で選択中の worktree を作業ディレクトリに、**右ペインへ埋め込んだライブシェル**を開きます（疑似ターミナル: portable-pty + vt100）。左ペインの一覧は表示したままなので、シェルを操作しながら worktree を見渡せます。`session create` で作ったセッションの worktree を選んで `terminal` を実行すれば、そこで `claude` などの AI エージェントを起動して開発できます。
>
> 一覧の先頭には、どのセッションにも属さない **ルート行（`root`）** が常設されています（[031](../../issues/031-root-mode.md)）。ワークスペースを開いた直後はこのルート行が選択・アクティブ状態で、ここで `terminal` / `agent` を実行するとセッションの worktree ではなく**ワークスペースルート**でシェル / Agent が起動します。セッションに入らずにルートで作業したいときに使います。
>
> `agent` はその「`terminal` を開いて Agent CLI を起動する」一連の操作をまとめたショートカットです。`terminal` と同じ埋め込みシェルを起動し、設定の Agent CLI（既定は `claude`。Config 画面やローカル設定で `gemini` などに変更可能）を**シェルの引数として渡して**起動します（stdin にタイプするとシェルが行をエコーして長いコマンド行がペインに映り込むため、引数渡しでエコーを防いでいます。Agent 終了後は対話シェルに戻ります）。
>
> このとき usagi 自身の issue MCP サーバ（[`usagi mcp`](03-mcp.md)）を Agent CLI に組み込んで起動するため、エージェントは起動直後から `issue_create` / `issue_list` / `issue_update` などの tool でタスクを操作できます。Claude はインラインの `--mcp-config '{"mcpServers":{"usagi":{"command":"usagi","args":["mcp"]}}}'` で注入します。Gemini はインライン注入用のフラグを持たない（MCP サーバは `settings.json` 経由で設定）ため、現状は素のまま起動します。
>
> あわせて Claude には `--append-system-prompt` で「usagi が管理するセッション専用 worktree 内で起動している」旨をシステムプロンプトとして渡します。`agent` で開くシェルは既にセッションの worktree（ルート行選択時はワークスペースルート）をカレントディレクトリにしているため、エージェントが手順書どおりに新しい worktree を作ろうとする冗長な動きを防ぎ、その場で作業に着手させます。Gemini は同等のフラグを持たないため現状は注入しません。
>
> #### `Ctrl-O` で切替へズームアウト
>
> 没入（埋め込みターミナル）中は **`Ctrl-O` だけが予約キー**で、`Esc` を含む他のキーはすべてシェル（や `claude`）へ転送されます（フォーカス中の衝突を避けるため切り替え専用の予約キーが必要）。`Ctrl-O` を押すと**切替（Switch）モード**へ一段ズームアウトし、左ペインそのものにキーボードが移ります。そこで `↑`/`↓`（`k`/`j`）でカーソルを動かし、`Enter`/`l` で別セッションを確定します——確定先が**ライブなら再アタッチ**（没入）、**アイドルなら在席**に着きます。切替で `c` を押すと**左ペイン内のインライン名入力**で新規セッションを作成でき、もう一度 `Ctrl-O` を押すと統括まで戻ります。**切り替え先はそのセッション自身の状態をそのまま表示します**——既にシェルや `claude` が動いていればその画面に、まだ起動していなければ素のシェルに着きます（ペインを開いたコマンドが `agent` でも、切り替え先で `claude` を勝手に起動し直すことはありません）。なお TUI 起動中はマウスレポート（DECSET 1000 + SGR 1006）を有効化してホイールイベントを横取りするため、マウスホイールで端末自身のビューポートが動いて（= usagi を開く前の端末内容が見えて）しまうことはありません。代替スクロールモード（DECSET 1007）はあえて使いません——ホイールを矢印キーに化けさせると本物の矢印と区別できず、ホイールをマウスとして報告しない端末（Apple Terminal.app など）で一覧が勝手に動いてしまうためで、マウスレポート 1 本に統一しています。**管理画面（ホームなど）の TUI 自体はスクロールしません**（ホイール・`PageUp`/`PageDown` は読み捨て）。スクロールできるのはこの埋め込みターミナルだけで、`Shift`+`PageUp`/`PageDown`・ホイール（報告される端末のみ）をペイン内のスクロール（スクロールバック、既定 1 万行）に充てます。スクロールの全体方針は [画面設計 5. ホーム画面](../design/05-home.md#ペインのスクロールとマウスホイール) を参照。
>
> | 入力 | 動作 |
> |---|---|
> | `Shift`+`PageUp` / `PageDown`・ホイール | 埋め込みターミナルの履歴（スクロールバック）をさかのぼる／戻す（キー入力で最新画面へ戻る） |
> | `Ctrl-O` | 切替へズームアウト（もう一度押すと統括まで。**シェル／Agent は生かしたまま**） |
> | （`Ctrl-O` 以外、`Esc` 含む） | そのままシェルへ転送 |
> | （切替中）`↑`/`↓`・`k`/`j` + `Enter` / `l` | カーソル行のセッションへ切り替え（ライブなら再アタッチ、アイドルなら在席） |
> | （切替中）`c` | 新規セッションを作成（左ペイン内のインライン名入力） |
> | （切替中）`Esc` / `h` | 開いた元のモードへ戻る |
>
> 各 worktree のシェルは画面を開いている間ずっと生き続け（[ターミナルプール](../04-orchestration.md#埋め込みターミナルの永続化)）、裏でも出力を受け取り続けます。そのため別セッションへ切り替えても元の `claude` はそのまま動いており、戻れば続きから操作できます。切り替えは常にその worktree のシェルにアタッチする（または無ければ素のシェルを spawn する）ので、`agent` で開いたセッションは `claude` の画面に、何もしていないセッションはアイドルなシェルに——というように**切り替え先ごとの状態**が見えます。ルート行（`root`）を選べばワークスペースルートのターミナルへ切り替わります。シェルを完全に終了したいときは、そのシェルにフォーカスして `exit` してください（画面を離れるとプールごと破棄され全シェルが終了します）。
>
> #### 使用中 Agent の表示・入力待ちの検知と通知
>
> 左ペインの各セッションは 2 行で表示され、Agent の稼働状態を**2 行目にアイコン＋テキスト**で示します（1 行目はカーソル・種別アイコン・名前と右端の git status）。シェル / Agent が**稼働中**（アタッチ中・裏稼働を問わずシェルが生存）の worktree は `▶ running`（緑）を表示します（プールが `MonitorHandle::live()` で公開する生存セッション集合に基づく。ベルに依存せず表示）。
> 加えて、裏で動いている `claude` などの Agent が処理を終えて入力待ちになる（ターミナルベル `^G` を鳴らす）と、アタッチ中でないセッションについては、左ペインの該当 worktree が `◆ waiting`（黄色、`▶` より優先）になり、デスクトップ通知（`🐰 <ブランチ名> が入力待ちです`、設定 `notifications_enabled` が ON のとき）が出ます。アタッチ中のセッション自身は入力待ちの対象外です（見えているため）。プールの監視スレッドがベル回数を約 200ms 間隔で監視し、判定は純粋な `SessionMonitor` が行います。複数セッションを並行で走らせ、入力が必要になったものだけに対応する、といった使い方ができます。入力待ちの検知はベルに依存するため、`claude` 側でターミナルベル通知を有効にしてください（鳴らさない CLI では入力待ちは出ませんが、稼働中 `▶` は表示されます）。詳細は [issue 028](../../issues/028-agent-wait-notify.md)。
