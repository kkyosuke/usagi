# 画面設計

`usagi hop` で起動する TUI の画面設計をまとめたディレクトリです。本書は実装状況を反映しており、
**実装済み** と **予定（usagi.ai から移植）** を明示して区別します。全体像は
[../01-overview.md](../01-overview.md) を参照してください。

## 目次

| # | 画面 | ドキュメント | 役割 | 状態 |
|---|---|---|---|---|
| 1 | 起動画面（Welcome） | [01-welcome.md](01-welcome.md) | アプリ起動時のトップメニュー。Open / New / Config / Quit を選択する | ✅ 実装済み |
| 2 | プロジェクト選択画面（Open） | [02-open.md](02-open.md) | 登録済みワークスペースの一覧から開く対象を選ぶ | ✅ 実装済み |
| 3 | 新規プロジェクト画面（New） | [03-new.md](03-new.md) | リポジトリのクローン、または既存ディレクトリの登録でワークスペースを追加する | ✅ 実装済み |
| 4 | 設定画面（Config） | [04-config.md](04-config.md) | グローバル設定を編集する | ✅ 実装済み |
| 5 | ホーム画面（Home） | [05-home.md](05-home.md) | 選択したワークスペースの worktree 一覧とコマンド入力を備えたメイン操作画面 | ✅ 実装済み（3 ペイン + コマンドモード） |

> 現時点で実装されている画面は上記 5 つです。Open でワークスペースを選択するとホーム画面へ遷移し、
> 左ペインにそのワークスペースの worktree 一覧、右ペインにコマンド履歴・出力、下部にコマンド入力欄を
> 表示します。`:`（または `i`）でコマンドモードに入り、TUI 内コマンドを実行できます。`session` は
> 実装済みで、`session <name>` でセッションを作成し、name を省くと名前入力モーダルが開きます。worktree を
> 選択する操作（セッション画面への遷移）と一部のコマンド（`space` / `ai` / `terminal` /
> `doctor`）はプレースホルダーで、実行すると「coming soon」を表示します。

## 画面遷移図

```text
                 [起動画面 Welcome]
                       │
      ┌────────┬───────┼────────┬────────┐
   Open(o)   New(e)  Config(c)  Quit(q) / Esc / Ctrl+C
      │        │        │           │
      ▼        ▼        ▼           ▼
 [プロジェクト [新規      [設定        アプリ終了
   選択画面]   プロジェクト 画面] ──(Esc)──▶ 起動画面へ戻る
      │        画面]                （"USAGI run away
      │（ワークスペース選択）          ( ^-^)ノ" を表示）
      ▼
 [ホーム画面 Home] ──(Esc / q)──▶ プロジェクト選択画面へ戻る
   （3 ペイン + コマンドモード）
```

※ Open を選ぶとプロジェクト選択画面へ遷移し、ワークスペースを選択するとホーム画面へ進みます。
ホーム画面で `Esc` を押すとプロジェクト選択画面へ戻ります。New を選ぶと新規プロジェクト画面へ、
Config を選ぶと設定画面へ遷移します。設定画面・新規プロジェクト画面で `Esc` を押すと起動画面へ
戻ります。

## 設計メモ（共通）

- 配色には `console` クレートのスタイル（magenta / green / cyan / yellow / red / dim）を使用。
- 中央寄せは「端末幅 − コンテンツ幅」を 2 で割った左パディングで実現（`centered_padding`）。
  端末が狭い場合は `saturating_sub` によりパディング 0 にフォールバックします。
- 各画面の描画ロジックは `src/presentation/tui/<screen>/`（状態 `state.rs`・描画 `ui.rs`・
  イベントループ `event.rs` に分離。ホーム画面はさらにコマンド解析/補完を `command.rs` に分離）、
  端末制御は `src/presentation/tui/screen.rs` に実装されています。
- 中央寄せ・うさぎ AA・タイトル/サブタイトル/フッターなどの共通描画は
  `src/presentation/tui/widgets/`（`mod.rs` の共通描画・`picker.rs`・`dir_picker.rs`）に集約しています。
