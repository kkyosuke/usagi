> Open でワークスペースを選択するとホーム画面へ遷移します。ホーム画面は左ペインにセッション一覧、右ペインに
> 操作対象（切替のプレビュー／在席のアクション UI／没入の埋め込みターミナル）を表示し、**切替・在席・没入**の
> 3 モードを行き来します（起動直後の既定は**切替**）。ワークスペース全体のコマンド（`session` / `config` など）は、
> `:`（コロン）で開く**コマンドパレット（統括 / Overview）**から実行します。詳細は [home/README.md](home/README.md) を参照してください。

## 画面遷移図

```text
            [起動スプラッシュ Splash]
                       │（数秒後・自動で遷移）
                       ▼
                 [起動画面 Welcome]
                       │
      ┌────────┬───────┼────────┬────────┐
   Open(o)   New(e)  Config(c)  Quit(q) / Esc / Ctrl+C
      │        │        │           │
      ▼        ▼        ▼           ▼
 [プロジェクト [新規      [設定        アプリ終了
   選択画面]   プロジェクト 画面] ──(Esc)──▶ 起動画面へ戻る
      │        画面]                （別れのボックスを
      │（ワークスペース選択）          表示）
      ▼
 [ホーム画面 Home] ──(Ctrl+C)──▶ アプリ終了（実行中なら確認モーダル）
   （3 モード: 切替/在席/没入。: でコマンドパレット）
      │
      └─(config コマンド)─▶ [設定画面 Config] ──(Esc / q)──▶ ホーム画面へ戻る
```

※ Open を選ぶとプロジェクト選択画面へ遷移し、ワークスペースを選択するとホーム画面へ進みます。
New を選ぶと新規プロジェクト画面へ、Config を選ぶと設定画面へ遷移します。設定画面・新規プロジェクト画面で
`Esc` を押すと起動画面へ戻ります。ホーム画面は基底の[切替](home/02-layout.md#切替switch既定)では `Esc` で戻らず、
終了は `Ctrl+C` のみです（実行中のセッションがあれば確認モーダルを挟みます）。ホーム画面で `:` を押して開く
[コマンドパレット](home/01-modes.md#コマンドパレット統括overview)で `config` を実行すると設定画面へ遷移し、`Esc` でホーム画面へ戻ります。

## 設計メモ（共通）

- 配色には `console` クレートのスタイル（magenta / green / cyan / yellow / red / dim）を使用。
- 中央寄せは「端末幅 − コンテンツ幅」を 2 で割った左パディングで実現（`centered_padding`）。
  幅は**表示桁（display width）**で測るため、CJK など全角グリフを含むタイトル・フッターも実際の
  描画幅で中央に寄ります。端末が狭い場合は `saturating_sub` によりパディング 0 にフォールバックし、
  さらにコンテンツが端末幅を超えるとき（狭い端末での長いタイトル・フッター）は**末尾を省略（`…`）して
  端末幅にクリップ**するので、行が折り返してフレームを崩すことはありません。
- フッターのキーヒントは端末幅を超えると、**先頭のモードタグと優先度の高いキーを残し、優先度の低い
  末尾のキーから順に落として `…` を付け**ます（全キーは `?` のチートシートで参照できます）。各ペインの行
  （左の一覧・右ペイン）も自分のペイン幅にクリップされるため、`│` の区切り線は常に同じ桁に留まります
  （レイアウトシフトなし）。
- 各画面の描画ロジックは `src/presentation/tui/<screen>/`（状態 `state.rs`・描画 `ui.rs`・
  イベントループ `event.rs` に分離。ホーム画面はさらにコマンド解析/補完を `command/` に分離）、
  端末制御は `src/presentation/tui/io/screen.rs` に実装されています。
- 各画面の再描画は `screen.rs` の `FramePainter` による**差分描画**です。`ui.rs` が返す
  「1 行 1 要素」のフレームを直前のフレームと比較し、**変化した行だけ**を ANSI で位置指定して
  書き換え、1 回の書き込みにまとめます。これにより、キー入力ごとの**ちらつき（フリッカー）が起きません**。初回描画と、モーダル・
  埋め込みターミナル・設定画面など**他画面が上書きした後**は `FramePainter::reset()` で画面全体を
  再描画します（描画差分の基準を破棄）。埋め込みターミナルも同じ差分ロジック（`diff_frame`）を
  共有し、加えて実カーソルをシェルの位置へ重ねます。カーソルの形状（DECSCUSR。ブロック／下線／
  バー）はアクティブなペインのプログラムが選んだものを毎回送り直すため、タブを切り替えると前のタブの
  形状が残らず、選択したペイン自身の形状に戻ります（ペインを離れると端末既定へ戻す）。
- 中央寄せ・うさぎ AA・タイトル/サブタイトル/フッターなどの共通描画は
  `src/presentation/tui/widgets/`（`mod.rs` の共通描画・`picker.rs`・`dir_picker.rs`）に集約しています。
