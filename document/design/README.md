# 画面設計

`usagi hop` で起動する TUI の画面設計をまとめたディレクトリです。全体像は
[../01-overview.md](../01-overview.md) を参照してください。

## 目次

| # | 画面 | ドキュメント | 役割 |
|---|---|---|---|
| 0 | 起動スプラッシュ（Splash） | [00-splash.md](00-splash.md) | 起動直後に一度だけ流れる装飾アニメーション。うさぎが左右に走り回ってからウェルカム画面へ切り替わる |
| 1 | 起動画面（Welcome） | [01-welcome.md](01-welcome.md) | アプリ起動時のトップメニュー。Open / New / Config / Quit を選択する |
| 2 | プロジェクト選択画面（Open） | [02-open.md](02-open.md) | 登録済みワークスペースの一覧から開く対象を選ぶ |
| 3 | 新規プロジェクト画面（New） | [03-new.md](03-new.md) | リポジトリのクローン、または既存ディレクトリの登録でワークスペースを追加する |
| 4 | 設定画面（Config） | [04-config.md](04-config.md) | グローバル設定を編集する |
| 5 | ホーム画面（Home） | [05-home.md](05-home.md) | 選択したワークスペースのセッション一覧とコマンド入力を備えた、4 モードのメイン操作画面 |

> Open でワークスペースを選択するとホーム画面へ遷移します。ホーム画面は左ペインにセッション一覧、右ペインに
> 操作対象（在席のアクション UI／没入の埋め込みターミナル）、下部にコマンド入力欄を表示し、**統括・切替・在席・没入**の
> 4 モードを行き来します。統括の下部コマンドラインは常にアクティブで、`session` / `config` などの TUI 内コマンドを
> 直接入力して実行します（モード起動のためのプレフィックスキーはありません）。詳細は [05-home.md](05-home.md) を参照してください。

## 画面遷移図

```text
            [起動スプラッシュ Splash]
                       │（数秒後・自動で遷移）
                       ▼
                 [起動画面 Welcome]
                       │
      ┌────────┬───────┼────────┬────────┐
   Open(o)   New(e)  Config(c)  Quit(q) / Esc / Ctrl+C
      │        │        │           │
      ▼        ▼        ▼           ▼
 [プロジェクト [新規      [設定        アプリ終了
   選択画面]   プロジェクト 画面] ──(Esc)──▶ 起動画面へ戻る
      │        画面]                （"USAGI run away
      │（ワークスペース選択）          ( ^-^)ノ" を表示）
      ▼
 [ホーム画面 Home] ──(Ctrl+C)──▶ アプリ終了（実行中なら確認モーダル）
   （4 モード: 統括/切替/在席/没入）
      │
      └─(config コマンド)─▶ [設定画面 Config] ──(Esc / q)──▶ ホーム画面へ戻る
```

※ Open を選ぶとプロジェクト選択画面へ遷移し、ワークスペースを選択するとホーム画面へ進みます。
New を選ぶと新規プロジェクト画面へ、Config を選ぶと設定画面へ遷移します。設定画面・新規プロジェクト画面で
`Esc` を押すと起動画面へ戻ります。ホーム画面は最外周の[統括](05-home.md#統括overview)では `Esc` で戻らず、
終了は `Ctrl+C` のみです（実行中のセッションがあれば確認モーダルを挟みます）。ホーム画面のコマンドラインで
`config` を実行すると設定画面へ遷移し、`Esc` でホーム画面へ戻ります。

## 設計メモ（共通）

- 配色には `console` クレートのスタイル（magenta / green / cyan / yellow / red / dim）を使用。
- 中央寄せは「端末幅 − コンテンツ幅」を 2 で割った左パディングで実現（`centered_padding`）。
  端末が狭い場合は `saturating_sub` によりパディング 0 にフォールバックします。
- 各画面の描画ロジックは `src/presentation/tui/<screen>/`（状態 `state.rs`・描画 `ui.rs`・
  イベントループ `event.rs` に分離。ホーム画面はさらにコマンド解析/補完を `command/` に分離）、
  端末制御は `src/presentation/tui/screen.rs` に実装されています。
- 各画面の再描画は `screen.rs` の `FramePainter` による**差分描画**です。`ui.rs` が返す
  「1 行 1 要素」のフレームを直前のフレームと比較し、**変化した行だけ**を ANSI で位置指定して
  書き換え、1 回の書き込みにまとめます。毎フレーム全画面をクリアして全行を再描画する方式を
  やめたことで、キー入力ごとの**ちらつき（フリッカー）を解消**しています。初回描画と、モーダル・
  埋め込みターミナル・設定画面など**他画面が上書きした後**は `FramePainter::reset()` で画面全体を
  再描画します（描画差分の基準を破棄）。埋め込みターミナルも同じ差分ロジック（`diff_frame`）を
  共有し、加えて実カーソルをシェルの位置へ重ねます。
- 中央寄せ・うさぎ AA・タイトル/サブタイトル/フッターなどの共通描画は
  `src/presentation/tui/widgets/`（`mod.rs` の共通描画・`picker.rs`・`dir_picker.rs`）に集約しています。
