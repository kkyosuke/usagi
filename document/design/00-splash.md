# 0. 起動スプラッシュ（Splash）

> [画面設計トップ](README.md) ｜ 次へ → [1. 起動画面（Welcome）](01-welcome.md)

`usagi hop` 実行時、[1. 起動画面（Welcome）](01-welcome.md) に入る前に一度だけ再生される装飾的なオープニング。
うさぎが画面を左右に走り回ったのち、ウェルカムメニューへ自動で切り替わります。代替スクリーンの初期化や
マウスレポートの方針はウェルカム画面と共通です（[01-welcome.md](01-welcome.md) を参照）。

## レイアウト

うさぎとタイトルのブロックを端末の縦中央に配置します（幅・高さが取得できない環境では 80×24 を
フォールバック値として使用）。うさぎは毎フレーム水平位置が変わり、左端から右端へ往復します。

```text
┌───────────────────────────────────────────────────────┐
│                                                       │
│                                                       │
│              ∩∩                                       │  ┐ 走るうさぎ（マゼンタ・太字）
│             ﾐ(･ㅅ･)                                    │  │ 進行方向の後ろにスピード線（ﾐ）。
│                                                       │  ┘ 上下にぴょこぴょこ跳ねながら左右へ走る
│                       USAGI                           │  ← タイトル（緑・太字）
│                                                       │
│                                                       │
└───────────────────────────────────────────────────────┘
```

## 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| 走るうさぎ | 3 行のアスキーアート。耳 `∩∩` と顔 `(･ㅅ･)`、進行方向の後ろに付くスピード線 `ﾐ`（右走り `ﾐ(･ㅅ･)` / 左走り `(･ㅅ･)ﾐ`） | マゼンタ・太字 |
| タイトル | `USAGI` | 緑・太字 |

## アニメーション

- うさぎは 1 回の往復（左端 → 右端 → 左端）を一定フレーム数かけて走り抜けると終了します。
- 各フレームで水平位置を進めつつ、ブロック内で 1 行ぶん上下に跳ねることで「跳ねながら走る」動きになります。
- 進行方向に合わせて左右を反転し、スピード線（`ﾐ`）が常に進む向きの後ろに付きます（顔の鼻は一つのまま）。
- フレームは一定間隔（[ホーム/インストールと共通の描画間隔](../05-settings.md)）で自動的に進み、
  全フレームの再生後にウェルカム画面へ切り替わります。所要時間は数秒です。

## 入力の扱い

スプラッシュは**入力を読み取りません**。自前のタイマーでフレームを進めるだけなので、再生中に押した
キーはバッファされ、そのまま次のウェルカムメニューに渡ります（タイプアヘッド）。ウェルカム画面が
ブロッキングで待ち受ける入力経路と競合しないため、再生中のキー入力が失われることはありません。

> 描画ロジックは `src/presentation/tui/splash/`（描画 `ui.rs`・イベントループ `event.rs`）、走るうさぎの
> アートは共有ウィジェット `src/presentation/tui/widgets/` に実装されています。
