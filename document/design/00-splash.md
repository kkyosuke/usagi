# 0. 起動スプラッシュ（Splash）

> [画面設計トップ](README.md) ｜ 次へ → [1. 起動画面（Welcome）](01-welcome.md)

`usagi hop` 実行時、[1. 起動画面（Welcome）](01-welcome.md) に入る前に一度だけ再生される装飾的なオープニング。
うさぎが左端から 1 匹ずつ増えていき（`usagi run 2`「増えていくうさぎ」と同じアニメーション）、満員になると
ウェルカムメニューへ自動で切り替わります。代替スクリーンの初期化やマウスレポートの方針はウェルカム画面と
共通です（[01-welcome.md](01-welcome.md) を参照）。

## レイアウト

うさぎ（3 行）とタイトルのブロックを端末の縦中央に配置します（幅・高さが取得できない環境では 80×24 を
フォールバック値として使用）。うさぎの行は**左端に寄せて固定**し、フレームが進むごとに右へ 1 匹ずつ
増えていきます。1 匹目は常に同じ列にとどまるため、増えても表示がガタつきません。

```text
┌───────────────────────────────────────────────────────┐
│                                                       │
│                                                       │
│ n_n   n_n   n_n                                       │  ┐ 増えるうさぎ（マゼンタ・太字）
│(｡･-･)(｡･-･)(｡･-･)                                      │  │ 左端から右へ 1 匹ずつ並んで増える
│ └┘    └┘    └┘                                         │  ┘ 1 匹目は左端で固定（位置はずれない）
│                       USAGI                           │  ← タイトル（緑・太字・中央寄せ）
│                                                       │
│                                                       │
└───────────────────────────────────────────────────────┘
```

## 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| 増えるうさぎ | 3 行のアスキーアート。耳 `n_n`・顔 `(｡･-･)`・足 `└┘` を 1 匹 6 列幅の固定セグメントで横に並べる（`usagi run 2` と共通） | マゼンタ・太字 |
| タイトル | `USAGI` | 緑・太字（中央寄せ） |

## アニメーション

- うさぎは 1 匹から始まり、数フレームごとに 1 匹増えて満員（既定 8 匹）になると終了します。
- うさぎの並びは左端に固定され、新しい 1 匹は常に右へ追加されます。各セグメントが固定幅のため、
  すでに表示されているうさぎは横にずれません（レイアウトシフトなし）。
- 行数は常に 3 行で一定なので、縦方向にもずれません。
- フレームは一定間隔（[ホーム/インストールと共通の描画間隔](../05-settings.md)）で自動的に進み、
  全フレームの再生後にウェルカム画面へ切り替わります。以前の左右往復より短い、ごく短い演出です。

## 入力の扱い

スプラッシュは**入力を読み取りません**。自前のタイマーでフレームを進めるだけなので、再生中に押した
キーはバッファされ、そのまま次のウェルカムメニューに渡ります（タイプアヘッド）。ウェルカム画面が
ブロッキングで待ち受ける入力経路と競合しないため、再生中のキー入力が失われることはありません。

> 描画ロジックは `src/presentation/tui/splash/`（描画 `ui.rs`・イベントループ `event.rs`）、増えるうさぎの
> アートは共有ウィジェット `src/presentation/tui/widgets/`（`usagi run 2` と共通）に実装されています。
