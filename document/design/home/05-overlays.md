## 更新通知（最新リリースの検知）

ホーム画面を開くと、バックグラウンドで GitHub リモートのリリースタグ（`v*`）を取得し、
**実行中ビルドより新しいバージョンが公開されていれば**、左ペイン最下部の[マスコットのうさぎ](02-layout.md#レイアウト)が
**アイドル時に**吹き出しで `アップデートがあるぴょん` ＋ 新バージョン（`v<X.Y.Z>`）を喋ります。吹き出しは黄色・太字（更新の
アクセント色）でうさぎの頭へ向かう尻尾（`┬`）が付き、うさぎ本体は[モード色](02-layout.md#レイアウト)のままなので、通知と
マスコットが見分けられます。差分が無い（最新と同じ・取得失敗・オフライン）ときは何も喋りません。ローディングや
バックグラウンドタスクがある間は、それらの「いま何をしているか」を同じ吹き出しで優先して喋り、更新通知は作業が落ち着いてから表示します。

```text
┌────────────────────────────────────────────────────────────────┐
│                  usagi · 4 sessions                            │  ← ヘッダー行
│             Switch › Closeup                          │
│                                                                │
│  ▎ ⌂   root              │  （右ペインはモードの内容を表示）   │
│  ▎     workspace root    │                                     │
│    ●   main              │                                     │
│                          │                                     │
│ ╭──────────────────────╮ │                                     │
│ │ アップデートがあるぴょん │ │  ← 左下のうさぎが吹き出しで喋る     │
│ │ v0.2.0               │ │                                     │
│ ╰─┬────────────────────╯ │                                     │
│  (\(\                    │                                     │
│  (o.o)?                  │                                     │
│ o(_(")(")                │                                     │
└────────────────────────────────────────────────────────────────┘
```

- 取得は git CLI（`git ls-remote --tags`）で行い、HTTP 依存を増やさず、ユーザーの git 認証/プロキシ設定を
  そのまま使います。ネットワークを待つ間も画面はブロックしません（別スレッドで実行し、結果が出たら次の
  再描画で通知が現れる）。
- 吹き出しはサイドバー幅に合わせて折り返すので、文言がはみ出して隣のペインを侵食しません。マスコットが出ない条件
  （セッション一覧が最下部まで届く・サイドバーをレールに畳む・幅が足りない）では、うさぎごと通知も引っ込みます。
- バージョン比較は純粋な `domain/version.rs`（`Version`）、最新判定は `usecase/update_check.rs`、タグ取得は薄い
  IO ラッパ `infrastructure/release.rs`（カバレッジ計測対象外）、結果の受け渡しは `home/update.rs`
  （`UpdateHandle`）、描画は純粋な `home/ui/`・`widgets/`（`workspace_rabbit_speaking`）に実装しテスト済み。

### クリックでアップデート

更新通知を喋っているうさぎ本体を**左クリック**すると、[アップデート確認モーダル](#アップデート確認モーダル)が開きます。
`y` / `Enter` で**バックグラウンドでアップデートを実行**し（README と同じインストールスクリプト
`curl … install.sh | bash` を再実行して `~/.usagi/bin/usagi` を最新リリースに置き換える）、`n` / `Esc` でやめます。

- 実行中は[ローディングうさぎ](#処理中の表示ローディングうさぎ)が `アップデート中…` を表示し、その間も usagi を使い続けられます
  （別スレッドで実行）。完了すると `アップデートしたよ！反映するには usagi を再起動してね 󰤇` を、失敗すると
  失敗メッセージを数秒うさぎが伝えます。
- **置き換えるのはディスク上のバイナリ**で、実行中のプロセスには反映されません。新しいバージョンで動かすには
  **usagi を再起動**してください（再起動するまで通知はそのまま残るので、再びクリックすればまた確認できます）。
- 実行コマンドの組み立てと結果メッセージは純粋な `usecase/self_update.rs`（`CommandRunner` 注入でテスト済み）、
  バックグラウンド実行と完了メッセージの共有は `home/update.rs` の確認モーダル状態と
  [ローディングうさぎ](#処理中の表示ローディングうさぎ)（`install_task`）に実装。実バイナリの起動（スレッド spawn と
  実プロセス）は薄いオーケストレータ `home/mod.rs`（カバレッジ計測対象外）が束ねます。

## バックグラウンドタスク（セッション作成・削除）

セッションの**作成**（`session create` / 選択の `c`）と**削除**（`session remove` / 削除モーダル / `close`）は
git の worktree 操作（worktree add・submodule init・worktree remove）で数秒かかることがあるため、**バックグラウンドの
ワーカースレッドで実行**します。コマンドを投げた瞬間にイベントループへ戻るので、**処理中も画面の操作（一覧移動・
タブ選択・別セッションへの没入など）は一切止まりません**。実行中・完了直後のタスクは、**左下のうさぎの吹き出し**で
代表タスク（複数ある場合は「ほか N 件」）を喋らせます。セッション作成・削除の進行中表示は**サイドバー内の
スケルトン行**で表し、上部右側は入力待ち（`◆ waiting`）通知のために空けます。

```text
┌────────────────────────────────────────────────────────────────┐
│        usagi · 4 sessions                          1 waiting  │  ← タイトルバー右は waiting 通知
│             Switch › Closeup                         │
│                                                                 │
│  ▎ ⌂   root              │  （右ペインは別モードの内容を表示）  │
│  ▎     workspace root    │                                     │
│    ●   main              │                                     │
│                          │                                     │
│    ✂   old-session       │  ← 削除中は既存行を置き換える        │
│        removing session  │                                     │
│        pruning worktree  │                                     │
│    +   wip               │  ← 作成中は + new の直前へ挿入       │
│        creating session  │                                     │
│        CPU 0% MEM 0MB    │                                     │
│ ╭──────────────╮         │                                     │
│ │ 削除中… old  │         │  ← 左下のうさぎが現在の作業を喋る    │
│ ╰─┬────────────╯         │                                     │
│  (\(\                    │                                     │
│  (o.o)?                  │                                     │
│ o(_(")(")                │                                     │
└────────────────────────────────────────────────────────────────┘
```

- 吹き出しは[更新通知](#更新通知最新リリースの検知)と同じマスコット吹き出しを使い、ローディング中 ＞ バックグラウンドタスク ＞
  更新通知の順で内容を選びます。タスクが複数あるときは 1 行目に代表タスク、2 行目に `ほか N 件` を出します。クリック反応中は
  反応アニメーションを優先し、終わると元の吹き出しへ戻ります。
- セッション作成は、対象ワークスペースの `+ new session` 行の直前に、実セッションと同じ 3 行（名前 / 詳細 / CPU・MEM）の
  **スケルトン行**を挿入して待機中を示します。完了時に同じ位置へ実セッション行が入るため一覧の高さがずれず、`+ new session`
  行はスケルトンの下に残って選択できます。新規タブ追加時のタブチップと同じ「配置される場所で待つ」見せ方で、ウェーブは
  タブのローディングと同じ青系のアクセントで流します。
- セッション削除は、対象セッションの既存 3 行を**その場で削除スケルトンに置き換えます**。行数を増やさないためスクロール位置・
  `+ new session` 行・クリック hit-test はずれません。削除スケルトンは `✂` と `removing session` / `pruning worktree` を使い、
  作成中の青系ウェーブと区別できる葉色（緑系）のウェーブで「刈り取っている」状態を示します。成功するとその行は一覧更新で消え、
  失敗・panic 時はスケルトンを解除して元の行を残します。
- バックグラウンドタスクの内部ビューは、実行中は時間由来のスピナー、完了後は `✓` / `✗` を持ち、完了後も短時間だけ保持します。
  画面上ではこのビューをマスコット吹き出しの文言選択に使い、セッション作成・削除そのものはサイドバー skeleton に寄せます。
- ワーカーは git 処理を**直列化**（共有ロック）して `state.json` の競合を防ぎます（同期処理のリネームも同じロックを取ります）。
  終わったものからイベントループが回収して結果をログへ出し一覧を更新します。**作成を投げてから完了まで他のキー・クリック・
  ホイール操作が無ければ、その新規セッションへ自動で集中（Closeup）します**（ユーザー自身の操作で作った直後なので、移動せず
  そのまま操作に入れるようにするため）。同じく **`close` は削除を投げた直後はいったん選択へ出ますが、完了まで他の操作が無ければ
  選択（Switch）のまま閉じたセッションの 1 つ上（上に無ければ下）のセッションへカーソルを移します**。root 行は landing 先にしないため、先頭セッションを
  閉じても `⌂ root` へ戻りません。作成・close の待ち時間中に別の操作をした場合、一覧だけを更新して現在の操作先を保ちます。
  `session remove <name>` や削除モーダルなど、一覧だけが変わる更新では**カーソルを動かしません**。
  なお自動集中は TUI からの作成だけで、MCP（`session_create`）経由の作成は TUI を介さないため集中には入りません。
- ホーム画面を**閉じる（戻る／終了）ときは、実行中のタスクの git 処理が終わるのを待ってから**抜けます。途中で打ち切って
  worktree や `state.json` を中途半端な状態で残さないためで、稼働中のタスクがあれば終了が数秒待たされることがあります。
- 表示優先順位は、右ペイン中央の**ローディングうさぎ**（後述の同期処理）が最優先、サイドバーでは**作成 / 削除 skeleton**、
  タイトルバー右上では**入力待ち通知**が優先です。入力待ち通知は、入力待ち（`◆ waiting`）のセッションが 1 件以上あるときに
  ` N waiting`（Nerd Font のベル、黄色太字）をタイトルバー右上へ 1 行で出し、左ペインをスクロール／折りたたみしていても
  回答・許可待ちの存在を知らせます。[更新通知](#更新通知最新リリースの検知)は右上ではなく左ペインのマスコットが喋るため、
  ローディングやタスクが無いときに表示します。
- 状態保持・時間由来のビューは純粋な `home/tasks.rs`（`TaskHandle`、テスト済み）、描画は `home/ui/`（サイドバー skeleton と
  マスコット吹き出し）、ワーカースレッドの起動は計測対象外の `home/mod.rs` に実装。

## 処理中の表示（ローディングうさぎ）

ターミナル / Agent の起動は**そのセッションにまだライブなペインが無いか（＝タブが無い）**、**すでにタブがあるか**で見せ方が変わる。

**タブが無いセッションの最初のペイン**を起動するときは、右ペイン中央に `usagi run 2` と同じ**増えていくうさぎ**を重ねて
表示する。この起動（PTY の生成と Agent CLI の投入）は短い同期処理で、完了までイベントループを止める。処理開始の直前に
短い最低表示時間ぶん複数フレームを描画してから処理を実行するため、「待っている」ことが伝わる。
**この同期処理中はローディングうさぎを最優先**で見せ、左下のうさぎは `エージェント起動中…` などのラベルを吹き出しで喋る。
終わるとそのペインへ没入する。

**すでにタブがあるセッションに集中の `terminal` / `agent` で新しいペインを足す**ときは、画面を止めず、
**新しいタブをタブストリップに追加**して**そのチップ上でローディングアニメーション**（全体を淡色にしつつ
青のバンドが左から右へ流れるウェーブ）を回す。ユーザーは集中のまま現在のペインを見たまま待ち、**ペインの準備が
整うと（シェルが最初の描画を出すと）そのタブへ自動で移動（没入）**する。ただし**準備が整うまでのあいだに何か操作した場合は
移動せず**、新しいタブはそのまま背景のライブなタブとして残る（タブストリップから選んで入れる）。この「操作が無ければ移動」は
バックグラウンドのセッション作成／削除と同じ interaction epoch の判定を使う。既に同じ CLI の agent タブがあるときは
新しいタブを作らずそのタブへ移動する。

このとき[1Password 参照の環境変数](#workspace-env-エディタ)（`op://…`）の解決も**中央うさぎではなく、このタブバー内のローディングで待つ**。
env 解決はワーカースレッドで走らせてイベントループを止めず（`op read` は数秒かかりうる）、解決が済むまではプールに実体の無い
**placeholder チップ**（`+ new` チップと同じ合成タブ）をタブストリップ末尾に出してウェーブし、解決が届いた時点でペインを spawn
して実タブへ切り替える。中央うさぎ＋`環境変数を解決中…` を出すのは、下記の**タブが無い最初のペイン**のときだけ。

```text
┌────────────────────────────────────────────────────────────────┐
│                  usagi · 4 sessions                            │
│             Switch › Closeup                          │
│                                                                │
│  ▎ ⌂   root              │                                      │
│  ▎     workspace root    │              n_n  n_n  n_n            │  ← 右ペイン中央に run 2 のローディング
│    ●   main              │             (｡･-･)(｡･-･)(｡･-･)       │
│                          │              └┘   └┘   └┘            │
│                          │      （右ペイン内容は非表示）       │
└────────────────────────────────────────────────────────────────┘
```

- （中央うさぎ＝タブ無しの最初のペイン）右ペインには旧来の `ターミナル起動中…` / `エージェント起動中…` の文字ラベルを出さず、中央のうさぎアニメーションだけで待機中を示す。ラベルは左下のうさぎが吹き出しで喋る。
- 処理直前に最低 180ms（60ms 間隔で 4 フレーム）だけ出して、処理が画面を描き直すまで残る。起動が速い場合でも `run 2` の増加が見える。
- 同期処理中の右ペインはローディング専用画面になり、集中のアクションメニュー（`agent` / `terminal` などの選択肢）やプレビューは背面に表示しない。うさぎは右ペイン内で中央配置し、右ペイン幅や高さが足りない場合は、はみ出さないよう描画を見送る。**この中央うさぎは、そのセッションにタブが無いときだけ出す**（タブがあれば下記のタブ内ローディングになる）。
- （タブ内ローディング＝既存タブありの新規ペイン）ペインはバックグラウンドで（アクティブにせず）spawn し、新しいタブをタブストリップに足してそのチップをウェーブ表示する。準備完了は各ペインの reader スレッドが進める画面世代（generation）で判定し、イベントループが毎フレーム監視する。操作が無ければそのタブへ再アタッチし、操作があれば追跡をやめて背景タブとして残す。
- 描画は純粋な `widgets/`（`multiplying_rabbits`）と `home/ui/`（中央うさぎ・タブ内ウェーブ）、状態保持は `home/state/`（`LoadingIndicator`・`PendingPane`）、
  処理直前のフレーム送出とバックグラウンド spawn／準備完了監視は `home/event/` に実装しテスト済み。
- （中央うさぎのときだけ）タブが無い最初のペインを起動する直前は、そのワークスペースの
  [1Password 参照の環境変数](#workspace-env-エディタ)（`op://…`）を解決するあいだ、**同じ増えていくうさぎを右ペイン中央に重ねて**
  表示し、うさぎの**下に `環境変数を解決中…` のラベル**を添える（`op read` は 1Password のロックや遅延で数秒かかりうるため）。
  タブがあるセッションの新規タブでは、これは上記のタブ内ローディング（placeholder チップ）に置き換わり、中央うさぎは出さない。
  参照が無い（またはすぐ解決できる）ワークスペースでは何も出さない。右ペイン幅・高さが足りず 1 羽も収まらないときは描画を見送る。

## TUI 内コマンド

コマンドパレット（`:` で開く・Workspace スコープ）／集中の右ペイン（Session スコープ）で実行できる
TUI 内コマンド（`session` / `terminal` / `agent` / `config` ＋ 共通の `man` / `history` / `clear` / `quit`）の
**構文と役割は [../03-commands/02-tui.md](../../03-commands/02-tui.md) が正本**です。本書は画面側の挙動に絞ります。

`man` / `history` / `session list` などテキストを読むコマンドは、結果帯ではなく画面中央の
**スクロール可能なテキストモーダル**に出力します（`↑↓`/`j`/`k`・PageUp/PageDown でスクロール、`Esc` / `Enter` / `q`
で閉じる）。それ以外のコマンドのレスポンスは、**直前の 1 件だけ**を入力欄直下の帯に表示します。

`session overview` で選んだ **アクティブな行（worktree またはルート行）** は左ペインで左端の `▎`（緑）バーと太字で
強調表示され、後続コマンド（`terminal` / `agent`）の実行対象になります。アクティブ表示（`▎`、エントリの全行にわたる。セッションは 3 行・ルートは 2 行）と
キーボードのカーソル（`>`、選択時のみ 1 行目に表示）は独立しています。一覧の先頭には、どのセッションにも属さない**ルート行**（`root`）が
常設され、これをアクティブにすると `terminal` / `agent` はワークスペースルートで起動します。

> TUI 内コマンドは `command/` の `Command` トレイト＋`CommandRegistry` として実装されており、ディスパッチ・
> 補完・`man` 一覧はすべてこのレジストリ経由です。拡張点としての設計は [../02-architecture.md](../../02-architecture.md#tui-内コマンドのレジストリ) を参照してください。

## 集中のアクションのオーバーレイ

[集中（Closeup）](01-modes.md#2-つの-mode)のアクション UI は、設定 `session_action_ui` が **Menu**（既定）でも
**Prompt** でも、右ペインにインライン展開せず、**右ペイン領域の中央に浮かぶ枠付きのオーバーレイモーダル**として表示します。
Menu ならコマンドの選択肢（`agent` / `chat` / `close` / `diff` / `terminal`）が、Prompt ならセッションスコープのコマンドラインが
そのボックスの本体になります（設定は*どちらの surface を出すか*だけを選び、浮かせ方は共通）。

```text
┌───────── サイドバー ─────────┬──────────────── 右ペイン ────────────────┐
│  ▎ 󰤇 main                  │  main  │ [1 agent] [+ new]              │  ← タブストリップは背後に見える
│  ▎    🤖 ▶                  │      ┌─ session: main ──────────────┐  │
│                            │      │ Run a command:                │  │  ← 中央に浮かぶメニュー
│                            │      │ › Launch Claude               │  │
│                            │      │   Open a shell                │  │
│                            │      │ ↑↓ move  Enter run  t … a …   │  │
│                            │      └───────────────────────────────┘  │
└────────────────────────────┴───────────────────────────────────────┘
```

Prompt のときも同じ位置に浮かび、本体がコマンドラインとヒントに変わります:

```text
┌───────── サイドバー ─────────┬──────────────── 右ペイン ────────────────┐
│  ▎ 󰤇 main          synced   │  main  │ [1 agent] [+ new]              │  ← タブストリップは背後に見える
│  ▎    ▶ running             │      ┌─ session: main ──────────────┐  │
│                            │      │ ❯ terminal▏                   │  │  ← 中央に浮かぶコマンドライン
│                            │      │                               │  │
│                            │      │ terminal  埋め込みタブを追加   │  │  ← usage / examples ヒント
│                            │      └───────────────────────────────┘  │
└────────────────────────────┴───────────────────────────────────────┘
```

- **中央オーバーレイ（モーダル）**で、[コマンドパレット](01-modes.md#2-つの-modal)やテキストモーダルと同じく
  全画面の暗幕ではなく、組み上げたワークスペース画面の上に枠付きボックスを合成します。**左サイドバーは常に見え**、
  フォーカス中セッションがライブなペインを持つときは**背後にタブストリップも残ります**。
- **没入からズームアウトした直後（既定 `Ctrl-O a` / `Ctrl-T`、`alt` 方式は `Alt-a`）は元のペインのタブに留まったまま浮かびます** —
  タブストリップに「+ new」チップは増えず（まだ何も作っていないため）、**そのペインのライブプレビューがボックスの背後に
  見えたまま**です。キーボードは浮いているアクション surface が握り、`Esc` はズームアウト直後なら元のペインへ
  再アタッチ、他のキーを押したあとなら surface だけを閉じてプレビューに戻ります。`Ctrl-N`/`Ctrl-P` やタブのクリックで
  タブ巡回を始めたときも閉じます。「+ new」タブを選んだとき（またはライブなペインが無いとき）は
  「+ new」の上に浮かび、背後の右ペインは空です。Menu も Prompt も同じ挙動です。
- ボックスの**タイトルにセッション識別子**（`session: <name>`）が載ります。本体は surface ごとに異なり、**Menu はコマンド行と
  操作ヒント**、**Prompt は `❯` コマンドラインと `usage`/`examples` ヒント**（[コマンドパレット](01-modes.md#2-つの-modal)の
  入力欄と同じ流儀）です。どちらも行構成・カーソル・ピッカーは集中インライン表示だったころと同一で、置き場所だけが
  右ペイン中央のボックスへ移りました。
- **どちらの surface も高さは固定**で、内容が変わってもボックスがリサイズしません（レイアウトシフトなし）。
  - **Menu は「一番サブメニューが多いピッカーを展開したときの行数」に固定**します。コマンド領域はその最大展開ぶんの高さを
    最初から確保するので、どのピッカーを開いても**ボックスの高さは変わらず**、サブ行はその場に展開されて `↑/↓ N more` に
    畳まれません。たたんだ状態では余白（空行）で高さを保ちます。最大展開ぶんが右ペインの高さに収まらないときだけ窓は中で
    スクロールし、カーソル（またはピッカーの選択行）が常に見えるよう追従、隠れた行があるときに上端・下端へ
    `↑ N more` / `↓ N more` を出します（メモエディタと同じ流儀）。最大展開ぶんは、コマンド行数に「サブ行が最も多いピッカー」
    （インストール済み Agent が複数あるときの `agent`、または 2 択の terminal / close）の行数を足したものです。
  - **Prompt はヒント欄を最大行数ぶん確保**します（`usage` 1 行 ＋ examples の最大行数）。ヒントがコマンド候補でも
    usage/examples でも無しでも、足りないぶんは空行で埋めて高さを一定に保つので、入力に応じてヒントが増減してもボックスは
    ガタつきません（`:` [コマンドパレット](01-modes.md#2-つの-modal)の入力欄と同じ流儀）。右ペインが低いときは
    収まるぶんに詰めます。
- **キー操作は surface ごとにそのまま**です。Menu は `↑↓`/`j`/`k` で移動、`Enter` で実行、`t`/`a` で terminal/agent、`Shift`+`c` で
  `close --force`、`→`/`Tab` で `agent`/`terminal`/`close` のピッカー展開。Prompt は文字入力・`Tab` 補完・`Enter` 実行で、
  `?`・`End` はコマンドラインの文字／キャレット移動として扱われます（チートシート／メモは開きません）。集中へ入る `Ctrl-O`
  （選択へ）・`Ctrl-O a`（没入から集中へ）などの[キー方式](04-keys.md#closeup)もそのままです。
- ほかの画面占有サーフェス（[ローディングうさぎ](#処理中の表示ローディングうさぎ)・開いている各[オーバーレイ](#セッションメモの編集)
  ＝メモエディタ・テキストモーダル・プレビュー・diff・`:` パレットなど）が出ているあいだは**浮かせません**。それらが
  キーボードを握って独自のボックスを描くので、二重に重ならないよう譲ります（背後の右ペインは空のまま）。

> 描画は `widgets` の `overlay_region_modal`（`boxed` ＋ `overlay_region_centered`。右ペイン矩形の中央にボックスを合成）を
> 使い、表示条件は `home/state.rs` の `HomeState::closeup_action_overlay` が単一情報源です（`home/ui/` の `render_frame` と
> `closeup_pane` の双方がこの述語を読み、surface の浮上と背面の描画 — 「+ new」タブでは空白、ペインのタブ上では
> ライブプレビュー — が食い違わないようにします）。

## 終了確認モーダル

ホーム画面を閉じる手段は **`Ctrl+C`** と **`Ctrl+Q`** の 2 つです（`Esc` は基底の[選択](02-layout.md#選択overview既定)では無効）。
どちらも閉じる前に画面中央へ**終了確認モーダル**を出し、誤操作で動作中の Agent を巻き込んで終了するのを防ぎます。
2 つの違いは確認を出す条件です。

- **`Ctrl+C`**: 実行中のセッション（埋め込みシェル / Agent がライブのもの）があるときだけ確認モーダルを挟みます。
  ライブなセッションが 1 つもなければ確認なしで即終了します。
- **`Ctrl+Q`**: 専用の終了キーで、ライブの有無にかかわらず**必ず**確認モーダルを出します（単押しでは決して終了しない）。
  ライブ端末（Closeup 内部状態）中でも効く**グローバルキー**で、没入では一旦ペインを抜けて同じモーダルを出します（全ペインは生かしたまま）。
  没入の `Ctrl+C` は Agent への割り込み（SIGINT）としてシェルへ流れるため、没入から確実に終了したいときは `Ctrl+Q` を使います。

```text
┌─ Quit usagi? ──────────────────────────────────┐
│ 2 session(s) still running.                    │
│                                                │
│ Close anyway? Running agents will be stopped.  │
│                                                │
│ y / Enter: close   n / Esc: cancel             │
└────────────────────────────────────────────────┘
```

ライブなセッションが無い状態で `Ctrl+Q` を押したときは、停止する Agent が無いので文言が変わります。

```text
┌─ Quit usagi? ──────────────────────────────────┐
│ No sessions are running.                        │
│                                                │
│ Quit usagi?                                     │
│                                                │
│ y / Enter: quit   n / Esc: cancel              │
└────────────────────────────────────────────────┘
```

- `y` / `Y` / `Enter`（またはもう一度 `Ctrl+C` / `Ctrl+Q`）で**終了を確定**（`Outcome::Quit`）。
- `n` / `N` / `Esc` で**キャンセル**し、直前のモードへそのまま戻る（セッションは動き続ける）。
- ライブ判定は端末プール（`MonitorHandle::live()`）の状態を毎フレーム反映した `HomeState::has_live_sessions()` による。
- モーダル基盤（`widgets` の `render_modal`）を再利用し中央寄せ・枠付きボックスで描画。状態は `home/state.rs`
  （`quit_confirm`）、描画は `home/ui/`、キー処理は `home/event/` の event loop 先頭で全モード共通に横取りする。

## アップデート確認モーダル

うさぎが[更新通知](#更新通知最新リリースの検知)を喋っているときに**うさぎ本体を左クリック**すると、画面中央へ
**アップデート確認モーダル**を出します。最新版を入れてよいか尋ね、確定すると[クリックでアップデート](#クリックでアップデート)を実行します。

```text
┌─ アップデート ──────────────────────────────┐
│ 最新版 v0.2.0 があるよ。                    │
│                                            │
│ ダウンロードして入れ替える？                │
│ （反映には usagi の再起動が必要）            │
│                                            │
│ y / Enter: 更新   n / Esc: やめる           │
└────────────────────────────────────────────┘
```

- `y` / `Y` / `Enter` で**アップデートを実行**し、モーダルを閉じます（進行は[ローディングうさぎ](#処理中の表示ローディングうさぎ)で表示）。
- `n` / `N` / `Esc` で**キャンセル**し、通知を喋るうさぎへそのまま戻ります。
- 終了確認モーダルと同居しません（そのモーダルが開いているあいだ、およびコマンドパレット・プレビュー・メモ編集が
  開いているあいだと没入中は、うさぎはクリックできません）。
- モーダル基盤（`widgets` の `render_modal`）を再利用。状態は `home/state.rs`（`update_confirm`）、描画は `home/ui/`、
  キー処理は `home/event/` の event loop で[終了確認モーダル](#終了確認モーダル)の直後に横取りする。

## セッション削除モーダル

`session remove` を **名前なし** で実行すると、コマンドは `Effect::OpenRemoveModal` を返し、
event loop が記録済みセッションのチェックリストを**中央オーバーレイ（モーダル）**として表示します。
単一ワークスペースではセッション名だけを並べ、統合（unite）モードでは表示中のすべてのワークスペースのセッションを
`<workspace>: <session>` の形で並べます。
[コマンドパレット](01-modes.md#2-つの-modal)やテキストモーダルと同じく**全画面の暗幕ではなく**、
背後のワークスペース（タイトルバー・モードラダー・サイドバー・両ペイン）の**上に重ねて合成**し、
ボックスが覆うのは中央の桁だけで左右・上下には背後の画面が見え続けます。
`session remove <name>` のように名前を渡した場合は従来どおりモーダルを介さず直接削除します。

```text
┌─ Remove sessions ─────────────────────────┐
│ Select sessions to remove (Space to ...). │
│                                           │
│ > [x] usagi: feature-a                    │
│   [ ] tools: feature-b                    │
│   [x] app: fix-login                      │
│                                           │
│ 2 selected                                │
│                                           │
│ Space: toggle   Enter: remove   Esc: ...  │
└───────────────────────────────────────────┘
```

- `↑`/`↓`（または `j`/`k`）でカーソルを移動し、`Space` でカーソル行の選択/非選択を切り替える。
- `Enter` で選択したセッションを**一括削除**。各セッションは[バックグラウンドタスク](#バックグラウンドタスクセッション作成削除)として
  ワーカースレッドへ投入される。モーダルは即座に閉じ、対象セッション行はサイドバー内の削除スケルトンに変わり、画面は止まらない。
  未コミット変更があるセッションは `--force` なしでは削除されず警告される（`session remove --force` で開くと破棄を伴って削除）。
- 何も選択していない状態の `Enter` は no-op でモーダルを開いたまま、`Esc` でキャンセルしてホーム画面へ戻る。
- セッション数が表示枠を超える場合はカーソルが収まるようスクロールし、隠れた件数を `↑ N more` / `↓ N more` で示す。
- モーダル基盤（`src/presentation/tui/widgets/` の `boxed` / `overlay_modal`）を再利用し、組み上げたワークスペース画面の上に
  中央寄せ・枠付きボックスを合成する。状態は `home/state.rs`（`RemoveModal`）、描画は `home/ui/`、キー処理は `home/event/` に実装。

## workspace env エディタ

[`env` コマンド](../../03-commands/02-tui.md#env)を実行すると、`Effect::OpenEnvEditor` を受けて event loop が
**workspace-env エディタ**を[コマンドパレット](01-modes.md#2-つの-modal)の**上に重なる中央オーバーレイ**として開きます。
このワークスペースの 1Password 参照の環境変数（`NAME=op://vault/item/field`。pane 起動時に解決して子プロセス環境へ注入する。
設定項目は [5. 設定](../../05-settings.md#設定項目)）を 1 行 1 件で編集する複数行エディタです。**パレットは開いたまま**背後に残るので、
保存でも取消でもオーバーレイを閉じると **コマンドパレットに戻ります**（Config 画面へは遷移しません）。同じ workspace binding は
[Config 画面](../04-config.md)をワークスペーススコープで開いたときの `Env Vars` 行からも編集できます。

```text
┌─ Env Vars ─────────────────────────────────────────┐
│ 1Password から解決する環境変数                     │
│ NAME=op://vault/item/field（1 行 1 件）            │
│                                                     │
│  1 GH_TOKEN=op://Private/GitHub/token               │
│  2 ·                                                │
│                                                     │
│ Ctrl-S 保存  Enter 改行  Esc 取消                   │
└─────────────────────────────────────────────────────┘
```

- 文字入力・`Enter`（改行）・`Backspace`/`Del`・`←`/`→`/`↑`/`↓`/`Home`/`End`（キャレット移動）で複数行バッファを編集する。
- `Ctrl-S` で**有効な binding だけ**をこのワークスペースのローカル設定（`<workspace>/.usagi/settings.json` の `env`）へ書き込み、
  エディタを閉じて コマンドパレットに戻る。`=` を含まない行・環境変数名として不正な行・reference が空の行は破棄し、同名は後勝ち。
  保存する値は 1Password reference だけで、解決した secret 本体は保存しない。保存に失敗した場合はエラーを結果帯に出す。
- `Esc` は保存せずにエディタを閉じて コマンドパレットに戻る。
- エディタは常に**同じ行数**を確保して描画する（`·` プレースホルダと空行で埋め、カーソル行が見えるようスクロールする）ので、
  binding を増やしてもボックスの高さ・位置は動かない（レイアウトシフトなし）。
- モーダル基盤（`overlay_modal`）を再利用し、パレットの上に中央寄せ・枠付きボックスを合成する。状態は `home/state.rs`
  （`EnvEditor`）、描画は `home/ui/`、キー処理は `home/event/`、binding の解析・整形は `domain/settings.rs`
  （`parse_env_bindings` / `format_env_bindings`）に実装。

## セッションメモの編集

各セッションには、用途・残タスク・リンクなどを書き留める**自由記述の複数行メモ**を持たせられます。
**ルート行（`⌂ root`）も同じメモを持てます**（ワークスペース全体の覚え書き）。
選択で `n`（または `Ctrl-E`）、集中・没入で `Ctrl-E` を押すと、メモの**エディタが右ペインの右上に枠付きボックスで重なります**（overlay。
既存のメモがあれば初期値として読み込まれます）。**全画面モーダルではなく**、サイドバー・タイトル・モードラダー・
フッターも、**背後の右ペイン（プレビュー / 埋め込みターミナル）も見えたまま**なので、画面が切り替わりません
（[選択の読み取り表示](#選択での読み取り表示次回-todo-の確認)と同じ位置・同じ見え方のまま編集に入れます）。
セッションのメモは `state.json` の[`SessionRecord.note`](../../data/02-workspace.md#セッションごとsessionrecord)に、ルートのメモはトップレベルの [`root_note`](../../data/02-workspace.md#トップレベルworkspacestate) に保存され、ブランチ名・識別子には影響しません。

```text
▎ ⌂ root          │ ● login   local  🤖 ☾ ready ┌ note (編集中) ─────┐
▎   workspace root│ ＜背後のプレビュー / 端末＞│ バリデーションを実装中  │
󰤇 ● login         │                          │ ・API は未着手        █│
▎ 🤖 ☾           │                          └────────────────────────┘
▎ ○ feat          │ ＜プレビューはボックスの左・下に見えたまま＞
 [note]  Ctrl-S: save / Esc: cancel / Enter: newline / ←→↑↓: move / Shift+←→↑↓: select
```

- 右ペインの右上に重なる `note` ボックスに複数行のメモ本文。編集中の行にはブロックキャレットが乗り、メモがボックスより縦に長いときはキャレット行が見える窓にスクロールします。ボックスは右ペインの幅・高さを覆い切らない大きさ（幅は最大 48 桁の右寄せ列、左に約 12 桁のプレビュー余白を残す）に抑え、**自分の桁だけを上書き**するので、背後のプレビュー / ターミナルが**ボックスの左と下に見えたまま**（先頭のヘッダー＝セッション識別子も横に並んで読めるので、ボックス自体にはセッション名を出しません）。操作キーはフッターに出ます。
- **編集中は枠とタイトルをアクセント色（`accent`）で描き、タイトルを `note (編集中)` にする**ので、[選択の読み取り表示](#選択での読み取り表示次回-todo-の確認)（プレーンな枠・タイトル `note`）と一目で見分けられます。色を出せない端末（`NO_COLOR` 等）でも `(編集中)` の表示で判別できます。
- 文字入力でキャレット位置に挿入、`Enter` で改行（行を分割）、`Backspace`/`Del` で前後の削除（行頭/行末では行を結合/引き上げ）。
- `←`/`→`/`↑`/`↓` で複数行をまたぐキャレット移動、`Home`/`End` で行頭・行末へ。日本語など複数バイト文字も 1 文字単位で移動・削除します（共通の `widgets::text_area`）。
- **`Shift`+`←`/`→`/`↑`/`↓`（および `Shift`+`Home`/`End`）で選択範囲を伸縮**します。選択中のセルは反転表示され、複数行にまたがる選択では行末の改行も選択帯に含めて見せます。**選択がある状態で `Del`/`Backspace` を押すと範囲ごと削除**し、文字入力・`Enter` は範囲を置き換えます。`Shift` の付かない移動キーは選択を解除します（端末が `Shift`+矢印を送らない場合は選択できませんが、通常の編集には影響しません）。
- `Ctrl-S` で保存して閉じる（空にして保存するとメモを解除）。`Esc` で保存せず閉じる。`Enter` を改行に使うため、保存は `Ctrl-S` に割り当てています。
- **没入から開いた場合**（`Ctrl-E`）は、閉じると（保存・取消どちらでも）元のペインへ再アタッチして埋め込みターミナルに戻ります。**集中から開いた場合**（`Ctrl-E`）は、閉じるとアクション UI（メニュー / プロンプト）の集中に戻ります（再アタッチするペインはありません）。**ルート行でも同じく開けます**（ワークスペースルートのメモを編集）。
- 状態は `home/state.rs`（`NoteEditor`）、描画は `home/ui/`（右ペイン）、キー処理は `home/event/` に実装します。

### 選択での読み取り表示（次回 TODO の確認）

中断したセッションに戻るときは、**選択でカーソルを合わせるだけ**でそのメモが読めます。エディタを開かなくても、
選択の**右ペイン右上にメモのボックス**（`note` 枠）が **overlay** で重なり、選択中セッションの「次回 TODO」が
すぐ目に入ります。行数は[最大 6 行に丸めて](../../06-conventions.md#可読性) `… (N more)` で省略します
（全文は `n` でエディタを開いて確認・編集）。メモが無いセッションでは何も出ません。

ボックスは**右ペインの右上に固定アンカー**で重なるため、背後のプレビューが**アイドル（休むマスコット）**
でもライブ（埋め込みターミナル）でも同じ位置**に出ます。プレビュー自体は常にフル高さで描かれ、ボックスは自分の
右側の桁だけを上書きするので、**カーソルでセッションを移動してもプレビューがずれません（レイアウトシフト＝CLS なし）**。
右寄せの列にすることで、**プレビュー（先頭のヘッダー＝どのセッションを選ぼうとしているかを示す識別子も含む）がボックスの
左に並んで見えたまま**になり、ライブ端末の出力がまだ少ない（情報がスパースな）ときでも、ボックスが画面を覆い尽くしません。

メモの表示は**カーソルに追従**します。`Esc` はメモの表示と無関係に**選択から抜けて元のモードへ戻り**ます（`Esc back`。基底の選択では戻り先がなく無効）。
メモを個別に閉じる操作はありません（カーソルを別のセッションへ動かせばそのセッションのメモに切り替わり、
メモの無いセッションでは何も出ません）。

```text
▎ ⌂ root          │ ● login   local  🤖 ☾ ready ┌ note ───────────────┐
▎   workspace root│ ＜背後のプレビュー / 端末＞│ ・API のエラー処理      │
󰤇 ● login         │                          │ ・#321 の指摘対応       │
▎ 🤖 ☾           │                          └────────────────────────┘
▎ ○ feat          │ ＜ヘッダーはボックスの左に並んで見える＞
```

## 履歴の永続化

実行したコマンドは `<repo>/.usagi/history.json` に 1 件ずつ追記され（`infrastructure/history_store.rs` の
`HistoryStore`）、次回の画面起動時に読み込まれて `history` コマンドと `↑`/`↓` 遡りに反映されます。
`history` は古い順（実行時刻順）に、実行時刻・成功/失敗（`✓` / `✗`）・セッション名・コマンドを併記して表示します。
`history <session>` は指定したセッションの履歴だけを表示します。書き込みはベストエフォートで、失敗しても画面操作は止めません。詳細は
[../data/02-workspace.md](../../data/02-workspace.md#historyjson) を参照。

## 読み込み失敗時の挙動

worktree 状態（`state.json`）の読み込みに失敗した場合は、空一覧の状態でエラー内容をコマンドパレットの結果帯（次に開いたとき）に表示し、
画面操作は継続できます。セッション作成・埋め込みターミナル・Agent 起動の概念は
[4. オーケストレーション](../../04-orchestration.md) を参照してください。

> 描画ロジックは `src/presentation/tui/home/`（画面状態 `state.rs`・描画 `ui.rs`・
> イベントループ `event/`・コマンドレジストリ `command/` に分離）に実装されています。
> worktree 一覧の元データ（`state.json`）の仕様は [../data/02-workspace.md](../../data/02-workspace.md) を参照してください。
