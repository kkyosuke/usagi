## 統合（unite）モードの積み重ね表示

[プロジェクト選択画面（Open）](../02-open.md#統合uniteモードで開く)で**複数のワークスペースをまとめて開く**と、左ペインはワークスペースごとの**グループ**を積み重ねて表示する（統合 / unite モード）。各グループは**ワークスペース名のヘッダ**で始まり、そのワークスペースの `⌂ root` 行と区切り線、セッション行が続く。

```text
▌ frontend                    ← グループヘッダ（ワークスペース名・太字）
  ⌂ root
    ──────────────
  ● main
  + new session               ← このワークスペースの新規作成行


▌ backend                     ← 次のワークスペース（前に 2 行の空き）
  ⌂ root
    ──────────────
  ○ feature
  + new session
```

- **単一ワークスペースのときはヘッダを出さない**（従来とまったく同じ表示）。ヘッダが出るのは 2 つ以上のワークスペースを開いたときだけ。
- **カーソル移動はヘッダ・区切り線・ワークスペース間の 2 行の空きを飛ばす**。`↑↓`/`jk` はグループの境界をまたいで全ワークスペースのルート行・セッション行・[新規セッション行](#新規セッション行-new-session)を順にたどる。
- **各ワークスペースに `⌂ root` 行と `+ new session` 行がある**。新規セッション作成（`c` ／ `session create`）はカーソルがいるグループのワークスペースに作られ、`close` ／ `session remove` ／ 表示名変更（`r`）／メモ編集は対象セッションが属するワークスペースに対して行う。ワークスペース単位のコマンドもカーソルのグループに解決される: `config`（`env` エディタ）はカーソルグループのワークスペースの `settings.json` を読み書きし、`issue` はそのワークスペースの issue ストアを一覧・表示する。root 行の `terminal` ／ `agent` もそのグループのワークスペースルートで起動する。
- **いまスコープになっているワークスペースをフッターとコマンドパレットに出す**。統合中はフッターのモードタグにカーソルグループのワークスペース名を添え（例 `[switch · backend]` ／ `[session: main · backend]` ／ `[command · backend]`）、`:` のパレット入力行の先頭にも `[backend]` の淡色タグを出す。これで `c` ／ `r` ／ `config` ／ `issue` がどのワークスペースに効くのかが一目で分かる（単一ワークスペースではタグは出さない）。
- **ワークスペースを折りたたむ（`Space`）**。root 行にカーソルがある状態で `Space` を押すと、そのワークスペースはセッション行・`+ new session` 行を隠し、**1 行のヘッダ `▸ <名前>  (N)`**（`N` はセッション数）だけに畳まれる。もう一度 `Space` で開く。ワークスペースが多いときに一覧を俯瞰しやすくするための表示切り替えで、折りたたみ状態はセッション中は保たれる（バックグラウンドの再同期を跨いでも維持され、usagi の再起動でリセットされる）。単一ワークスペースでは全体が消えてしまうため無効。折りたたんだワークスペースへ `c` で新規作成すると、作成のために自動で開く。

  ```text
  ▸ frontend  (2)               ← 折りたたんだワークスペース（ヘッダ 1 行のみ）

  ▌ backend
    ⌂ root
      ──────────────
    ○ feature
    + new session
  ```

- タイトルバーは統合中 `unite · ▸ <active> · N sessions across M workspaces` の形で、開いているワークスペース数と総セッション数を示す（折りたたんでもこの数は変わらない）。
- レール（畳んだサイドバー）でもグループは積み重なり、ワークスペースの境目にはフル幅サイドバーと同じ 2 行の空きを入れる（幅 5 桁に名前は入らないため）。折りたたんだワークスペースはレールでも 1 行の `▸` マーカーに畳まれる。
- **マウス操作**: 統合中もセッション行のクリック選択と PR バッジのクリック（ポップアップ）の両方が使える。座標→行の対応・ポップアップの配置はワークスペースごとのヘッダと境界の空きを織り込んで計算するので、どのグループのバッジでも正しいセッションのポップアップが開く。

## 新規セッション行（"+ new session"）

各ワークスペースのセッション一覧の**末尾**には、常時 **`+ new session`** 行が並ぶ。セッションが 1 つも無いときも表示され、新規作成の入り口が常に見える。統合（unite）モードでは**ワークスペースごとに**この行を持つので、どのワークスペースへ作るかがひと目でわかる（単一ワークスペースでは従来どおり一覧の一番下に 1 行）。

```text
  ⌂ root
    ──────────────
  ● main
  + new session               ← このワークスペースの末尾に常駐する新規作成行
```

- **カーソルが乗る行**である。`↑↓`/`jk` でそのワークスペースの最後のセッションの次に選択でき、`↑` でルート行から一巡すると最下部の行に着く。
- **フォーカス中にそのまま入力すると作成に入る**。行を選択した状態で印字可能な文字を打つと[インライン名入力](04-keys.md#切替のキー操作switch--既定)が開き、その文字が名前の 1 文字目になる。`Enter`（または `Ctrl-A`）は空の入力欄を開く。開いたあとは通常の作成と同じで `Enter` で作成・`Esc` で取り消し。
- **クリックでも入力モードに入る**。切替・在席では行を左クリックすると（セッション行のようなダブルクリックは不要で）その場でインライン名入力が開く。在席でクリックした場合は切替へ戻ってから入力欄を開く。没入ではサイドバー行の確定操作に合わせ、この行をダブルクリックすると切替へ戻って入力欄を開く。
- 入力欄が開いている間は、その行が入力欄そのものに変わる（`+ new: <名前>`）。作成先はカーソルがいるワークスペース（統合モードでもカーソルのグループ）。
- レール（畳んだサイドバー）でも同じ位置に `+` グリフだけを残し、クリック／フォーカスの的になる（入力欄自体は幅が無いため右ペインに出る）。

## 一覧が縦に溢れるとき（スクロール）

セッションが増えて一覧が左ペインの高さに収まらなくなると、サイドバーは**選択中の行が常に見えるように縦スクロール**します。カーソル（`↑↓`/`jk` や `Ctrl-O` のピッカー、クリック）が動くたび、その行が画面外に出ないぶんだけ表示位置をずらします。

- **上詰めが基本**: 選択行が上のほうにある間はスクロールせず、先頭（ルート行）から表示します。選択行の下端が画面の折り返しを超えたときだけ、その行が最下部に収まるところまで送ります。行を上へ戻すと巻き戻ります。
- **末尾で止まる**: 一番下（最下部の `+ new session` 行）を選んだときは一覧の末尾までスクロールし、それ以上は送りません（末尾より先の空白は見せません）。
- **クリック・PR バッジも追従**: スクロール中でも、セッション行のクリック選択と PR バッジのクリックは画面上の見た目どおりの行に当たります。スクロールで画面外へ流れたセッションの PR ポップアップは開きません（バッジが描かれていないため）。
- **レール（畳んだサイドバー）でも同じ**: フル幅と同じ行構成のまま、選択中エントリが見えるようにスクロールします。
- **統合（unite）モードでも同じ**: ワークスペースヘッダ・区切り線・ワークスペース間の空きを織り込んだうえで、選択中の行を画面内に保ちます。

なお一覧がスクロールするほど埋まっているときは、左ペイン最下部の[マスコット](02-layout.md#レイアウト)は場所を譲って表示しません（一覧が短く下端に余白があるときだけ憩います）。

## 直前のセッションへ切り替え（Ctrl-^）

**`Ctrl-^`** は**直前にフォーカスしていたセッション**へ一足飛びに移ります（vim の `Ctrl-^` / tmux の `last-window` と同じ発想）。2 つのセッションを行き来するとき、いちいち**切替**を開いてカーソルを動かさずに往復できます。

- どのモード（統括・切替・在席・没入）でも有効です。没入中はシェルへ送らず横取りします（[予約キー](04-keys.md#没入のキー操作attached--terminal--agent-実行中)）。
- 飛んだ先がライブなら**没入**で再アタッチ、そうでなければ**在席**に着地します（`Enter` でセッションを開くときと同じ判定）。
- 切り替えるたびに「いま離れるセッション」が次の直前として記録されるため、もう一度 `Ctrl-^` を押すと元に戻ります（トグル）。
- フォーカスを一度も移していない（直前が無い）ときや、直前のセッションが削除済みのときは何もしません。直前の記録はセッション名で保持するので、バックグラウンドの再同期でリストが組み直されても飛び先を見失いません。

## セッション一覧の同期（外部変更の反映）

サイドバーのセッション一覧は、**この画面の外で行われたセッションの作成・削除も自動で反映**します。別のセッションのペインで動く Agent が MCP（`session_create` / `session_remove`）で作った・消したセッション、別ウィンドウの usagi、CLI からの操作は、いずれも `state.json` を書き換えるだけでこの画面へ通知は届きませんが、バックグラウンドの監視が `state.json` の更新を検知して一覧を読み直し、**約 1 秒以内に**サイドバーへ反映します（キー入力やペイン離脱を待ちません）。

- 監視は `state.json` の最終更新時刻を定期的に見て、変化したときだけ記録を読み直します。何も変わっていなければ読み直さないため、静止した画面のコストはごくわずかです。
- 読み直しは記録（`state.json`）からのみ行い、git status の再計測（コミット・push・マージの反映）は伴いません。**新しく現れたセッションの git status は、次の本格的な再同期（ペイン離脱・再起動）で埋まります**。それまでは記録済みの状態で表示します。
- カーソル・アクティブ行はセッション名で保持するので、一覧が組み直されても選択位置は動きません（削除済みセッションを指していたときはルート行へ戻ります）。

自分のこの画面から作成・削除したセッションは、その操作のバックグラウンド処理が完了した時点で（同じ経路の完了通知として）反映されるため、この監視を待つ必要はありません。

## サイドバーの開閉（Ctrl-B）

左のセッション一覧は**フル幅**と**レール**の 2 状態を切り替えられます。**切替・在席では `Ctrl-B`**（画面共通のトグル。在席は既定の `prefix` 方式なら `Ctrl-O s` でも開閉できる）で、**没入では[キー方式](04-keys.md#没入のキー操作attached--terminal--agent-実行中)**に従い `Ctrl-O s`（`alt` 方式は `Alt-s`）で切り替えます（没入では bare `Ctrl-B` はシェルへ流れる）。開く初期状態は設定 `sidebar`（`full` / `rail`）で決まり、これらはそこからの実行時トグルです（[設定項目](../../05-settings.md#設定項目)）。畳むと右ペイン（特に没入のターミナル）が即座に再レイアウトされ広がります。

- **フル幅（`full`）** — 各セッションを名前・Agent 状態・リソース使用量まで 3 行で並べる既定の一覧（幅は端末幅の約 1/3、16〜40 桁にクランプ）。git のブランチ状態（`dirty` / `pushed` / `synced` など）はサイドバーには出さず、[在席・没入の見出し](02-layout.md#在席focus)（名前 ＋ status ＋ Agent）で確認する。
- **レール（`rail`）** — 一覧を幅 5 桁の縦ストリップに畳み、**右ペイン（特に没入のターミナル）へ幅を譲ります**。**各エントリはフル幅と同じ 3 行構成のまま**で、ガターの右にグリフを置きます。

  ```text
  ▎ <鮮度>                  1 行目: 鮮度ドット ⌂/●/◐/○
  ▎ <label> <agent status>  2 行目: ステータスラベルグリフ ＋ Agent 状態 ▶/◆/☾/✓（無ければ空欄。レールは幅 1 桁の phase グリフのみで AI マークは付けない）
  ▎                         3 行目: 空欄（フル幅のリソース使用量行に対応。レールには数字を出す幅がない）
  ```

  **左端のアクティブバー `▎`（緑）**が 3 行すべてに走ります。選択中のセッションは左端に `󰤇` / `▎` / `▎` を出し、切替中は赤、サイドメニューで選択した後は緑で残ります。名前は出ませんが「**どれがアクティブで・Agent が何をしているか**」は残ります。

  **行数はフル幅と完全に一致する**（ルート 2 行＋区切り線＋セッションごと 3 行）ため、`Ctrl-B` で開閉しても各セッションの縦位置はずれず（レイアウトシフトなし）、変わるのは幅だけです。

畳んでもアクティブセッションを見失わないよう、**タイトルバーに常にアクティブ名を表示**します（`usagi · ▸ <active> · N sessions`、`▸` がアクティブ印。ルート行がアクティブのときはワークスペース名）。アクティブ名の欄は**端末幅から決まる固定幅**（長い名前はクリップ、短い名前はパディング）なので、ラベル全体の幅はアクティブセッションが変わっても一定です。したがって**中央寄せのタイトルは名前の長さで左右にずれません**（レイアウトシフトなし）。

**切替（Switch、既定）も `Ctrl-B` のトグルを尊重**します。レールに畳んでもピッカーはそのまま使え（選択中セッションの `󰤇` / `▎` / `▎` と切替中の非選択行の淡色はレール上にも出ます）、右ペインのプレビューはレール幅に合わせて広がります。ただしレール（5 桁）は名前入力には狭すぎるため、新規作成 `c` / 表示名編集 `r` の入力欄は**左ペインのインライン行ではなく右ペイン**に出します。フル幅のときは左ペインでインライン編集しますが、その出方は 2 つで異なります。新規作成は末尾の `+ new:` 行（作成先ワークスペースのブロック内）を入力欄にする一方、**表示名編集は選択中セッションの名前行そのものをその場で編集フィールドに変える**（選択マーカー `󰤇` と鮮度ドットはそのまま、名前欄がキャレット付きの入力になり、編集中は右端のメモ欄を隠して名前に幅を回す）ため、リネームのために一覧の下端へ移動する必要はありません。統合（unite）で複数 workspace を表示している場合、フル幅の `+ new:` はカーソルがある workspace のブロック内に出ます。後続の workspace があるブロックでは、workspace 間の固定 2 行ギャップを入力欄に使うため、入力欄を開いても下の workspace の縦位置はずれません。没入の埋め込みターミナルもレールに合わせて即リサイズされるので、畳むとそのぶん広く使えます。

![サイドバーの開閉（フル ⇄ レール）](../assets/sidebar-toggle.png)

- **切替（既定）** — キーボードが左ペインへ移り、一覧をカーソルで選択／`c` でインライン作成／`r` でカーソル中セッションの表示名を**選択行の名前欄でその場でインライン編集**します（カーソル以外の行は淡色に落として選択中を際立たせます）。右ペインには、そのセッションを選んだときに開く画面のプレビューが出ます。
- **在席** — 右ペインに**アクション UI**（[Menu / Prompt](02-layout.md#在席のアクション-uimenu--prompt)）が出て、そのセッションの `agent` / `close` / `terminal` を操作します。
- **没入** — 右ペインが**埋め込みターミナル**（ライブシェル / Agent）になります。

ワークスペース全体のコマンド（`session` / `config` など）は、これらのモードに重なる
[コマンドパレット（`:` で開く）](01-modes.md#コマンドパレット統括overview)で実行します。

## 桁幅の数え方（East Asian 幅）

セッション名・ステータスラベルの桁確保は、**端末が実際に描く表示幅**で数えます。East Asian **Ambiguous** 幅の文字（`→ ① ※ ★` など）は日本語端末で **2 桁**に描かれるため 2 桁ぶんを確保し、名前がそれらを含んでも続くラベル欄・1 行目右端のメモ欄がずれません（名前が桁に収まらなければ末尾を `…` でクリップします）。一方、サイドバーの UI グリフ（`● ◐ ○ ◇ ◆ ▶` などの幾何学記号や Nerd Font アイコン）は端末で **1 桁**に描かれるため 1 桁として数えます。この数え方はフル幅の各行・切替プレビューと没入/在席の見出し（名前 ＋ status ＋ Agent）に共通で、右ペインのタブ帯もこの見出し幅を基準に並びます。

## 更新時刻

2 行目の**右端（差分バッジの左隣）**に、そのセッションが**最後に更新されてからの経過**を相対表記で淡色表示します。
どのセッションが新しく、どれが放置されているかをひと目で見分けるための目印です。

- **表記**: 1 分未満は `now`、以降は `Nmin ago` / `Nh ago` / `Nd ago`。時計のずれで未来時刻になった場合は `now` に丸めます。
- **基準時刻**: そのセッションが跨ぐ全リポジトリの worktree の **`updated_at` の最大値**（最も新しく触れたリポジトリ）。worktree が無いセッションは作成時刻にフォールバックします。
- **更新の刻み**: 画面を描き直したフレームの時刻で計算します。秒刻みでは描き直さない（再描画コストを抑える）ため、ラベルは何か変化があったとき・定期的なセッション再同期のときに更新されます。
- **狭いとき**: 2 行目右端のまとまりの優先順は **差分バッジ > ahead/behind マーカー > 更新時刻**。Agent 状態の表示が削られるほど狭い幅では、更新時刻を**先に省き**、次いで ahead/behind を省きます（バッジと Agent 状態を優先）。リソース使用量は 2 行目のクラスタではなく独立した 3 行目なので、狭いときはこの行ごと末尾を省略（クリップ）して収めます。
- **レール（畳んだサイドバー）には出しません**（幅 5 桁に収まらないため）。

## ahead/behind マーカー

2 行目の**右端（差分バッジの左隣）**に、そのセッションのブランチが**既定ブランチからコミット単位でどれだけ離れているか**を
`↑N ↓M` で示します。**`↑N`（ahead、シアン）はブランチにあって既定ブランチに無いコミット数**、**`↓M`（behind、マゼンタ）は
既定ブランチにあってブランチに無いコミット数**です。差分バッジが「行数」を示すのに対し、こちらは「コミット数」を示し、
**未マージの作業（ahead）**と**既定ブランチに対して古くなっている度合い（behind）**をひと目で見分けます。

- **片側のみ表示**: 0 の側は出しません（ahead だけなら `↑N`、behind だけなら `↓M`、両方あれば `↑N ↓M`）。ただし**桁の確保は描画単位**で、表示中のどこかのセッションがその側を使っていれば、0 の側もその桁ぶんを空白で確保して `↑`/`↓` の列を揃えます（どのセッションもその側を使っていなければ、その側の桁は丸ごと詰めます）。
- **数え方**: `git rev-list --left-right --count` で既定ブランチ（`origin/<default>` を優先、無ければローカル）との差を数えます。status の判定（`new` / `synced` など）と同じ計測を共有します。
- **集約**: セッションが複数リポジトリに跨るときは、全リポジトリの ahead / behind を**合算**して 1 つ表示します。
- **出ない条件**: 既定ブランチそのもの・detached HEAD・既定ブランチと差のない（ahead も behind も 0 の）セッション・計測できなかったとき。
- **狭いとき**: 差分バッジ・Agent 状態が優先で、収まらなければ（更新時刻の次に）省かれます。
- **レール（畳んだサイドバー）には出しません**。

## リソース使用量

セッションの **3 行目**（2 行目の Agent 行の下、名前の真下にインデント）に、そのセッションで動いているプロセス群が
使っている **CPU とメモリ**を、**CPU アイコン + 数値**・**メモリアイコン + 数値**（` 8%   120MB` のように）の形で淡色表示します。どのセッションが実際に CPU・メモリを食っているかを
ひと目で見分けるための目印です。2 行目の固定桁クラスタ（更新時刻・↑↓・差分）は詰めずに、増えてきた情報をこの専用行が引き受けます。

- **対象**: そのセッションで生きているシェル（と、その配下の agent CLI など）の**プロセスツリー全体**を合算した値です。`usagi` は agent を `setsid` でセッションリーダーとして起こすため、シェルの PID を起点に子孫プロセスまでたどって集計します。
- **表記**: 先頭に **CPU（マイクロチップ `nf-fa-microchip`）・メモリ（サーバー `nf-fa-server`）の Nerd Font アイコン**を置き、`CPU` / `MEM` の語は出しません（git status と同じ流儀。要 Nerd Font、未対応フォントでも数値で判読可）。メモリは Font Awesome 4 系の `nf-fa-server` を使います（git status のアイコンと同じ範囲で、古い／一部の Nerd Font でも `?` にならず表示できるため。FA5 の `nf-fa-memory` は古いフォントに含まれずフォールバックの `?` になります）。CPU は CPU 1 コアぶんを 100% とした整数値（マルチコアでは合算で 100% を超えることがあり、`top` と同じ読み方）、メモリは 1 GiB 未満は整数 MB（`120MB`）、以上は小数 1 桁の GB（`1.2GB`）。CPU 値は**固定幅で右に詰めて揃える**ので、桁数が変わっても（`0%` → `100%`）後続のメモリ値が同じ桁に並び、表示がガタつきません。
- **出る条件**: **全セッションで常に**出します。計測値が無い（生きたプロセスを持たない・まだ計測前の）セッションも、行を畳まず `0%` / `0MB` と表示します。これによりセッションがライブになったり止まったりしても一覧の縦位置がずれません。
- **更新の刻み**: バックグラウンドの監視スレッドが**約 1 秒ごと**に計測します（CPU は前回計測からの差分で求めるため、瞬間値ではなく直近 1 秒の平均です）。生きたセッションが 1 つも無いときは計測自体を行わないので、**何も動いていない切替の画面は静止して CPU を消費しません**。
- **合計**: 全セッションぶんを合算した**ワークスペース合計**を、左ペイン最下部の[マスコット](02-layout.md#レイアウト)の**足の行の右隣**（うさぎの足のライン）に、同じアイコン + 数値（` 23%   512MB` のように）で表示します。
- **負荷帯と色分け**: マスコット脇のワークスペース合計では、**CPU の数値とメモリの数値をそれぞれの負荷帯で色分け**します（軽い＝淡色 / 中＝黄 / 高＝赤）。CPU は **30% 以上で中・120% 以上で高**（全プロセスツリー・全コアの合算値に対する判定）、メモリは **512 MiB 以上で中・2 GiB 以上で高**。CPU の負荷帯は[マスコット](02-layout.md#レイアウト)の表情・動き（中負荷でこわばり、高負荷で力む・汗・震え）も決めます。なお**セッション 3 行目の使用量は常に淡色**で、色分けはマスコット脇の合計だけに掛かります。
- **レール（畳んだサイドバー）には出しません**（幅 5 桁に収まらないため）。

## ステータスラベルカラム

1 行目の**名前の右隣**に、そのセッションにユーザーが手で付けた**ステータスラベル**（[`session_labels`](../../05-settings.md#ステータスラベルsession_labels) マスタから選んだ 1 件）を `<グリフ> <名前>` で色付き表示します。git 由来のブランチ状態（[在席・没入の見出し](02-layout.md#在席focus)に出る status）や Agent 状態とは**別レイヤーの、人が付ける意味づけ**（todo / レビュー中 …）で、切替（Switch）の `Tab` / 数字キーで付け外しします（[04-keys.md](04-keys.md#切替のキー操作switch--既定)）。

- **付け方**: 切替で `Tab` / `Shift`+`Tab` が候補を巡回、`1`〜`9` が直接指定、`0` が解除（[キー操作](04-keys.md#切替のキー操作switch--既定)）。
- **色**: ラベル定義の `color`（`gray` / `red` / `green` / `yellow` / `blue` / `magenta` / `cyan`）を[テーマパレット](../../06-conventions.md#技術スタック)に解決して塗ります。既定は淡色（`gray`）。
- **桁の固定**: カラム幅は表示中で最も広いラベルに合わせて確保し、ラベルの無い行（ルート行を含む）は同じ幅の空白で埋めるので、メモ欄がずれません。長い名前は幅上限（12 桁）でクリップします。
- **出ない条件**: どのセッションもラベルを持たない、またはマスタが空のときはカラムごと消え、従来どおりの表示になります。
- **レール（畳んだサイドバー）**: 2 行目のグリフ位置に、ラベルの**色付き 1 文字グリフだけ**を出します（名前は幅の都合で省略）。

保存先は [data/02-workspace.md](../../data/02-workspace.md) の `state.json`（セッションの `label_id`）、マスタの定義は [05-settings.md](../../05-settings.md#ステータスラベルsession_labels) が正本です。

## メモマーカー

1 行目の**右端**（旧アクティブ印の跡で、いまは空いている桁）に、その行が
**メモ（note）を持つときだけ**付箋グリフ（`nf-fa-sticky_note`、黄色）を出します。どの行に書き置きが
あるかをひと目で見分けるための目印で、メモ本文は出しません（本文は切替プレビューの浮動ボックスや
`n` / `Ctrl-E` のエディタで読み書きします）。

- **出る条件**: その行のメモが空でないとき。**ルート行（`⌂ root`）もメモを持てる**ので、ルートのメモが空でなければ同じマーカーが出ます（ルートのメモは `state.json` のトップレベル `root_note`、セッションのメモは各 `note`）。
- **桁の固定**: マーカーの有無で行がずれないよう、マーカーの桁はメモが無くても同じ幅を確保します。
- **アイコン**: [Nerd Font](https://www.nerdfonts.com/) を入れた端末で表示されます（未対応フォントでは代替の枠が出ます）。
- **レール（畳んだサイドバー）には出しません**（幅 5 桁にメモ桁の余裕がないため）。

## 差分バッジ

2 行目の**右端**に、そのセッションが**既定ブランチ（`main` など）からどれだけ変更したか**を `+N -M`（追加行
**緑**・削除行**赤**）で示します。どのセッションが進んでいて、どれが手つかずかをひと目で見分けるための目印で、
Agent 状態（同じ行の左寄せ）とは独立して出ます——Agent が動いていないセッションでもバッジは表示されます。

- **数え方**: 各リポジトリの worktree について、**既定ブランチとの[マージベース](https://git-scm.com/docs/git-merge-base)から作業ツリーまで**の差分行数（`git diff --numstat`）。コミット済みと未コミットの両方を含み、既定ブランチがあとから進んだぶんは数えません（純粋にそのセッションが加えた変更だけ）。バイナリファイルは行数に数えません。git の追跡対象外（未 `add`）の新規ファイルは差分に含まれません（その点は 1 行目の `dirty` 表示が補います）。
- **集約**: セッションが複数リポジトリに跨るときは、全リポジトリの `+N` / `-M` を**合算**して 1 つ表示します。
- **出ない条件**: 既定ブランチそのもの・detached HEAD・既定ブランチと差のない（手つかずの）セッション・差分が読めなかったとき。`+0 -0` は表示しません。
- **桁揃え**: `+N` / `-M` の数値はそれぞれ固定幅（表示中の最大桁数）に右詰めするので、変更行数が行ごとに桁違いでも `+` と `-` がどのセッションでも同じ列に並びます。
- **狭いとき**: 右端のまとまりが収まらない幅では、**差分バッジと PR バッジ（[後述](#pr-バッジ)）を最優先**で右端に残します。次いで Agent 状態、ahead/behind、最後に更新時刻の順で省きます（省くかどうかは表示中で最も長い Agent ラベルを基準に行を揃えて判定するので、どの行も同じ列構成になります）。
- **レール（畳んだサイドバー）には出しません**（幅 5 桁に `+N -M` は収まらないため）。

数え方・集約規則は [02-architecture.md](../../02-architecture.md) 経由のコード（`infrastructure/git` の `diff_stat` / `ahead_behind`、`domain/workspace_state` の `DiffStat` / `AheadBehind`）が実装の所在で、永続化先は [data/02-workspace.md](../../data/02-workspace.md) の `state.json`（worktree の `diff` / `ahead_behind`）です。

## PR バッジ

2 行目の**右端（差分バッジの右隣）**に、そのセッションに紐づく **Pull Request** を **PR アイコン（nerd font）＋件数**（リンク色の淡い青・下線、リンクであることを示す）で 1 つにまとめて表示します（例: PR が 4 本なら ` 4`、1 本でも ` 1`）。番号を `#442 #447 #451 #452` のように全部並べると 2 行目が PR で埋まり、Agent 状態（`🤖 ▶` など）がクリップされてしまうため、**行には件数だけ**を示し、**個々の番号はバッジをクリックして開くポップアップで見せます**（下記）。

- **取得元**: usagi は GitHub に問い合わせません。エージェントやシェルが出力した PR の URL（`https://<host>/<owner>/<repo>/pull/<N>` 形。GitHub / GitHub Enterprise）を、ターミナル出力のリンク検出と同じ走査で拾い、`<N>` を番号として取り出します。**画面にあるものを全部**（URL 単位で重複排除して）集めるので、複数リポジトリに PR を開いたセッションも 1 つの件数に合算します。
- **即時反映（切替・在席の両方で全セッション）**: 没入中のペインだけでなく、サイドバーの裏で生きている全ペインも監視対象です。PR URL が出力されると、そのセッションへ再アタッチしたり次回の git 再同期を待ったりせず `#N` バッジへ反映します。反映は「今開いているセッションか」やモードに依存せず、**切替・在席のどちらでも、開いていないセッションの分も**出ます。一覧を組み直す各経路（初回描画・`state.json` 更新検知の約 1 秒ごとの読み直し・git 再同期）が、そのつど保存先の PR を各セッションへ読み込むため、モードを切り替えてもバッジは消えません。
- **永続化と単一情報源**: 拾った PR は worktree をキーにした保存先（`pr-links/`）に **URL 単位で重複排除しつつ蓄積**します。この保存先が PR の正本で、一覧を組み直す各経路はここを **`state.json` の worktree の `pr`（配列）へ追記マージ**して反映します（保存先が空なら `state.json` の値をそのまま残す＝バッジを消さない）。git 再同期はマージ結果を `state.json` に書き戻します。エージェントが各 URL を出すのは一度きりでも、**バッジは再起動後も出続けます**。
- **クリックでポップアップを開く**: フル幅サイドバーで**バッジ（アイコン＋件数）をクリック**すると、`#442 #447 …` の**番号一覧**を小さな枠（タイトル `PR`）で右ペイン寄りに**ピン留め**表示します。番号が多いときは枠の中で折り返します。ホバーの吹き出しと違い**マウスを動かしても閉じない**ので、ポインタを枠の上へ運んで番号をクリックできます。**切替・在席・没入のどのモードでも**開きます（バッジを描くのはフル幅サイドバーのときだけなので、レールでは開きません）。クリックの検知には**マウスレポート**が要るため、それを報告しない端末（Apple Terminal.app など）では出ません。
- **番号クリックで開く**: ポップアップ内の **`#<番号>` をクリック**すると、その 1 件の PR を既定ブラウザで開きます（エージェント返信内の URL クリックと同じブラウザ起動経路を共有）。**枠の外をクリックするか、キーボードを触る（`Esc` など）と閉じます**。枠の中で番号以外を押しても閉じません。**役割分担**: 番号一覧を出すのは**バッジのクリック**、その 1 件を開くのは**番号のクリック**です。バッジ以外（行の余白・Agent ラベル側）をクリックしても何も起きず、行の**ダブルクリック**はそのセッションへの切り替え（[没入のキー操作](04-keys.md#没入のキー操作attached--terminal--agent-実行中)）に割り当てられています。
- **集約**: セッションが複数リポジトリに跨るときは、全 worktree の PR を**順に集めて重複 URL を除いた**件数・一覧にします。
- **桁揃え・隙間の確保（CLS なし）**: アイコン＋件数のバッジは固定幅（アイコン＋スペース＋件数の最大桁数）の桁に収め、桁幅は描画ごとに表示中セッションの最大値で決めるので、件数が行ごとに違っても揃って並びます。**どのセッションにも PR が無いときも PR 列は最小幅（アイコン＋スペース＋1 桁）を空けたまま**確保するので、あるセッションが最後の PR を得たり失ったりしても、**左隣の差分バッジ（`+N -M`）や更新時刻が右へずれません**（[共通の設計メモ](../README.md)の CLS 原則）。
- **出ない条件**: PR の URL をまだ一度も観測していないセッション。ルート行（ワークスペース自体）にも出ません。
- **狭いとき**: 差分バッジと並んで**右端に最優先で残します**（アイコン＋件数に畳んだので、複数 PR でも 3 桁前後で収まり、更新時刻・ahead/behind より先に Agent 状態が削られても PR は残ります）。
- **レール（畳んだサイドバー）には出しません**（幅 5 桁にバッジの桁がないため）。

実装の所在は [02-architecture.md](../../02-architecture.md) 経由のコード（リンク検出 `presentation/tui/home/terminal/link` の `pr_links` / `parse_pr_url`、保存 `infrastructure/pr_link_store`、型 `domain/workspace_state` の `PrLink`、行のバッジ・バッジのヒットテスト・ピン留めポップアップの配置／クリック判定は `presentation/tui/home/ui` の `panes`）で、永続化先は [data/02-workspace.md](../../data/02-workspace.md) の `state.json`（worktree の `pr`）です。

git のブランチ状態（status）はサイドバーの行には出さず、[在席・没入の見出し](02-layout.md#在席focus)（名前 ＋ status ＋ Agent）にだけ `<アイコン> <語>`（`local` / `pushed` / `synced`）で出します。status はセッションが跨ぐ全リポジトリのブランチ状態を**最も未完のもの**へ集約します（`new` < `dirty` < `local` < `pushed` < `synced`）。
したがって `synced` は**全リポジトリのブランチが既定ブランチに取り込み済み**のときだけ表示され、1 つでも未コミット／未マージ／未 push のリポジトリがあれば控えめな状態が出ます。単一リポジトリのワークスペースでは
そのリポジトリの状態がそのまま出ます。各状態の判定根拠（ahead/behind での `new` と `synced` の切り分け）と**再計算のタイミング**は
[../data/02-workspace.md](../../data/02-workspace.md#status-ブランチのライフサイクル状態) を参照。

左ペインの先頭には、どのセッションにも属さない **ルート行（`⌂ root`）** が常設されます。ワークスペースを
開いた直後はこのルート行がアクティブ（左端に緑の `▎`）で、ここで `terminal` / `agent` を実行すると
セッションの worktree ではなく**ワークスペースルート**でシェル / Agent が起動します。
ルート行の直下には常に**区切り線**（`─`、淡色）を引き、ワークスペースルートとその下のセッション群を分けます。
worktree の記録が無い（`state.json` 未生成など）場合は区切り線の下に `no sessions`（淡色）を表示します。
区切り線・`no sessions` はどちらも `root` の文字位置に揃えてインデントします。

```text
┌─────────────────────────┐
│  ▎ ⌂   root             │  ← ルート行（1 行目：左端ガターにアクティブ ▎）
│  ▎     workspace root   │  ← ルート行（2 行目：ガターバーは 2 行にわたる）
│        ───────────────  │  ← 区切り線（root の下、淡色。常設）
│     ○  feat/login       │  ← 以下、各セッション（区切り線の下に並ぶ）
│        🤖 ◆            │  ← 2 行目: AI マーク（ロボットグリフ）＋ Agent 状態（アイコンのみ）
└─────────────────────────┘
```

> のように**アイコンのみ**で表示します（`ready` / `running` / `waiting` / `done` の語は添えません。`🤖` は説明用の代替表記で、実際は Nerd Font のロボットグリフ）。レール（幅 5 桁）は phase グリフのみで AI マークは付けません。語つきの Agent ラベル（`🤖 ◆ waiting` など）は[在席・没入の見出し](02-layout.md#在席focus)に出ます。
