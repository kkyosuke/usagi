# 2. プロジェクト選択画面（Open）

> [画面設計トップ](README.md) ｜ ← 前へ [1. 起動画面（Welcome）](01-welcome.md) ｜ 次へ → [3. 新規プロジェクト画面（New）](03-new.md)

起動画面で `Open`（`o` / Enter）を選ぶと遷移する、登録済みワークスペースの一覧から
開く対象を選ぶ画面。一覧は `workspaces.json` 由来で、ワークスペース名のアルファベット順
（大文字小文字を区別しない）に並びます。起動画面と同じ代替スクリーン上に描画されます。

> ワークスペースを選択するとホーム画面（[5. ホーム画面](home/README.md)）へ遷移し、そのワークスペースの
> worktree 一覧を表示します。ホーム画面で `Esc` を押すとこのプロジェクト選択画面へ戻ります。

## レイアウト

リスト全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。
タイトル直下に検索バー（フィルター）を常時表示し、通常文字を入力するとフォーカス操作なしで、名前に
入力文字列を含むワークスペースだけに一覧を絞り込みます。各ワークスペースは 2 行で表し、
1 行目に「カーソル + 名前 + パス」、2 行目に統計（セッション数・未完了 issue 数・最終利用からの
経過時間）をアイコン付き・淡色で表示します。

うさぎ AA は[起動画面（Welcome）と共通の行](01-welcome.md#レイアウト)に置くため、画面を行き来しても
位置がガタつきません（レイアウトシフトなし）。

```text
┌───────────────────────────────────────────────────────┐
│                       (\(\                            │  ┐ うさぎ AA（マゼンタ・太字）
│                       (='-')                          │  │
│                      o(_(")(")                        │  ┘
│                    Open Project                       │  ← タイトル（緑・太字）
│                    Single │ Unite                     │  ← タブ。アクティブ側はシアン太字
│              Choose one registered workspace          │  ← サブタイトル（淡色）
│                                                       │
│      Filter: type to filter                           │  ← 検索バー。type で入力
│                                                       │
│      > my-project    /path/to/my-project              │  ┐ プロジェクト一覧（1 件＝2 行）
│          ⎇ 3 sessions   ● 5 open   ◷ 2d ago           │  │  1 行目：選択行は赤 ">" + 名前シアン太字
│        other-repo    /path/to/other-repo              │  │         パスは淡色（長いと先頭を … で省略）
│          ⎇ 0 sessions   ● 12 open  ◷ 3w ago           │  ┘  2 行目：統計行（淡色・アイコン付き）
│                                                       │  ← 通知行（任意・黄色／読み込み失敗時など）
│                                                       │
│ ←→ tab / ↑↓ move / type filter / Enter open / …      │  ← フッター（淡色）
└───────────────────────────────────────────────────────┘
```

ワークスペースが 1 つも登録されていない場合は、一覧の代わりに
`No workspaces yet. Choose "New" to add one.`（淡色）を中央に表示します。
検索バーの入力に一致するワークスペースが無い場合は、一覧の代わりに
`No projects match the filter.`（淡色）を中央に表示します。

統計行のアイコンは、端末の文字フォントで描画される単色グリフ（絵文字ではない）です。

| アイコン | 意味 | 値の出所 |
|---|---|---|
| `⎇` | セッション数（セッションごとの worktree） | `state.json`（[../data/02-workspace.md](../data/02-workspace.md)） |
| `●` | 未完了（`done` 以外）の issue 数 | `.usagi/issues/`（[../data/03-issues.md](../data/03-issues.md)） |
| `◷` | 最終利用からの経過時間。`just now` / `5min ago` / `3h ago` / `2d ago` / `3w ago`、1 か月を超えると `YYYY-MM-DD` の絶対日付 | `workspaces.json` の `updated_at`（[../data/01-global.md](../data/01-global.md)） |

各ワークスペースのパスが見つからない・読めない場合でも、その行の統計は 0 として表示し、一覧全体は描画を続けます。

## 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| タイトル | `Open Project` | 緑・太字 |
| タブ | `Single │ Unite`。`←` / `→` で切り替える。現在のタブをシアン太字、もう片方を淡色で表示する | アクティブ：シアン・太字。非アクティブ：淡色 |
| サブタイトル | `Choose one registered workspace` | 淡色（dim） |
| 検索バー（フィルター） | `Filter: type to filter`。通常文字を入力するとフォーカス操作なしでフィルターへ追加する。カーソルは表示しない。入力した文字列を含むワークスペース名だけを表示する（大文字小文字は区別しない） | ラベル・未入力時のヒント：淡色。入力値：シアン |
| プロジェクト一覧 | 登録済みワークスペースをアルファベット順で 2 行表示。1 行目は「カーソル + 名前(幅揃え) + パス」、2 行目は統計行。パスはブロック幅に収まるよう先頭を省略 | 選択行：`>` 赤太字／名前 シアン太字。パス・統計行は常に淡色 |
| 統計行 | 各ワークスペースのセッション数・未完了 issue 数・最終利用からの経過時間をアイコン付きで表示 | 淡色（dim） |
| 空表示 | 登録が無い場合のメッセージ | 淡色（dim） |
| 一致なし表示 | フィルターに一致するワークスペースが無い場合のメッセージ | 淡色（dim） |
| 通知行 | 読み込み失敗時などのメッセージ。未表示時も 1 行ぶんの枠を確保しレイアウトを安定させる | 黄色 |
| フッター | 操作ヘルプ。通常の単一オープンでは `←→ tab / ↑↓ move / type filter / Enter open / Esc back` | 淡色（dim） |

## キー操作

| キー | 動作 |
|---|---|
| `↑` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `←` | `Single` タブへ切り替える。チェック状態は内部に残るが、`Enter` はカーソル行 1 件だけを開く |
| `→` | `Unite` タブへ切り替える。初期状態では何もチェックしない |
| 通常文字 | その文字をフィルター文字列へ追加し、ワークスペース名を絞り込む（`j` / `k` / `q` / `u` も通常文字として入力する） |
| `Backspace` | フィルター末尾の 1 文字を削除する |
| `Space` | [統合（unite）モード](#統合uniteモードで開く)へ入り、カーソル行をチェックする。通常表示ではチェック列を空白として予約するため、統合モードへ入ってもワークスペース名の列はズレない |
| `Enter` | カーソル行 1 件を開く。[確定時のうさぎの演出](#確定時のうさぎの演出)を経てホーム画面の**切替（Switch）モード**を表示する。開く対象のディレクトリが存在しない場合は開かず、その行へカーソルを移して[削除確認](#存在しないワークスペースの削除確認)を表示する |
| `Esc` | 起動画面へ戻る |
| `Ctrl+C` / `Ctrl+Q` | アプリを終了 |

## 統合（unite）モードで開く

複数のワークスペースを **1 つのホーム画面にまとめて**開ける。**単一オープンとまとめて開くで
UI を分ける**ため、Open 画面には `Single │ Unite` タブを表示する。通常の `Single` タブはチェック表示の
ない単一オープン専用で、`→`（または `Space`）を押したときだけ明示的な統合（unite）タブへ
切り替える。チェック列の幅は通常画面でも空白として予約するため、切り替えてもワークスペース名・
パスの列は横にズレない。統合モードではタブの `Unite` がシアン太字になり、サブタイトルが
`Select workspaces to open together` に変わり、各プロジェクト行にチェック列（`✓`）が現れる。

- `→` で統合モードへ入ると、初期状態では何もチェックしない。必要な行を `Space` でチェックしてから開く。
- `Space` で統合モードへ入ると、カーソル行をチェックする。直近の統合選択が記憶されていても、
  `Space` を押した行が選ばれる。
- 統合モード中は `Space` でカーソル行のチェックをトグルし、`Enter` でチェックした
  全ワークスペースを同時に開く。チェックが 0 件のまま `Enter` を押した場合は開かず、通知行に `Select at least one workspace to unite.` を表示する。
- `Esc` は統合モードを抜けて通常の単一オープン UI に戻る。戻った後の `Enter` は、内部に
  残ったチェックに関係なく**カーソル行 1 件だけ**を開く。
- **1 件**を開くと従来どおりの単一ワークスペースのホーム画面。**2 件以上**を開くと
  [統合（unite）モード](home/README.md)で、左ペインに各ワークスペースが
  グループとして積み重なって表示される（ワークスペース名のヘッダ＋そのワークスペースの
  `⌂ root` 行とセッション）。
- **直近の統合選択を記憶**する。2 件以上をまとめて開いたときの選択は保存される（保存先は [../data/01-global.md](../data/01-global.md)）。通常の `→` で `Unite` タブへ入った直後は何もチェックしないため、必要なワークスペースをその場で選ぶ。単一オープンはこの記憶を上書きしない。

```text
│                    Open Project                       │  ← タイトル（緑・太字）
│                    Single │ Unite                     │  ← Unite タブがシアン太字
│           Select workspaces to open together          │  ← サブタイトル（淡色）
│                                                       │
│      > ✓ my-project    /path/to/my-project            │  チェック列（✓）＋カーソル（>）
│            ⎇ 3 sessions   ● 5 open   ◷ 2d ago         │
│        ✓ other-repo    /path/to/other-repo            │  複数チェックで unite
│            ⎇ 0 sessions   ● 12 open  ◷ 3w ago         │
│                                                       │
│ ←→ tab / ↑↓ move / Space toggle / Enter open checked… │  ← フッター（統合モード）
```

| キー（統合モード中） | 動作 |
|---|---|
| `↑` / `↓` | カーソルを上下に移動（端でラップ）。通知をクリア |
| `←` | `Single` タブへ戻る。チェック状態は内部に残るが、通常の `Enter` はカーソル行 1 件だけを開く |
| `→` | `Unite` タブを維持する |
| `Space` | カーソル行のチェックをトグルする。チェックが 0 件になることも許可する |
| `Enter` | チェックした全ワークスペースをまとめて開く |
| `Esc` | 統合モードを抜けて通常の単一オープン UI に戻る |
| `Ctrl+C` / `Ctrl+Q` | アプリを終了 |

## 存在しないワークスペースの削除確認

`Enter` で選んだワークスペースのパスがディスク上に存在しない（worktree ごと削除された等）場合は、
ホーム画面を開かずに削除確認のモーダルを表示します。一覧と同じく中央に枠付きで描画され、
答えるまでモーダルが画面を占有します。

```text
┌─ Workspace not found ──────────────────────┐
│ "mosou-wars" no longer exists on disk.      │  ← 対象名（淡色）
│                                             │
│ Remove it from the list?                    │  ← 確認文
│                                             │
│ y / Enter: remove   n / Esc: cancel         │  ← キーヒント（淡色）
└─────────────────────────────────────────────┘
```

| キー | 動作 |
|---|---|
| `y` / `Y` / `Enter` | `workspaces.json` から該当エントリを削除し、一覧からも除いて元の画面へ戻る。結果は通知行に表示（成功時 `Removed "<名前>".`／失敗時 `Failed to remove "<名前>": <理由>`） |
| `n` / `N` / `Esc` | 削除せずモーダルを閉じる |
| `Ctrl+C` / `Ctrl+Q` | アプリを終了 |

> ディレクトリの実体は削除しません。レジストリ（`workspaces.json`）からの登録解除のみを行います。
> 削除自体の仕様は [../data/01-global.md](../data/01-global.md) を参照してください。

## 確定時のうさぎの演出

`Enter` でワークスペースを開くと、次の順番で制御が進みます。

1. ワークスペースを選択する。
2. プロジェクト一覧を**非表示**にする（画面を空にする）。
3. うさぎ AA のアニメーションを再生する。上部中央に表示していたうさぎが、空になった画面を、
   画面の**左下**——ホーム画面のステータス行が出る位置のすぐ上——へ滑り降りる。
   このアニメーションの**裏側（別スレッド）でワークスペースのロードを開始**する（`state.json`・
   issue・設定・コマンド履歴の読み込み）。いずれもディスク読み込みのみで、アニメーションが
   終わる頃には済んでいるため、合流（join）はほぼ一瞬で済む。
4. ホーム画面の**切替（Switch）モード**を表示する。ロードが先行しているため最初の 1 フレームが
   ブロックされず、うさぎが左下に着いた瞬間にスムーズに描画される。

エージェント CLI の探索（在席の AI エージェント選択に並ぶ顔ぶれ）は上記のロードには含めない。
各 CLI を `--version` で起動して確かめるためアニメーションより長引きうるので、ローカル LLM の判定と
同じく**ホーム画面を表示したあとに別スレッドで実行し、結果が出た時点で差し替える**（探索が済むまで
選択肢は空＝既定エージェントのみ）。最初の 1 フレームをサブプロセスで待たせないための切り分けである。

```text
┌───────────────────────────────────────────────────────┐
│                       (\(\                            │  ← 開始位置（ヘッダーのうさぎ）
│                       (='-')        ╲                 │  （一覧は非表示。うさぎだけが残る）
│                      o(_(")(")        ╲               │
│                                         ╲             │
│                                          ↓            │
│                                                       │
│   (\(\                                                │  ← 着地位置（左下）
│   (='-')                                              │
│  o(_(")(")                                            │
│   Pick a session …（ホーム画面 切替モードのステータス行）│
└───────────────────────────────────────────────────────┘
```

- スプラッシュと同様に**入力を読まず**、フレーム間を一定間隔（[共通の描画間隔](../05-settings.md)）で
  自走します。演出中に押されたキーはバッファに残り、ホーム画面側で処理されます。
- 一覧はアニメーション開始と同時に消え、以降は空の画面の上をうさぎだけが移動します。
- 開始位置はうさぎ AA を実際に描いていた行・列、着地位置は左下（ステータス行の 1 行上）で、
  両者を直線補間しながら 1 フレームずつ下ろします。全フレーム再生後にホーム画面へ切り替わります。

## 読み込みと確定後の挙動

- ワークスペースの読み込みに失敗した場合は、空一覧の状態でエラー内容を通知行に表示します。
- `Enter` でワークスペースを選択するとホーム画面の切替（Switch）モードへ遷移します。このとき選択した
  ワークスペースの最終利用時刻（`updated_at`）を現在時刻に更新します（統計行の表示に反映）。
  一覧の並びはアルファベット順のままで、最終利用時刻では並び替えません。
- ホーム画面から戻る（`Esc`）とこの一覧へ復帰します。復帰時には今開いたワークスペースへ
  カーソルを合わせ、一覧の並び（アルファベット順）はそのまま維持します。
  アプリ終了（`Ctrl+C`）はそのまま上位へ伝播します。

> 描画ロジックは `src/presentation/tui/open/`（一覧状態 `state.rs`・描画 `ui.rs`・
> イベントループ `event.rs` に分離）に実装されています。一覧の元データ（`workspaces.json`）の
> 仕様は [../data/01-global.md](../data/01-global.md) を参照してください。
