# 4. 設定画面（Config）

> [画面設計トップ](README.md) ｜ ← 前へ [3. 新規プロジェクト画面（New）](03-new.md) ｜ 次へ → [5. ホーム画面（Home）](05-home.md)

設定の閲覧・編集を行う画面。**どこから開いたかで編集対象（スコープ）が分かれます。**

| 開き方 | スコープ | 編集対象 |
|---|---|---|
| 起動画面で `Config`（`c` / Enter）を選ぶ | **グローバル** | `~/.usagi/settings.json`（アプリ全体の設定） |
| ホーム画面のコマンドモードで `config` を実行 | **ワークスペース（ローカル）** | `<workspace>/.usagi/settings.json`（そのワークスペース固有の上書き） |

> CLI の `usagi config` も**グローバル設定**を対象にしますが、本画面（TUI のセレクタ）ではなく
> `$EDITOR` での編集／一覧表示です。詳細は [3. コマンドリファレンス（CLI）](../03-commands/01-cli.md) を参照。

2 つのスコープは画面を共有せず、表示・編集される項目はスコープごとに分かれます。グローバルとローカルを
1 画面で同時に編集することはありません（ローカル画面でもグローバル設定は読み込みますが、各上書きが未設定の
ときに「何にフォールバックするか」を表示するためだけに使います）。

各設定項目の値は常に `< 値 >` の形（左右セレクタ）で表示し、選択中の項目を `←→` で切り替えます。
編集はメモリ上に保持され、変更した項目はラベル左の黄色 `●`（および値の黄色）で「未保存の変更あり」と
明示されます。`settings.json` への書き込みは、フィールドの下にある **Save ボタン**（変更があるときだけ
有効化）を押したときに行われます。起動画面と同じ代替スクリーン上に描画されます。

> 設定の項目・既定値・保存先（`settings.json`）の仕様は [../05-settings.md](../05-settings.md)、
> [../data/01-global.md](../data/01-global.md)（グローバル）、[../data/02-workspace.md](../data/02-workspace.md)
> （ローカル）を参照してください。本書は画面の見た目と操作に絞ります。

## グローバル設定（起動画面）

起動画面の `Config` から開いたときのスコープ。アプリ全体の設定（`~/.usagi/settings.json`）を編集します。
設定全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。

```text
┌───────────────────────────────────────────────────────┐
│                       (\(\                            │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                           │  │
│                      o(_(")(")                        │  ┘
│                       Config                          │  ← タイトル（緑・太字）
│            Adjust your global preferences             │  ← サブタイトル（淡色）
│                                                       │
│          Theme              < System >                │  ┐ 設定一覧（全項目 `< 値 >`）
│          Default Workspace  < (none) >                │  │  値は常に山かっこで囲み列を揃える
│      >   Notifications      < On >                    │  │  選択行：左端の赤 ">" + 値が明色
│          Restore Panes      < On >                    │  │  起動時にペインを復旧（agent は会話を再開）
│        ● Agent CLI          < Gemini >                │  │  変更済み：ラベル左に黄色 ● + 値も黄色
│          Session Action UI  < Menu >                  │  │  在席の右ペイン UI（Menu / Prompt）
│          Local LLM            Install                 │  │  未導入は `Install`（緑・山かっこなし）、導入後は `< On >`/`< Off >`
│          Local LLM Model      qwen2.5-coder:7b        │  ┘  導入後はアクション。Space/Enter でモデル選択モーダル
│                                                       │
│                           [ Save ]                    │  ← Save ボタン（変更時のみ緑で有効）
│                                                       │
│        Saved 🐰                                       │  ← 通知行（任意・黄色）
│                                                       │
│    ↑↓: move · ←→: change · Enter: save · Esc: back    │  ← フッター（淡色）
└───────────────────────────────────────────────────────┘
```

| 項目 | 値 | 循環する選択肢 |
|---|---|---|
| Theme | UI のカラーテーマ | `Light` → `Dark` → `System`（OS 追従）の順に循環 |
| Default Workspace | 既定で開くワークスペース | `(none)` ＋ 登録済みワークスペース名を順に循環。未登録時は変更不可（ヒントを表示） |
| Notifications | デスクトップ通知（`agent` の入力待ち・完了）の ON/OFF | `On` ⇄ `Off` をトグル |
| Restore Panes | 起動時に各セッションのペイン（agent / terminal）を復旧（agent は会話を再開） | `On` ⇄ `Off` をトグル（[04-orchestration.md#ペインの復旧](../04-orchestration.md#ペインの復旧)） |
| Agent CLI | usagi が起動する AI エージェント CLI | **PATH に存在する（インストール済みの）agent だけ**を `Claude` → `Codex` → `sakana.ai` → `Gemini` の順に循環（`sakana.ai` は Codex 互換 CLI `codex-fugu` を起動）。未インストールの agent は選択肢に出さない。ただし現在保存されている値が未インストールでも、画面を開いただけで失わないよう選択肢に残す |
| Session Action UI | 在席（Focus）の右ペインのアクション UI スタイル | `Menu`（選べるリスト）⇄ `Prompt`（コマンドライン）をトグル（[05-home.md](05-home.md#在席のアクション-uimenu--prompt)） |
| Local LLM | ローカル LLM 委譲の有効化（`ollama` ランタイムの導入と on/off） | 未導入時は値が `Install`（アクション）で、`Space` / `Enter` でランタイムのインストールモーダルを開く。導入後は `On` ⇄ `Off` をトグル |
| Local LLM Model | 委譲先の Ollama モデル | ランタイム未導入のうちは操作不可（`—` を淡色表示）。導入後はアクション行になり、`Space` / `Enter` で**モデル選択モーダル**を開く。使用中モデルが未取得なら値に `(未導入)` を併記 |

> ほとんどの項目は共通の chooser widget で描画し、現在値を常に `< 値 >` の形で表示します。例外は `Local LLM` の
> **インストールアクション**と `Local LLM Model` の**モデル選択アクション**で、これらは選択肢を循環するのではなく動作（モーダル）を
> 起動するため、山かっこなしの緑のラベルで表示します。ランタイム未導入のあいだ `Local LLM Model` は淡色の `—` で操作不可です。

> **ローカル LLM の導入**: ローカル LLM の有効化は **2 段階**で、どちらの導入も**バックグラウンドで進みます**。
>
> 1. **ランタイム導入（`Local LLM` 行）**: 未導入のうちは値が `Install` のアクションで、`Space` / `Enter` で
>    **インストールモーダル**を開き、ランタイム導入に必要な **sudo パスワード**を伏せ字で入力させます。`Enter` で確定すると
>    公式インストーラ（`curl -fsSL https://ollama.com/install.sh | sh`）を別スレッドで開始し、sudo を事前認証して非対話で
>    導入します（モデルはここでは取得しません）。
> 2. **モデル選択（`Local LLM Model` 行）**: ランタイム導入後、`Space` / `Enter` で**モデル選択モーダル**を開きます。
>    提供モデル（`qwen2.5-coder:7b` / `:3b` / `:1.5b` / `qwen2.5:7b`）を導入状態のマーカー付き（`✓ 導入済` / `⬇ 未導入`）で
>    一覧し、↑↓ で選び `Enter` で決定します。**導入済み**はそのまま使用モデルに採用し、**未導入**は `ollama pull <model>`
>    （sudo 不要）をバックグラウンドで開始してから採用します。`Esc` でキャンセル。
>
> いずれの導入も**生出力は画面に出さず**（`pulling manifest …` などで TUI が崩れないよう抑制）、導入中も Config を含む
> usagi の他機能をそのまま操作できます（`Esc` で他画面へ移動しても継続）。進行は全画面共通の
> [**ローディングうさぎ**オーバーレイ](#インストール中のローディングうさぎ)で表示します。完了すると `Local LLM` 行が `On` の
> **on/off トグル**へ切り替わる／選んだモデルが使用モデルになり（次のキー操作時に反映）、結果を通知行に表示します。失敗時は
> オーバーレイに結果を表示します。既定は無効で、usagi が勝手に有効化することはありません
> （[3.4 ローカル LLM MCP サーバ](../03-commands/04-llm-mcp.md)）。

> `settings.json` には `workspace_root`（新規プロジェクトのクローン先ベースディレクトリ）も
> 保存されますが、本画面では編集しません（`usagi config --edit` で変更します）。

> Save を押すとグローバル設定（`~/.usagi/settings.json`）のみを保存します。このスコープでは
> ワークスペース固有の設定は表示・編集しません。

## ワークスペース設定（ホーム画面の config）

ホーム画面のコマンドモードで `config` を実行したときのスコープ。起動中のワークスペース
（`<workspace>/.usagi/settings.json`）だけに効く **ローカルの上書き**を編集します。グローバル設定は
ここには表示されず、対象は次の 4 項目だけです。

```text
┌───────────────────────────────────────────────────────┐
│                       (\(\                            │  ┐ うさぎ AA
│                      (='-')                           │  │
│                      o(_(")(")                        │  ┘
│                   Workspace Config                    │  ← タイトル（緑・太字）
│            Adjust this workspace's settings           │  ← サブタイトル（淡色）
│                                                       │
│      >   Agent CLI       < Global (Claude) >          │  ┐ ローカル設定一覧（5 項目）
│        ● Notifications   < Override: Off >            │  │  選択行：左端の赤 ">"
│          Restore Panes   < Global (On) >              │  │  変更済み：ラベル左に黄色 ●
│          Default Branch  < develop >                  │  │
│          Branch Source   < Remote >                   │  ┘
│                                                       │
│                           [ Save ]                    │  ← Save ボタン
│                                                       │
│    ↑↓: move · ←→: change · Enter: save · Esc: back    │  ← フッター
└───────────────────────────────────────────────────────┘
```

| 項目 | 値 | 循環する選択肢 |
|---|---|---|
| Agent CLI | このワークスペースの Agent CLI 上書き | `Global (実効値)` → インストール済みの agent を `Override: Claude` → `Override: Codex` → `Override: sakana.ai` → `Override: Gemini` の順に循環（未インストールは出さない。既存の上書き値が未インストールでも選択肢に残す） |
| Notifications | このワークスペースの通知 ON/OFF 上書き | `Global (実効値)` → `Override: On` → `Override: Off` の順に循環 |
| Restore Panes | このワークスペースのペイン復旧 ON/OFF 上書き | `Global (実効値)` → `Override: On` → `Override: Off` の順に循環 |
| Default Branch | `session create` で worktree を切る基点ブランチ | `Default (auto)` → リポジトリの各ブランチ名 の順に循環（実在ブランチを検出） |
| Branch Source | 上のブランチをローカル形／リモート形のどちらで使うか | `Local` ⇄ `Remote` をトグル（未設定時は `Default (Remote)` を表示） |

- Agent CLI と Notifications は **「グローバルに従う / ローカルで上書き」** を 1 つの chooser で切り替えます。
  `Global (...)` は未上書き（グローバル設定にフォールバック）で、かっこ内に現在の実効値を表示します。
  `Override: ...` を選ぶとそのワークスペース専用の値になります。再度 `Global` まで循環させれば上書きを解除できます。
- Default Branch はグローバルに対応項目がなく、`Default (auto)`（リポジトリの検出済み既定ブランチ）と、対象
  リポジトリの実在ブランチ（`list_branches` で検出）を循環選択します。`auto` から `←→` でブランチ名を選ぶとその
  ブランチを基点として保存されます。リポジトリでない・ブランチが無いワークスペースでは選択肢が無く no-op です。
- Branch Source もグローバルに対応項目がないため、`Local`（ローカルのブランチ）と `Remote`（リモート追従の
  ブランチ）の 2 値だけを切り替えます。未設定時は既定値を `Default (Remote)` と表示し、`←→` で切り替えると
  明示値（`Local` / `Remote`）として保存されます。
- Save を押すと、そのワークスペースのローカル設定（`<workspace>/.usagi/settings.json`）のみを保存します。
  全項目を未上書きに戻した場合もファイルは残し（中身は空に近い JSON）、「グローバルに従う」を意味します
  （削除はしません）。
- 詳細なデータ仕様は [../data/02-workspace.md](../data/02-workspace.md) の `settings.json`（ローカル設定）を参照。

## widget による値の表示

設定値は共通の chooser widget（`src/presentation/tui/widgets/` の `chooser`）で統一して描画します。

- **chooser**: 現在値のみを表示する左右セレクタです。値を常に `< Dark >` のように山かっこで囲み、
  `←→` で前後の選択肢に切り替えられることを示します。山かっこは全行で同じ列に揃います。フォーカスの
  状態は色で表し、選択行は明色（シアン・太字）、それ以外は淡色（dim）で描画します。
- **アクション行**: `Local LLM`（未導入時）の `Install` や `Local LLM Model`（導入後）のモデル名のように、選択肢を
  循環するのではなく動作（モーダル）を起動する行は chooser ではなく**山かっこなしの緑のラベル**で描画し、選択行では太字にします。
- **操作不可行**: ランタイム未導入時の `Local LLM Model` のように、まだ操作できない行は淡色の `—` で描画します。
- **変更の明示**: 保存済みの値と異なる項目は、ラベルの左に黄色の `●` を表示し、あわせて値も黄色・太字に
  して未保存の編集であることを示します（`●` はカーソルの隣の専用列に置くため、選択と変更を同時に表せます）。

## 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| タイトル | グローバル: `Config` ／ ワークスペース: `Workspace Config` | 緑・太字 |
| サブタイトル | グローバル: `Adjust your global preferences` ／ ワークスペース: `Adjust this workspace's settings` | 淡色（dim） |
| 設定一覧 | 各設定を「カーソル列 + 変更マーク列 + ラベル(幅揃え) + 値」で 1 行ずつ表示 | 選択行：左端 `>` 赤太字／ラベル シアン太字。値は chooser（常に `< 値 >`）、ただしアクション行（`Install` / モデル名）は山かっこなしの緑ラベル、操作不可行（未導入時のモデル）は淡色の `—`。変更済みはラベル左に `●`（黄色）＋値も黄色 |
| Save ボタン | 設定一覧の下に 1 行空けて配置する `[ Save ]`。値の列に揃える | 変更ありで緑・太字（有効）、変更なしで淡色（無効）。選択行は `>` 赤太字 |
| 通知行 | 保存結果などのメッセージ（例: `Saved 🐰`）。未表示時も 1 行ぶんの枠を確保しレイアウトを安定させる | 黄色 |
| フッター | 操作ヘルプ `↑↓: move · ←→: change · Enter: save · Esc: back` | 淡色（dim） |

## キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（設定一覧と Save ボタンの間をラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾の Save ボタンの次は先頭へラップ）。通知をクリア |
| `→` / `l` | 選択中の項目の値を次の選択肢へ切り替え（メモリ上のみ。保存はしない）。Save ボタン上・`Local LLM` の `Install` 上・`Local LLM Model`（モーダルで選ぶ）上では無効 |
| `←` / `h` | 選択中の項目の値を前の選択肢へ切り替え（メモリ上のみ。保存はしない）。Save ボタン上・`Local LLM` の `Install` 上・`Local LLM Model` 上では無効 |
| `Space` | `Local LLM` の `Install` 行でインストールモーダルを、`Local LLM Model`（導入後）行でモデル選択モーダルを開く（他の行・スコープでは無効） |
| `Enter` | Save ボタン上では編集内容を `settings.json` へ保存。`Local LLM`（未導入時）上ではインストールモーダルを、`Local LLM Model`（導入後）上ではモデル選択モーダルを開く。その他の設定項目上では `→` と同じく値を次へ切り替え |
| `q` / `Esc` | 前の画面へ戻る（起動画面から開いた場合は起動画面、ホーム画面の `config` から開いた場合はホーム画面。未保存の編集は破棄） |
| `Ctrl+C` / `Ctrl+Q` | アプリを終了 |

### インストールモーダルのキー操作

`Local LLM` の `Install` を起動するとモーダルが開き、開いている間はすべてのキーを受け取ります。

パスワードは共通の入力ウィジェットで編集され、キャレット位置での挿入・削除と左右移動に対応します（伏せ字でもブロックカーソルが正しい伏せ字 `•` の上に表示されます）。

| キー | 動作 |
|---|---|
| 文字キー | sudo パスワードのキャレット位置に 1 文字挿入（伏せ字 `•` で表示） |
| `Backspace` / `Del` | キャレットの前／後の 1 文字を削除 |
| `←` / `→` | キャレットを 1 文字左／右へ移動 |
| `Home` / `End` | キャレットを行頭／行末へ移動 |
| `Enter` | 入力した sudo パスワードでランタイム導入を**バックグラウンドで開始**し、モーダルを閉じて操作に戻る |
| `Esc` | モーダルを閉じてキャンセル（導入しない） |
| `Ctrl+C` / `Ctrl+Q` | アプリを終了 |

### モデル選択モーダルのキー操作

ランタイム導入後に `Local LLM Model` 行で `Space` / `Enter` を押すとモデル選択モーダルが開き、開いている間はすべてのキーを受け取ります。各モデルは導入状態のマーカー（`✓ 導入済` / `⬇ 未導入`）付きで一覧されます。

| キー | 動作 |
|---|---|
| `↑` / `k` | カーソルを 1 つ上のモデルへ移動（末尾へラップ） |
| `↓` / `j` | カーソルを 1 つ下のモデルへ移動（先頭へラップ） |
| `Enter` | カーソル位置のモデルを採用。**未導入**なら `ollama pull`（バックグラウンド実行＋スピナー）で取得してから採用 |
| `Esc` | モーダルを閉じてキャンセル（採用しない） |
| `Ctrl+C` / `Ctrl+Q` | アプリを終了 |

> キー操作はグローバル／ワークスペースのどちらのスコープでも共通です。

## インストール中のローディングうさぎ

ローカル LLM の導入はバックグラウンドで進むため、進行状況は**全画面共通のオーバーレイ**として画面右上に表示します。
起動画面・プロジェクト選択・新規作成・Config・ホームのどの画面にいても表示され、導入中も各画面の機能を操作できます。

```
                              ∩∩
                            (･ㅅ･)づ⠹ LLM 導入中… qwen2.5-coder:7b
```

- **跳ねるうさぎ＋ブレイルスピナー**が一定間隔（約 110ms）で動きます。
- **うさぎの表情は経過時間で切り替わります**（約 1.5 秒ごと）。進捗率を表すものではなく、ダウンロード量に依らず
  時間で勝手に変化します（`ollama pull` は進捗を返さないため、時間ベースで「動いている」ことだけを伝えます）。
- 導入が完了すると、数秒間だけ結果（成功は `✓`、失敗は `✗`）と完了メッセージを同じ位置に表示してから消えます。
- キー入力が無くても、導入中は一定間隔で再描画してアニメーションを進めます。

> オーバーレイは描画段（`FramePainter`）で全フレームに重ねるため、各画面は自分で描画する必要がありません。
> 進行状態はプロセス共通の `src/presentation/tui/install_task.rs` が一元管理します（同時に走る導入は 1 つだけ）。

## 読み込みと保存の挙動

- 設定の読み込みに失敗した場合は、既定値の状態でエラー内容を通知行に表示します。
- 値の切り替え（`←→` / 設定項目上の `Enter`）はメモリ上の編集のみで、ディスクには書き込みません。
  保存済みの値から変わった項目は黄色＋ `●` で示し、変更がある間は Save ボタンが緑（有効）になります。
- Save ボタンで `Enter` を押すとそのスコープの `settings.json`（グローバルスコープなら
  `~/.usagi/settings.json`、ワークスペーススコープなら `<workspace>/.usagi/settings.json`）へ保存し、
  `Saved 🐰` を通知して変更マークを消します。変更が無い状態で押した場合は `No changes to save 🐰` を表示し、
  書き込みは行いません。保存に失敗した場合は `Failed to save: <理由>` を表示し、画面はそのまま継続します
  （再操作・離脱が可能）。
- 既定ワークスペースの選択肢が無い（ワークスペース未登録）状態で値を切り替えようとすると、
  `No workspaces to choose from 🐰` を表示します（グローバルスコープのみ）。

> 描画ロジックは `src/presentation/tui/config/`（設定状態 `state.rs`・描画 `ui.rs`・
> イベントループ `event.rs` に分離）、設定の永続化は `src/usecase/settings.rs` と
> `src/infrastructure/storage.rs` に実装されています。
