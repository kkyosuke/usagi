# 1. 起動画面（Welcome）

> [画面設計トップ](README.md) ｜ ← 前へ [0. 起動スプラッシュ（Splash）](00-splash.md) ｜ 次へ → [2. プロジェクト選択画面（Open）](02-open.md)

`usagi hop` 実行時、[0. 起動スプラッシュ](00-splash.md) の再生後に表示される画面。代替スクリーン（alternate screen）上に描画され、
終了時に元の端末状態へ復帰します。代替スクリーン中はマウスレポート（DECSET 1000 + SGR 1006）と
端末エコー無効化を有効にし、ホイール操作で画面が意図せずスクロールしないようにします。この
スクロール／マウスホイールの全体方針は [5. ホーム画面](home/04-keys.md#ペインのスクロールとマウスホイール) を参照してください。

## レイアウト

端末サイズに応じて本文ブロックを水平方向に中央寄せして描画します（幅・高さが取得できない環境では
80×24 をフォールバック値として使用）。本文（うさぎ AA・タイトル・2 列メニュー・通知行）はフッターの
上で縦中央寄せします。2 列メニューは、左側に従来の Open / New / Config / Quit、右側に最近使った
ワークスペース（Recent）を最大 3 件のカードで表示します。Recent は上から `1` / `2` / `3` で直接開けます。
左右の間には淡色の縦 divider を置き、divider の列をうさぎ AA / タイトルの中心列に合わせます。

**このときのうさぎ AA の行が、全トップレベル画面で共通のマスコット位置の正本です。**
スプラッシュ・[プロジェクト選択画面（Open）](02-open.md)・[新規プロジェクト画面（New）](03-new.md)・
[設定画面（Config）](04-config.md) はいずれもこの行にうさぎ AA を合わせるため、画面を行き来しても
うさぎが上下にガタつきません（レイアウトシフトなし）。各画面の本文がこの行に収まらないほど縦に短い
端末でのみ、うさぎはその画面に収まる位置まで引き上げられます。

```text
┌───────────────────────────────────────────────────────┐
│                                                       │  ← 上部パディング
│                       (\(\                            │  ┐
│                       (='-')                          │  ├ うさぎ AA（マゼンタ・太字）
│                      o(_(")(")                        │  ┘
│                                                       │
│                       USAGI                           │  ← タイトル（緑・太字）
│                                                       │
│                         │                             │  ┐
│       > Open           o │  Recent                    │  │
│                         │  ┌─ 1 repo ───────────────┐ │  │
│         New            e │  │ ◷ 11min ago ⎇ 2 #1 ● 4│ │  ├ 2 列メニュー
│                         │  └────────────────────────┘ │  │   左：従来メニュー
│         Config         c │  ┌─ 2 tools ──────────────┐ │  │   中央：divider
│                         │  │ ◷ 3h ago    ⎇ 1 #0 ● 0│ │  │   右：Recent カード（1〜3）
│         Quit           q │  └────────────────────────┘ │  │
│                         │  ┌─ 3 docs ───────────────┐ │  │
│                         │  │ ◷ 2d ago    ⎇ 0 #0 ● 2│ │  │
│                         │  └────────────────────────┘ │  ┘
│                                                       │
│                 Opened "repo" 🐰                      │  ← 通知行（任意・黄色）
│                                                       │
│      v0.1.0 | ↑↓: move / Enter or letter: select / 1-3: recent / q: quit │  ← フッター（淡色）
└───────────────────────────────────────────────────────┘
```

## 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| うさぎ AA | 3 行のアスキーアート `(\(\` / `(='-')` / `o(_(")(")`| マゼンタ・太字 |
| タイトル | `USAGI` | 緑・太字 |
| 左メニュー | Open / New / Config / Quit の 4 項目。各行は「カーソル + ラベル(10桁左詰め) + キー(5桁右詰め)」 | 選択行：`>` 赤太字／ラベル シアン太字／キー 黄色。非選択行：素のテキスト |
| Divider | 左メニューと Recent の間の縦線。うさぎ AA / タイトルの中心列に揃える | 淡色（dim） |
| Recent | 最近使ったワークスペースを最大 3 件。`workspace::overviews` の並び（最終利用日時の新しい順）から先頭 3 件をカード表示し、上から `1` / `2` / `3` で直接開く。カード本文は `◷` 最終利用からの経過時間、`⎇` セッション数、`#` PR 数、`●` open issue 数を表示する。3 件未満の空き枠は `—` と `No recent workspace` のカード | 見出し：緑太字。数字：黄太字。カード枠・本文：淡色（dim） |
| 通知行 | メニュー操作の結果メッセージ（例: New 完了後の `Opened "repo" 🐰`）。未表示時は何も描画しない | 黄色 |
| フッター | バージョンと操作ヘルプ `v<version> \| ↑↓: move / Enter or letter: select / 1-3: recent / q: quit` | 淡色（dim） |

## メニュー項目

| ラベル | キー | 動作 |
|---|---|---|
| Open | `o` | プロジェクト選択画面へ遷移 |
| New | `e` | 新規プロジェクト画面へ遷移 |
| Config | `c` | 設定画面（**グローバルスコープ**）へ遷移。アプリ全体の設定を編集する |
| Quit | `q` | アプリを終了する |
| Recent 1〜3 | `1` / `2` / `3` | 対応する最近使ったワークスペースを直接開く。未表示の番号は何もしない |

## キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `Enter` | 選択中の項目を実行。Open / New / Config は対応する画面へ遷移、Quit なら終了 |
| `o` / `e` / `c` | 対応するメニュー項目を直接実行（ショートカット） |
| `1` / `2` / `3` | 右列 Recent の 1〜3 件目を直接開く。開いたワークスペースは Open 画面から開いたときと同じく最終利用日時を更新する |
| `q` / `Esc` / `Ctrl+C` / `Ctrl+Q` | アプリを終了 |

## 終了時の挙動

- 正常終了（Quit / `q` / `Esc` / `Ctrl+C` / `Ctrl+Q`）時：代替スクリーンを解除しカーソルを復帰したうえで、
  手を振るマスコットと別れのメッセージを枠付きのボックスで表示します（枠は淡色 dim、うさぎは
  シアン）。

  ```
  ╭──────────────────────────────╮
  │    (\(\                       │
  │    ( ^ω^)ﾉ  またね、ぴょん！   │
  │   o(_(")(")                   │
  ╰──────────────────────────────╯
  ```
- キー読み取りが中断された場合（シグナル等）：終了として扱います。
- キー読み取りでエラーが発生した場合：別れのメッセージは出さずに端末を復帰し、エラーを返します。

これらの端末復帰（代替スクリーンの解除・マウスレポートと代替スクロールモードの復帰・カーソルの
復帰）は `AlternateScreenGuard`（RAII ガード）により、早期 return やパニック時でも保証されます。

> 描画ロジックは `src/presentation/tui/welcome/`（メニュー定義 `menu.rs`・描画 `ui.rs`・
> イベントループ `event.rs`）に実装されています。
