# 1. 起動画面（Welcome）✅ 実装済み

> [画面設計トップ](README.md) ｜ 次へ → [2. プロジェクト選択画面（Open）](02-open.md)

`usagi hop` 実行時に最初に表示される画面。代替スクリーン（alternate screen）上に描画され、
終了時に元の端末状態へ復帰します。代替スクリーン中はマウスレポート（DECSET 1000 + SGR 1006）を
有効化し、ホイール操作を端末側のスクロールや矢印キー合成ではなくマウスイベントとして受け取って
破棄します。これによりホイールでメニューや画面全体が意図せずスクロールしません。代替スクロール
モード（DECSET 1007）はあえて使いません——ホイールを矢印キーに化けさせると本物の矢印と区別できず、
ホイールをマウスとして報告しない端末では一覧が勝手に動いてしまうためです。マウスレポート 1 本に
統一することで、どの端末でも TUI 自体は確実にスクロールしません。あわせて TUI 表示中は端末エコー
（`ECHO`）を無効化します（`console` がキー読み取りの合間に cooked モードへ戻すため、流入する
マウスレポート列がカーネルエコーで画面に出力されるのを防ぐ。出力処理 `OPOST` は維持するので描画は
そのまま）。

## レイアウト

端末サイズに応じて全要素を水平方向に中央寄せして描画します（幅・高さが取得できない環境では
80×24 をフォールバック値として使用）。

```text
┌────────────────────────────────────────────────────┐
│                                                      │  ← 上部パディング
│                       (\(\                           │  ┐
│                      (='-')                          │  ├ うさぎ AA（マゼンタ・太字）
│                      o(_(")(")                        │  ┘
│                                                      │
│                       USAGI                          │  ← タイトル（緑・太字）
│                                                      │
│                 > Open           o                   │  ┐
│                                                      │  │
│                   New            e                   │  ├ メニュー（選択行は
│                                                      │  │   赤の ">" + シアン太字ラベル）
│                   Config         c                   │  │
│                                                      │  │
│                   Quit           q                   │  ┘
│                                                      │
│                 Opened "repo" 🐰                     │  ← 通知行（任意・黄色）
│                                                      │
│      v0.1.0 | ↑↓: move / Enter: select / q: quit     │  ← フッター（淡色）
└────────────────────────────────────────────────────┘
```

## 構成要素

| 要素 | 内容 | スタイル |
|---|---|---|
| うさぎ AA | 3 行のアスキーアート `(\(\` / `(='-')` / `o(_(")(")`| マゼンタ・太字 |
| タイトル | `USAGI` | 緑・太字 |
| メニュー | Open / New / Config / Quit の 4 項目。各行は「カーソル + ラベル(10桁左詰め) + キー(5桁右詰め)」 | 選択行：`>` 赤太字／ラベル シアン太字／キー 黄色。非選択行：素のテキスト |
| 通知行 | メニュー操作の結果メッセージ（例: New 完了後の `Opened "repo" 🐰`）。未表示時は何も描画しない | 黄色 |
| フッター | バージョンと操作ヘルプ `v<version> \| ↑↓: move / Enter: select / q: quit` | 淡色（dim） |

## メニュー項目

| ラベル | キー | 動作 | 状態 |
|---|---|---|---|
| Open | `o` | プロジェクト選択画面へ遷移 | ✅ 実装済み |
| New | `e` | 新規プロジェクト画面へ遷移 | ✅ 実装済み |
| Config | `c` | 設定画面（**グローバルスコープ**）へ遷移。アプリ全体の設定を編集する | ✅ 実装済み |
| Quit | `q` | アプリを終了する | ✅ 実装済み |

## キー操作

| キー | 動作 |
|---|---|
| `↑` / `k` | 選択を 1 つ上へ移動（先頭で押すと末尾へラップ）。通知をクリア |
| `↓` / `j` | 選択を 1 つ下へ移動（末尾で押すと先頭へラップ）。通知をクリア |
| `Enter` | 選択中の項目を実行。Open / New / Config は対応する画面へ遷移、Quit なら終了 |
| `o` / `e` / `c` | 対応するメニュー項目を直接実行（ショートカット） |
| `q` / `Esc` / `Ctrl+C` | アプリを終了 |

## 終了時の挙動

- 正常終了（Quit / `q` / `Esc` / `Ctrl+C`）時：代替スクリーンを解除しカーソルを復帰したうえで、
  別れのメッセージ `USAGI run away ( ^-^)ノ` を表示します。
- キー読み取りが中断された場合（シグナル等）：終了として扱います。
- キー読み取りでエラーが発生した場合：別れのメッセージは出さずに端末を復帰し、エラーを返します。

これらの端末復帰（代替スクリーンの解除・マウスレポートと代替スクロールモードの復帰・カーソルの
復帰）は `AlternateScreenGuard`（RAII ガード）により、早期 return やパニック時でも保証されます。

> 描画ロジックは `src/presentation/tui/welcome/`（メニュー定義 `menu.rs`・描画 `ui.rs`・
> イベントループ `event.rs`）に実装されています。
