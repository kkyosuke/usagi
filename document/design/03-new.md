# 3. 新規プロジェクト画面（New）

> [画面設計トップ](README.md) ｜ ← 前へ [2. プロジェクト選択画面（Open）](02-open.md) ｜ 次へ → [4. 設定画面（Config）](04-config.md)

起動画面で `New`（`e` / Enter）を選ぶと遷移する、プロジェクトを追加するフォーム画面。
**作成方法（Type）を上部のタブで切り替えられます。**

- **Clone** — Git リポジトリをクローンして新しいディレクトリを作成する。VS Code の
  「Clone Git Repository…」や IntelliJ の「Get from Version Control」を参考に、
  **URL を入力するとディレクトリ名が自動で導出される** UX を採用しています。
- **Existing** — 既にローカルに存在するディレクトリをワークスペースとして登録する。
  クローンは行わず、ディレクトリの末尾セグメントからワークスペース名を自動導出します。

起動画面と同じ代替スクリーン上に描画されます。

## レイアウト

フォーム全体を 1 つのブロック（幅 52）として端末中央に寄せ、ブロック内は左揃えで描画します。
先頭にモード切り替えタブを置き、タブの状態に応じて以降の入力フィールドが切り替わります。

うさぎ AA は[起動画面（Welcome）と共通の行](01-welcome.md#レイアウト)に置くため、画面を行き来しても
位置がガタつきません（レイアウトシフトなし。フォームが収まらないほど縦に短い端末でのみ引き上げられます）。

```text
┌─────────────────────────────────────────────────────────────────┐
│                       (\(\                                      │  ┐ うさぎ AA（マゼンタ・太字）
│                      (='-')                                     │  │
│                      o(_(")(")                                  │  ┘
│                    New Project                                  │  ← タイトル（緑・太字）
│   Clone a repository or register an existing directory          │ ← サブタイトル（淡色）
│                                                                 │
│        Type  (←→ to switch)                                     │  ← モード選択ラベル（淡色）
│      > [Clone]   Existing                                       │  ← タブ（選択中: 角括弧＋シアン太字／フォーカス時: 赤">"）
│                                                                 │
│        Repository URL                                           │  ← ラベル（淡色）
│      > https://github.com/owner/repo.git█                       │  ← 入力欄（フォーカス: 赤">" + シアン太字 + ブロックカーソル）
│                                                                 │
│        Location  (Space to browse)                              │
│        /Users/you/git                                           │  ← クローン先のベース（既定値を事前入力／Space でブラウザ）
│                                                                 │
│        Directory                                                │
│        repo                                                     │  ← URL から自動導出（シアン）
│                                                                 │
│        Branch (optional)                                        │
│        repository default                                       │  ← 未入力時はプレースホルダー（淡色斜体）
│                                                                 │
│        that does not look like a repository URL                 │  ← 通知行（任意・赤太字／バリデーション失敗時）
│                                                                 │
│ ←→ type · ↑↓/Tab field · Space browse · Enter create · Esc back │ ← フッター（淡色）
└─────────────────────────────────────────────────────────────────┘
```

`Existing` に切り替えると、URL / Location / Branch の代わりに次の 2 フィールドを表示します。
モードを切り替えてもフィールド数の差でレイアウトがずれないよう、フィールドブロックの高さは
常に背の高い Clone モード分だけ確保され、ヘッダー・モードタブ・フッターの位置は固定されます。

```text
│        Directory  (Space to browse)                  │
│        /path/to/an/existing/project                  │  ← 既存ディレクトリのパス（Space でブラウザ）
│                                                      │
│        Name                                          │
│        project                                       │  ← パスの末尾セグメントから自動導出（シアン）
```

## 入力フィールド

モードタブはどちらのモードでも先頭に表示され、Tab 巡回の起点になります。

**Clone モード**

| フィールド | 役割 | 備考 |
|---|---|---|
| Repository URL | クローン対象の Git URL | HTTPS / `git@host:owner/repo.git`（scp 形式）/ `ssh://` に対応 |
| Location | クローン先のベースディレクトリ | 設定の `workspace_root`（未設定なら `~/git`）を既定値として事前入力。手動編集も可能。最終的なクローン先は `<Location>/<Directory>` |
| Directory | クローン先ディレクトリ名 | URL の末尾セグメント（`.git`・末尾スラッシュ除去）から自動導出。手動編集も可能 |
| Branch (optional) | チェックアウトするブランチ | 未入力時はリポジトリのデフォルトブランチ |

**Existing モード**

| フィールド | 役割 | 備考 |
|---|---|---|
| Directory | 登録する既存ディレクトリのパス | 既にローカルに存在するディレクトリを指定する |
| Name | ワークスペース名 | パスの末尾セグメントから自動導出。手動編集も可能 |

## 名前の自動導出（エディタ準拠の挙動）

Clone モードの **Directory**、Existing モードの **Name** はどちらも同じ挙動で自動導出されます。

- Clone: URL を入力するたびに Directory を `末尾セグメント` から再導出します
  （例: `https://github.com/owner/repo.git` → `repo`）。
- Existing: Directory（パス）を入力するたびに Name を末尾コンポーネントから再導出します
  （例: `/home/me/my-app` → `my-app`）。
- ユーザーが導出先フィールドを手動で編集すると自動導出を停止し、以降の元フィールド変更で上書きしません。
- 導出先フィールドを空になるまで削除すると自動導出が再び有効化され、次の元フィールド変更で復活します
  （その場で即座に再充填はせず、ユーザーが独自名を入力できる余地を残します）。
- Clone で URL の末尾がスラッシュの場合（例: `owner/` 入力途中）はセグメントが未確定とみなし、何も提案しません。

## キー操作

入力フィールドはすべて共通の入力ウィジェットで編集され、キャレット位置での挿入・削除と左右移動に対応します（マルチバイト文字でも文字単位でずれません）。キャレットはカーソル位置の文字を反転表示するブロックカーソルで描画し（行末では空白 1 マスを反転）、移動しても後続テキストはずれません。あわせて端末の実カーソルもブロックカーソルの位置（全角文字を含む表示幅で算出）へ移動・表示するため、日本語変換中の未確定文字列（IME のプリエディット）も入力欄の位置に表示されます。

| キー | 動作 |
|---|---|
| `←` / `→` | モードタブにフォーカス中は作成方法（Clone / Existing）を切り替え。テキストフィールドにフォーカス中はキャレットを 1 文字左／右へ移動 |
| `Home` / `End` | フォーカス中のフィールドのキャレットを行頭／行末へ移動 |
| 文字キー | フォーカス中のフィールドのキャレット位置に挿入（モードタブにフォーカス中は無視） |
| `Backspace` | キャレットの前の 1 文字を削除 |
| `Del` | キャレット位置の 1 文字を削除 |
| `Space` | ディレクトリ系フィールド（Clone の Location / Existing の Directory）にフォーカス中はディレクトリ選択モーダルを開く。それ以外のフィールドではキャレット位置に空白文字を挿入 |
| `Tab` / `↓` | 次のフィールドへフォーカス移動（先頭のモードタブを含めて巡回） |
| `Shift+Tab` / `↑` | 前のフィールドへフォーカス移動 |
| `Enter` | フォームを検証して確定。失敗時は赤い通知を表示 |
| `Esc` | 起動画面へ戻る |
| `Ctrl+C` | アプリを終了 |

## バリデーションと確定後の挙動

**Clone モード**

- URL が空、または URL らしき形（パス区切りと末尾セグメント）を持たない場合は確定せず、
  通知行にエラーメッセージを表示します。
- Location が空の場合も確定せず、`choose where to create the project` を通知します。
- 検証に成功すると、その場でリポジトリのクローンとワークスペース登録を実行します（`usecase::project::create`）。
  具体的には次を行います。
  1. クローン先 `<Location>/<Directory>` が既に存在する場合は中断し、既存チェックアウトを壊さない
  2. `git clone`（Branch 指定時は当該ブランチをチェックアウト）でリポジトリを取得
  3. クローンしたディレクトリをワークスペースとして `workspaces.json` に登録（`usecase::workspace::add`）
  4. `<repo>/.usagi/state.json` に初期 worktree 状態を同期（`usecase::workspace_state::sync`）

**Existing モード**

- Directory（パス）が空の場合は確定せず、`choose an existing directory` を通知します。
- パスから名前が導出できず Name も空の場合は `enter a name for the workspace` を通知します。
- 検証に成功すると、その場で既存ディレクトリをワークスペースとして登録します（`usecase::project::register_existing`）。
  1. 指定パスがディレクトリでない（存在しない）場合は中断し、`<path> is not a directory` を通知
  2. ディレクトリを Name でワークスペースとして `workspaces.json` に登録（`usecase::workspace::add`）
  3. **Git リポジトリの場合のみ** `<repo>/.usagi/state.json` に worktree 状態を同期。
     Git 管理下にない素のディレクトリはクローン同様に登録しつつ、同期はスキップします

- どちらのモードも、成功すると起動画面へ戻り `Opened "<name>" 🐰` の通知を表示します。失敗した場合
  （不正な URL・ネットワーク・名前衝突・存在しないパスなど）は `Could not create project: <理由>` を通知し、
  メニューは維持されるためユーザーは修正して再試行できます。

## ディレクトリ選択モーダル

ディレクトリ系フィールド（Clone の **Location** / Existing の **Directory**）にフォーカス中に
`Space` を押すと、ファイルシステムを検索・ブラウズしてディレクトリを直感的に選べる
モーダルダイアログを画面中央に重ねて表示します。フォームの上に枠付きボックスとして描画され、
確定・キャンセルすると元のフォームへ戻ります。

```text
┌─ Select directory ───────────────────────────────────────┐
│ /Users/you/git                                           │  ← 現在ブラウズ中のディレクトリ（シアン太字）
│ Search: pro▏                                             │  ← 検索クエリ（インクリメンタル絞り込み）
│                                                          │
│ > projects/                                              │  ← カーソル行（赤">" + シアン太字）
│   prototypes/                                            │  ← サブディレクトリ一覧（末尾 "/"）
│   … 3 more                                               │  ← 表示しきれない件数（10 件超でスクロール）
│                                                          │
│ ↑↓ move · → open · ← up · Enter select · Esc cancel      │  ← 操作ヒント（淡色）
└──────────────────────────────────────────────────────────┘
```

| キー | 動作 |
|---|---|
| 文字キー | 検索クエリに追記し、現在ディレクトリ直下のサブディレクトリを大文字小文字無視の部分一致で絞り込み |
| `Backspace` | 検索クエリの末尾を 1 文字削除 |
| `↑` / `↓` | 一致した一覧内でカーソルを移動 |
| `→` | カーソル中のサブディレクトリへ降りる（その階層の中身を表示。クエリはリセット） |
| `←` | 親ディレクトリへ上がる |
| `Enter` | **現在ブラウズ中のディレクトリ**を選択してフィールドへ反映し、モーダルを閉じる |
| `Esc` | 選択せずにモーダルを閉じる |
| `Ctrl+C` | アプリを終了 |

- 開始ディレクトリは、フィールドに既に値があればその値、空なら既定の Location、それも空なら
  ファイルシステムのルート（`/`）です。
- 一覧できないディレクトリ（権限不足など）に入った場合は一覧を空にし、`⚠ <理由>` を表示します。
- 一覧表示エリアの高さは常に固定（最大 10 行 + スクロール時の「… N more」1 行分）です。
  一致件数・空・エラー・スクロールのいずれの状態でも同じ高さを確保するため、絞り込みで件数が
  変わってもモーダルの枠が伸縮せず、中央寄せ位置が上下にジャンプしません（CLS の解消）。
- 選択した結果は Existing の Directory ではワークスペース名（Name）の自動導出にも反映されます。
- モーダル本体（枠付きボックス）と検索リストは再利用可能なウィジェットとして
  `src/presentation/tui/widgets/`（`boxed` / `render_modal` / `picker` / `dir_picker`）に実装されています。

> 描画ロジックは `src/presentation/tui/new/`、URL のパース・ディレクトリ名導出は
> `src/domain/repository.rs`、クローン / 既存登録 + 同期の一連の処理は `src/usecase/project.rs`
> （Git 操作・リポジトリ判定は `src/infrastructure/git.rs`）に実装されています。
> ディレクトリ選択モーダルは `src/presentation/tui/widgets/dir_picker.rs`
> （汎用の検索リストは `picker.rs`、モーダル枠は `widgets/mod.rs` の `boxed` / `render_modal`）にあります。
