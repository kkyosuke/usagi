# コマンドモードの入力候補・ヒント表示

## 概要

ホーム画面のコマンドモード（`:` 入力）は、これまで入力欄とフッターのヘルプだけで、
どんなコマンドが使えるか・どんな引数を取るかは `man` を実行しないと分かりませんでした。
本 issue では、入力欄の**直上**に入力中の内容に応じた候補・ヒントをリアルタイム表示し、
コマンドを覚えていなくても操作できる導線を追加します。

候補・ヒントは #002 のコマンドレジストリ（`CommandRegistry`）と #008 で各コマンドが宣言する
`description` / `usage` / `examples` をそのまま流用します（新しい情報源は増やしません）。

## やること

- `CommandRegistry::suggest(input)` を追加し、入力に応じた `Hint` を返す。
  - コマンド語を入力中（空白なし）: 名前が前方一致するコマンドを `Hint::Commands`（名前＋説明）で返す。入力が空なら全コマンド。
  - 引数を入力中（コマンド語のあとに空白）: 解決したコマンドの `Hint::Usage`（`usage` / `examples`）を返す。未知のコマンドは `Hint::None`。
- `HomeState::hint()` で現在の入力に対する `Hint` を公開する（純粋・描画用）。
- `ui.rs` で入力欄の直上にヒント行を描画する。
  - コマンド一覧: ヘッダー（空入力なら `commands`、絞り込み中は `matches`）、最有力候補に `›` マーカー、入力済み接頭辞を太字強調、表示上限（6 件）超過は `… and N more`。
  - 引数ヒント: `usage <usage>` と `e.g. <example>`。
  - ヒント行は本体（2 ペイン）の表示行を借り、本体には最低 1 行を残す。画面全体の高さは不変。
- サイドバーモード・`terminal` 実行中は非表示。

## 完了条件

- コマンドモードで文字を打つと、入力欄の直上に一致するコマンド候補（名前＋説明）がリアルタイム表示される。
- コマンド語のあとに空白を入れると、そのコマンドの `usage` と `examples` が表示される。
- ヒント表示で画面の高さが変わらず、本体ペインが完全に潰れない。
- `domain` への依存追加なし。ロジックは純粋でテスト可能（カバレッジ 100% 維持）。
