## 概要

v2（`crates/`）のターミナル/エージェント画面に表示された **http(s) URL を検出し、安全性を検証する純粋コア**を追加する。これはクリックでブラウザを開く機能（→ #389）の土台となるレイヤーで、UI 配線やブラウザ起動は含まない。純粋関数＋100% 単体テストで完結させ、`pub` API として公開する（未使用でも lib crate の `pub` は dead_code 警告にならないため単独でマージ可能）。

## 背景（なぜ新規か）

- 出力中 URL を開く機能は **v1 では実装済み（#43 = done）**。実体は `v1/src/presentation/tui/home/terminal/link.rs`（クロスプラットフォーム・テスト済み）で、リリースが出荷しているのは今も v1。
- しかし **v2 の書き直し（`crates/`）ではこの機能が欠落**しており、v2 には汎用の URL 検出器が存在しない（`regex` 依存も無し）。本 issue は v1 の純粋検出器を v2 のデータモデルへ移植する。
- v2 の URL ロジックは PR URL 正規化（`usagi_core::domain::pr_inventory::canonicalize`、`crates/tui/src/usecase/application/pr.rs:73` の `canonical_browser_url`）のみで、これは **GitHub PR URL 専用**。汎用リンクには使えない。

## やること

新規純粋モジュール（例: `crates/tui/src/usecase/application/terminal_link.rs`）を追加し、v1 `link.rs` の純粋部分を移植する。ただし入力は v2 の **ANSI を含まないグリッド** `TerminalScreen::cells()` / `cells_with_scrollback()`（`Vec<String>`、`crates/tui/src/usecase/application/terminal_screen.rs:277/292`）とし、`vt100::Screen` へは依存しない。

- **クリック位置の URL 抽出**: あるセル座標（行・表示列）がどの http(s) URL 上に載っているかを返す（クリックで開く用）。
- **全画面リンクセル走査**: 画面（＋スクロールバック可視範囲）上のすべての URL が占めるセル集合を返す（下線などの可視化用）。ホバー中の 1 リンクだけを返す派生も用意する。
- **行折り返しの結合**: URL が行末で折り返して次行へ続く場合、論理行として 1 本の URL に結合する（v1 `logical_line` 相当。v2 では `Cell.continuation`／グリッド幅から論理行を組み立てる）。
- **span 抽出**: 空白で run を分割し、`https://` / `http://` の位置から URL 文字が続く限り伸ばす（v1 `url_spans`）。
- **URL 文字判定**: `is_ascii_graphic()`（印字可能 ASCII のみ。空白・制御文字は含まない）。CJK・全角記号（`（` `、` `。` など）が空白なしで隣接した場合はそこで URL を終端する（v1 `is_url_char`）。
- **末尾トリム**: 文末記号・引用符（`. , ; : ! ? > " ' \``）は常に落とし、閉じ括弧 `)` `]` `}` は不均衡なときだけ落とす（`..._(disambiguation)` のような対を保つ。v1 `trim_trailing`）。
- **スキーム限定**: 認識するのは `https://` / `http://` のみ。ホスト無しの裸スキームは URL としない。

### セキュリティ検証（本 issue の要点）

ブラウザ起動へ渡す前段の防御として、検出結果を厳格に検証する `validate`/`canonicalize` 相当の純粋関数を用意する（起動側 #389 がこれを defense-in-depth で再利用する）。

- **スキーム allowlist**: `http` / `https` 以外（`javascript:` / `file:` / `data:` / `mailto:` / 任意のカスタムスキーム）は一律拒否する。検出器がスキームを http(s) に限定しているため通常は混入しないが、起動直前にも明示検証する。
- **制御文字・エスケープの排除**: URL に制御文字（`0x00–0x1F` / `0x7F`）・ESC・改行・空白が含まれるものは拒否する（`is_ascii_graphic` により検出段階で排除されるが、検証関数でも明示的に弾き、ANSI エスケープ列やターミナル制御列がブラウザ引数に渡らないことを保証する）。
- 検証は「検出済み文字列 → 開いてよい URL か Option/Result」で表現し、拒否理由がテストで固定できる形にする。

## 対象外（このあと #389 で扱う）

- マウスクリックのヒットテスト配線・選択ドラッグとの両立・ブラウザ起動（`BrowserOpener`）・可視化描画・docs 更新。本 issue は純粋な検出＋検証ロジックとそのテストのみ。
