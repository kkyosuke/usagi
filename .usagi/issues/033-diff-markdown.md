# 右ペイン統合プレビュー（diff / Markdown / ファイル）

## 概要

ホーム画面の右ペインを、ライブターミナルのプレビューだけでなく、**選択中セッションの差分（diff）や
Markdown ファイルをレンダリング表示できる統合プレビュー**に拡張します。AI エージェントが何を変更したか、
あるいは `README.md` などのドキュメントの内容を、TUI から離れずに右ペインで確認できるようにします。

現状、右ペインは「コマンド履歴/出力」か「ライブターミナル（読み取り専用プレビュー）」に切り替わります
（[document/design/05-home.md](../document/design/05-home.md) の右ペイン仕様）。これに **プレビューモード**を
追加し、ブラウザのタブを切り替えるように右ペインの表示内容を選べるようにします。

- **diff 表示**: アクティブセッションと main の差分を右ペインに表示する。フルスクリーンの Diff ビューア
  （[012-diff](012-diff.md)）と表示ロジックを共有し、右ペインでは要約版を表示する。
- **Markdown 表示**: `.md` ファイルを右ペインで整形（見出し・リスト・コードブロック・強調などの簡易
  レンダリング）して表示する。今回追加したい中心機能。

## やること

- `preview` コマンド（あるいは既存の右ペイン切り替えキー）でプレビューモードへ入り、対象を選べるようにする。
  - `preview diff` — アクティブセッションと main の差分（ファイル単位の一覧 + ハンク、追加/削除行の色分け）。
  - `preview <path.md>` / `preview README` — 指定 Markdown ファイルを右ペインにレンダリング表示。
- Markdown の簡易レンダリングを実装する（見出し・箇条書き・番号付きリスト・コードブロック・インライン
  コード・強調/太字・リンクテキスト程度。装飾は ratatui の Span/Style で表現）。
- diff 表示は [012-diff](012-diff.md) の生成・色分けロジックを再利用し、右ペイン向けの描画にまとめる。
- 右ペインの 3 状態（コマンド履歴/出力・ライブターミナル・プレビュー）の切り替えとフォーカス・スクロール
  方針を [document/design/05-home.md](../document/design/05-home.md) の方針（TUI 自体はスクロールしない）に
  沿って整理する。プレビュー（diff / Markdown）は内容が縦に長くなり得るため、**プレビューペイン内のスクロール**を
  どう扱うか（埋め込みターミナルと同様にペイン内スクロールを許可するか）を本 issue で決める。
- 純粋なレンダリングロジック（Markdown → Span 列、diff → 行列）を `presentation/tui/` 配下のテスト可能な
  関数に切り出し、カバレッジ 100% を維持する。Git 差分取得は infrastructure 層に置く。

## 完了条件

- 右ペインで `.md` ファイルを整形表示できる（見出し・リスト・コードブロック・強調が視認できる）。
- 右ペインでアクティブセッションと main の差分を色分け表示できる。
- 右ペインの表示モード（履歴/出力・ライブターミナル・プレビュー）を破綻なく切り替えられ、レイアウトが
  ずれない（CLS が起きない）。
- レンダリングロジックにテストが追加され、カバレッジ 100% を満たす。
