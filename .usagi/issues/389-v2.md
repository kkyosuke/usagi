## 概要

v2（`crates/`）のターミナル/エージェント右ペインで、表示中の **http(s) URL をクリックすると OS 既定ブラウザで開ける**ようにする。URL 検出・検証は #387 の純粋コアを使い、本 issue は**ヒットテスト配線・選択ドラッグとの両立・クロスプラットフォームなブラウザ起動・可視化・失敗時フィードバック・回帰テスト・docs 更新**を担う。これで v1（#43 = done）が出荷している機能を v2 へ復帰させる。

Agent ペインと Terminal ペインは同一の描画/PTY パイプライン（`TerminalScreen`/`TerminalSession`）を通るため、共有経路に実装すれば両方に効く。

## 依存

- **#387**（URL 検出・検証コア）に依存。検出・検証はそちらの純粋関数を呼び出す。

## やること

### 1. クリック→リンクオープンのジェスチャ配線

- 左クリック（押下→ドラッグ無し→離す）で、クリックセルを `terminal_point_at`（`crates/tui/src/presentation/views/workspace.rs:901`、純粋・テスト済み）で保持行/列へ変換し、#387 の検出関数で URL を特定して**開くエフェクト**を発火する。
- **選択ドラッグとの両立（最重要）**: ドラッグで非空の選択が成立していればそれは**コピー**、選択が生じない素のクリックだけを**リンクオープン**とする（v1 `pane.rs` `open_clicked_url` の規則: 非空選択があれば開かない）。現状 v2 のポインタ処理は `handle_terminal_pointer`（`crates/tui/src/presentation/mod.rs:1517`）／`intercept_live_terminal_control`（同:1559）にあり、`PointerKind::Up` でコピー確定している。ここに「選択が空なら URL ヒットテスト→オープン」を足す。
- **入力誤送信の防止**: クリック/ポインタ列は既に分類器で `Swallowed`（`crates/tui/src/usecase/terminal_input.rs:229`）となり子 PTY へバイト転送されない。この不変を維持し、リンクオープンのクリックがシェル/エージェントへ入力として渡らないことをテストで固定する。
- URL 上でないセルのクリックは無害な no-op（何も開かず、選択・スクロールにも影響しない）。

### 2. クロスプラットフォームなブラウザ起動（既存機構の再利用＋拡張）

- 既存の `BrowserOpener` トレイト（`crates/tui/src/usecase/application/pr.rs:21`、「実装はシェルを起動してはならない」）とエフェクト経路を再利用する。ただし現状の `canonical_browser_url`（同:73）は **GitHub PR URL 専用**なので、汎用 URL 用に **#387 の検証関数**を使う開くエフェクト（例: `Effect::OpenUrl { url }`）とそのハンドラを追加する（PR オープン経路 `controller_open_pull_request`／`mod.rs:1823` と同型で、`browser.open()` 失敗時に `BackendEvent::Notice` を返す）。
- 実アダプタ `PlatformBrowserOpener`（`src/runtime/tui.rs:1378`、`#[coverage(off)]`）は現在 **macOS `open` / Linux `xdg-open` のみで Windows 非対応**。v1 `open_command`（`v1/.../terminal/link.rs:404`）に倣い **Windows `cmd /C start "" <url>`**（空文字はタイトル引数）を追加する。いずれも argv で spawn し、**URL 引数をシェルに解釈させない**（制御文字・エスケープは #387 で排除済み＋ argv 渡しで二重防御）。
- 未対応プラットフォーム／spawn 失敗は**安全なフィードバック**（既存の "Could not open browser: …" 系 Notice）に落とし、シェルやエージェントを乱さない。

### 3. 可視化（クリック可能アフォーダンス）

- 検出済みリンクセルに**下線**などの装飾を付け、クリック可能であることを示す（#387 の全画面リンクセル走査を使い、`render_row`/`render_row_selected`（`terminal_screen.rs:712/727`）の styled-row 投影を拡張）。選択のインバース表示と両立させる。ホバーで当該リンクだけ強調する派生があれば望ましいが、少なくとも常時下線の可視化を satisfy する。

### 4. docs 更新

- `document/` 配下の該当ドキュメント（ターミナル/エージェント TUI の操作説明）へ、クリックでリンクを開く挙動・選択コピーとの使い分け・対応 OS を追記する。**「記載＝実装済み」**の規約に従い、本 PR で実装した挙動のみ書く。ユーザー向け変更なら `README.md` も更新する。

## 回帰させないこと（テスト観点）

以下が壊れないことを、純粋部分の単体テスト＋ fake（`BrowserOpener`/`ClipboardPort` の偽実装）による経路テストで固定する。

- テキスト選択ドラッグ→コピー（#53）／ドラッグ中はリンクを開かない。
- マウスホイールスクロール・スクロールバック（10k 行）オフセット時のヒットテスト整合。
- CJK/全角混在・ANSI(SGR) 装飾付き行での検出とセル対応。
- 行末で折り返した長い URL を 1 本として開く。
- 同一リンクを繰り返しクリックしても毎回開ける（状態が消費されない）。
- ブラウザ起動失敗／未対応プラットフォームで安全なフィードバックになり、クラッシュ・入力乱れが起きない。
- Agent ペイン・Terminal ペインの双方でクリックオープンが効く。
- 子 PTY へクリックが入力として転送されない。

## 完了条件

- 画面上（Agent/Terminal 両ペイン）の http(s) URL をクリックすると OS 既定ブラウザで開く。
- macOS / Linux / Windows で開ける（argv 渡し・シェル非経由）。
- ドラッグ選択→コピーが従来どおり動き、選択中はリンクを開かない。
- リンクが下線などでクリック可能と分かる。
- ブラウザ起動失敗時に安全なフィードバックを返し、クリックが子 PTY へ入力送信されない。
- 上記「回帰させないこと」がテストで固定され、追加コードのカバレッジ 100%（実 IO アダプタのみ `#[coverage(off)]`、選択ロジックは fake で検証）。
