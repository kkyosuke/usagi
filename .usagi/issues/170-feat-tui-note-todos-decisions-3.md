## 目的

セッションメモのオーバーレイ（選択の `n` / `Ctrl-E`、集中・没入の `Ctrl-E`）は現在 3 つのタブ（note / todos / decisions）を `Tab` で循環切り替えして 1 つずつ表示している（#157）。切り替えないと他の区画が見えないため、**3 つを 1 つのオーバーレイ内に縦積みで同時に表示**し、いま何が書かれているかを一覧できるようにする。そのうえで**どのペインが編集対象（フォーカス中）かが一目で分かる**強調を付ける。

## 変更内容

- **同時表示（縦積み）**: `note` → `todos` → `decisions` の 3 ボックスを右ペイン右上の同じ列（既存の `note_box_width` の幅）に縦に積んで描画する。背後のプレビュー / 端末はこれまでどおりボックスの左と下に見えたまま（合成は既存の `overlay_right` の流儀）。
- **フォーカス移動**: `Tab` / `Shift-Tab` は「タブ切り替え」ではなく**フォーカス（編集対象）の移動**として同じ循環順（note → todos → decisions）を維持する。キー割り当て・保存（`Ctrl-S`）・取消（`Esc`）は変えない。
- **フォーカスの強調**: フォーカス中のペインは**アクセント色の枠＋タイトルに状態表記**（`note (編集中)` / `todos (編集中)` / `decisions (表示中)`）。非フォーカスのペインはプレーンな枠＋素のタイトル。`NO_COLOR` 環境でもタイトル表記だけで判別できる（現行の「編集中」表記の流儀を踏襲）。
- **高さ配分**: 右ペインの高さから 3 ボックスの枠ぶんを引いた予算を、非フォーカスペインへ小さいキャップ（読み取り表示は超過分を `… (N more)` で省略）、フォーカスペインへ残り（note は現行どおりキャレット追従の窓）で配分する。**3 ボックスが最低 1 行ずつも入らない低い端末では、フォーカス中のペイン 1 つだけの表示にフォールバック**する（現行の単独ボックスと同じ見え方）。
- **各ペインの操作は現行どおり**: note は自由編集（`Ctrl-S` 保存）、todos は対話編集（`j`/`k` 移動・`Space` トグル・`a` 追加・`e` 編集・`d` 削除・インライン入力）、decisions は読み取り専用。フッターの操作ヒントはフォーカス中ペインに応じて出す。todos の選択マーカー `›` はフォーカス中だけ表示する。
- MCP（`session_note_*` / `session_todo_*` / `session_decision_*`）と保存フォーマットは変更しない（表示・編集 UI のみ）。

## 実装箇所

- 状態: `home/state/modal.rs`（`NoteTab` → フォーカスの意味に。`NoteEditor` の循環はそのまま）
- 描画: `home/ui/panes.rs`（`note_overlay` を 3 ボックス縦積みに、高さ予算・フォールバック）
- フッター: `home/ui/chrome.rs`（ヒント文言）
- キー処理: `home/event/handlers.rs`（挙動は同じ。コメント・doc の更新）

## テスト・確認

- 3 ペインが同時に描画されること、フォーカス中ペインだけがアクセント枠＋状態表記になること。
- `Tab` / `Shift-Tab` でフォーカスが循環し、キー入力がフォーカス中ペインへ届くこと（note 編集・todos 対話編集・decisions は編集不可）。
- 低い端末（3 ボックスが入らない高さ）でフォーカス中ペインのみの表示にフォールバックすること。非フォーカスペインの `… (N more)` 省略。
- カバレッジ 100% 維持。`document/design/home/05-overlays.md`（セッションメモの編集）と参照元（02-layout.md 等）の更新。
