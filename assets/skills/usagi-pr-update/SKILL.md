# PR を更新する

すでに出している PR に変更を重ねた後で、PR の概要とレビュー対応を最新の状態に揃える手順。プログラミング言語に依存しない共通ルール。

## 前提: gh が認証で使えないとき

このスキルの手順は `gh` を使う。まず `gh auth status` を確認し、通っていればそのまま `gh` を使う。未認証・認証エラーで使えない場合は、既に `GH_TOKEN` または `GITHUB_TOKEN` が環境変数として渡されていないか確認する。どちらも無ければ、ユーザーに `gh auth login` を促して中断する（トークンを勝手に探さない）。

環境変数でトークンが渡されている場合は、そのまま `gh` を実行する。

   ```bash
   # GH_TOKEN を安全な方法で環境変数にセット済みの状態で実行する
   gh pr edit <番号> --title "…" --body "…"
   gh pr comment <番号> --body "…"
   ```

- トークンは PR 本文・コメント・コミット・ログ・コマンド履歴に残さない。値をそのまま貼り付けたり表示したりしない。

## 1. 概要（本文）を更新する

PR の本文を、**新規作成と同じルール**で現在の実態に合わせて書き直す。ルールの詳細は `usagi-pr-create` スキルに従う。要点だけ再掲する。

- タイトル・本文は**相手（チャット者）の言語**で書く。
- 依存する前段 PR があれば、本文の先頭にマージ順を明記する（依存が解消されたら消す）。
- 「何を実行したか / なぜ / 変更点 / 動作確認」の 4 見出しを、追加・変更した内容まで反映して更新する。
- スコープが変わったらタイトルも実態に合わせる。

```bash
gh pr edit <番号> --title "<更新後のタイトル>" --body "<更新後の本文>"
```

## 2. レビューに返信する

PR にレビュー（コメント・変更要求）が付いている場合は、**1 件ずつコメントで返信**する。黙って直すだけにしない。

1. 付いているレビューを洗い出す。

   ```bash
   gh pr view <番号> --comments
   # 行コメント（レビュースレッド）も含めて見る場合:
   gh api repos/{owner}/{repo}/pulls/<番号>/comments
   ```

2. それぞれのコメントに対して、相手の言語で返信する。返信には次を含める。
   - 対応したか／しないか（対応しない場合は理由）。
   - 対応した場合は、どう直したか（該当コミットや変更箇所への参照）。

   ```bash
   # PR 全体への返信
   gh pr comment <番号> --body "<返信>"
   # 個別のレビュースレッドへの返信
   gh api repos/{owner}/{repo}/pulls/<番号>/comments/<コメントID>/replies -f body="<返信>"
   ```

3. すべてのレビューコメントに返信が付いた状態にする。未対応のまま放置しない。

## 完了チェック

- [ ] 本文を相手の言語で、4 見出し（何を/なぜ/変更点/動作確認）すべて最新化した
- [ ] 依存・マージ順の記載が現状と合っている
- [ ] すべてのレビューコメントに返信した（対応可否と理由・対応内容を含む）
